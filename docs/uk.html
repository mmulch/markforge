Посібник користувача MarkForge v1.8.0

1. Вступ

MarkForge — це сучасний багатофункціональний редактор Markdown, створений на основі Python і PyQt6. Він поєднує зручне для зосередженого написання середовище з попереднім переглядом у режимі реального часу (панелі пліч-о-пліч), повноцінною інтеграцією робочого процесу Git, підтримкою діаграм, відтворенням математичних формул та імпортом/експортом PDF — усе в одному зручному настільному застосунку з двома панелями.

Незалежно від того, пишете ви технічну документацію, README для проектів або публікуєте вміст безпосередньо до репозиторію GitHub, MarkForge забезпечує повний цикл редагування та публікації без потреби відкривати термінал.

2. Встановлення

Linux (Debian / Ubuntu)

Завантажте пакет .deb зі сторінки Releases на GitHub, а потім встановіть його командою:

sudo apt install ./markforge_*.deb

Пакет автоматично встановлює всі необхідні системні залежності (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments). Віртуальне середовище не потрібне.

Windows

Завантажте MarkForge-Setup.exe зі сторінки Releases і запустіть інсталятор. Майстер налаштування проведе вас через процес встановлення та автоматично створить ярлики в меню «Пуск».

Примітка

Інсталятор Windows містить середовище виконання Python та всі залежності. Окреме встановлення Python не потрібне.

З вихідного коду

Вимоги: Python 3.8 або новіша версія, CMake 3.16 або новіша версія.

За допомогою CMake (рекомендовано):

# 1. Clone the repository
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Configure the build (creates a local virtual environment)
cmake -B build

# 3. Install Python dependencies into the local venv
cmake --build build --target install_deps

# 4. Run the application
cmake --build build --target run

Безпосередній запуск Python (без CMake):

pip install -r requirements.txt
python src/main.py
Порада

Для операцій Git через SSH також встановіть paramiko: pip install paramiko

3. Огляд інтерфейсу

Розмітка

Вікно MarkForge поділено на три панелі змінного розміру, розділені перетягуваними роздільниками:

ПанельРозташуванняОпис
Структура + Дерево файлівЛіворучЛівий розділ містить два складені розділи: Структура документа (дерево заголовків) вгорі та Дерево файлів (файловий браузер) внизу. Перемикання структури через Вигляд → Показати структуру, дерева файлів через Ctrl+B.
РедакторПо центруТекстовий редактор із підсвічуванням синтаксису Markdown та номерами рядків.
Попередній переглядПраворучЖиве відтворення HTML поточного документа. Перемикання за допомогою Ctrl+Shift+P.

Перетягніть будь-який роздільник для зміни розміру панелей. Розміри панелей зберігаються між сеансами.

Структура документа

Панель Структура документа відображає всі заголовки (H1–H6) у вигляді навігаційного дерева в лівому бічному панелі.

  • Натисніть на будь-який заголовок, щоб перемістити курсор редактора на цей рядок.
  • Структура автоматично оновлюється під час введення.
  • Перемикання видимості через Вигляд → Показати структуру. Налаштування зберігається між сеансами.

Панель інструментів

Панель інструментів під рядком меню забезпечує доступ одним клацанням до найпоширеніших дій: Новий, Відкрити, Зберегти, Вставити посилання, Вставити зображення, Вставити таблицю, Вставити PlantUML, Вставити Mermaid, Перемкнути попередній перегляд і Налаштування. Усі дії на панелі інструментів дублюють пункти меню та клавіатурні скорочення.

Рядок стану

Рядок стану внизу вікна завжди відображає:

  • Кількість слів & час читання — кількість слів і приблизний час читання (наприклад, 342 слова · 2 хв читання), оновлюється в реальному часі.
  • Прогрес цілі слів — якщо встановлено ціль за словами через Вигляд → Встановити ціль слів, з'являється компактна смуга прогресу.
  • Позиція курсору — поточний рядок і стовпець, наприклад: Рядок 12, Стовпець 5.

Заголовок вікна

У заголовку вікна відображається ім'я поточного відкритого файлу. Коли є незбережені зміни, до імені файлу додається зірочка (*), наприклад README.md *. Якщо файл не відкрито, у заголовку відображається Untitled.

4. Редактор

Центральна панель містить повнофункціональний текстовий редактор, оптимізований для написання Markdown.

Підсвічування синтаксису

MarkForge застосовує підсвічування синтаксису Markdown до документа під час введення. Заголовки, маркери виділення, посилання, фрагменти коду, огороджені блоки коду та інші конструкції Markdown мають різні кольори. Підсвічування автоматично адаптується під час перемикання між світлою та темною темами редактора.

Номери рядків

Поле ліворуч від редактора відображає номери рядків. Поточний рядок виділено ненав'язливим фоновим кольором, щоб допомогти вам відстежувати позицію у довгих документах.

Перенесення рядків

Перенесення рядків увімкнено за замовчуванням. Довгі рядки переносяться відповідно до видимої ширини редактора. Перемикання перенесення рядків у будь-який час доступне через Вигляд → Перенесення рядків. Налаштування зберігається між сеансами.

Теми редактора

Доступні дві вбудовані теми редактора:

  • Темна (за замовчуванням) — темний фон з контрастними кольорами синтаксису.
  • Світла — білий фон, що підходить для яскравих умов.

Переключення тем у Вигляд → Налаштування → Тема редактора. Зміна набирає чинності негайно без перезапуску.

Текст-заповнювач

Якщо файл не відкрито, редактор вимкнений і відображає повідомлення: «Відкрийте або створіть файл Markdown, щоб розпочати редагування.» Редактор активується, щойно ви створюєте новий файл або відкриваєте наявний.

Операції редагування

Доступні всі стандартні операції редагування тексту:

  • Ctrl+Z — Скасувати
  • Ctrl+Y — Повторити
  • Ctrl+X / Ctrl+C / Ctrl+V — Вирізати / Копіювати / Вставити

Знайти & Замінити

Натисніть Ctrl+F, щоб відкрити панель пошуку, або Ctrl+H — щоб відкрити її з рядком заміни. Панель з'являється над редактором.

  • Введіть текст у поле пошуку — усі збіги миттєво підсвічуються в редакторі.
  • Переміщайтеся між збігами за допомогою кнопок ↑ / ↓, або натисніть Enter (наступний) / Shift+Enter (попередній) у полі пошуку.
  • Лічильник збігів показує поточну позицію (наприклад 3 / 12).
  • Aa — увімкнути пошук з урахуванням регістру.
  • [W] — пошук лише цілих слів.
  • Натисніть Замінити, щоб замінити поточний збіг, або Замінити все — щоб замінити всі входження одразу.
  • Натисніть Escape або клацніть ✕, щоб закрити панель і прибрати всі підсвічення.

Якщо під час відкриття панелі в редакторі є виділений текст, поле пошуку автоматично заповнюється цим текстом (лише одно-рядкові виділення).

Перевірка правопису

MarkForge може підкреслювати неправильно написані слова червоною хвилястою лінією під час введення. Перевірка правопису вимкнена за замовчуванням.

  • Увімкнути через Вигляд → Перевірка правопису (перемикач).
  • Вибір мови словника через Вигляд → Мова перевірки правопису. Доступні мови: English, Deutsch, Español, Français, Italiano, Nederlands, Português, العربية, فارسی (через pyspellchecker) та Tiếng Việt*, Svenska*, Українська*, ಕನ್ನಡ*, हिंदी* (через enchant/Hunspell).
  • Слова всередині фрагментів вбудованого коду (`…`) та URL-адрес ніколи не підкреслюються.
  • Абревіатури ВЕЛИКИМИ ЛІТЕРАМИ (HTML, API, URL…) пропускаються.
  • Налаштування перевірки правопису та вибрана мова зберігаються між сеансами.
  • Потрібна бібліотека pyspellchecker. Якщо не встановлена, перемикач мовчки ігнорується.
  • Мови, позначені *, потребують бібліотеки pyenchant та системних словників Hunspell (наприклад, hunspell-vi у Debian/Ubuntu).

5. Попередній перегляд у реальному часі

Права панель відображає повністю відтворену HTML-версію вашого документа Markdown, що оновлюється автоматично під час введення.

Відтворення

Попередній перегляд використовує таблицю стилів, натхненну GitHub, для чистого та зручного для читання виведення. Він підтримує всі розширення Markdown, зокрема таблиці, списки завдань, виноски, підсвічування коду, математику та діаграми. Конвеєр відтворення обробляє текст через бібліотеку Python markdown з розширеннями extra, codehilite, toc і sane_lists.

Збереження позиції прокручування

Під час редагування документа попередній перегляд оновлюється після короткої затримки дебаунсингу 300 мс. Перед оновленням HTML MarkForge зчитує поточну позицію прокручування з веб-перегляду і відновлює її після відтворення, тому попередній перегляд не переходить до початку, поки ви працюєте всередині документа.

Теми попереднього перегляду

Доступні дві теми попереднього перегляду:

  • Темна (за замовчуванням) — темний фон, що відповідає загальній темі застосунку.
  • Світла — білий фон для досвіду читання, схожого на друкований текст.

Переключення через Вигляд → Налаштування → Тема попереднього перегляду. Зміна набирає чинності негайно.

Перемикання попереднього перегляду

Натисніть Ctrl+Shift+P або скористайтеся Вигляд → Показати попередній перегляд, щоб приховати або показати панель попереднього перегляду. Приховання попереднього перегляду надає редактору більше простору на екрані.

Примітка

Математичні формули (MathJax) і діаграми Mermaid потребують підключення до інтернету під час першого використання, оскільки бібліотеки завантажуються з CDN. Діаграми PlantUML завжди завантажуються з plantuml.com і також потребують доступу до інтернету.

6. Операції з файлами

Новий файл

Натисніть Ctrl+N або виберіть Файл → Новий. Якщо поточний документ містить незбережені зміни, MarkForge запропонує зберегти їх перед продовженням. Відкриється новий порожній документ і редактор буде активовано.

Відкрити файл

Натисніть Ctrl+O або виберіть Файл → Відкрити…, щоб відкрити діалог перегляду файлів. Можна вибрати файли Markdown (.md, .markdown), текстові файли (.txt) та всі інші файли. Дерево файлів оновиться, щоб показати теку з відкритим файлом.

Відкрити теку

Виберіть Файл → Відкрити теку… (Ctrl+Shift+O), щоб вибрати каталог. Дерево файлів відображатиме всі файли .md та зображення у цьому каталозі. Клацніть будь-який файл у дереві, щоб відкрити його в редакторі.

Зберегти

Натисніть Ctrl+S або виберіть Файл → Зберегти. Якщо файл було відкрито з репозиторію Git, замість цього з'явиться діалог commit & push Git (див. Збереження через Git). Для локальних файлів файл негайно записується на диск, а зірочка у заголовку вікна зникає.

Зберегти як

Натисніть Ctrl+Shift+S або виберіть Файл → Зберегти як…, щоб зберегти поточний документ в іншому місці або з іншою назвою.

Дерево файлів

Ліва панель відображає плаский список файлів .md і зображень у поточному кореневому каталозі. Підкаталоги, що не містять безпосередньо відповідних файлів, приховано для зменшення захаращення. Клацніть файл, щоб відкрити його. У заголовку дерева відображається [GIT] назва-репозиторію зеленим кольором, коли відкрито файл під управлінням Git.

Порада

Перемикайте дерево файлів за допомогою Ctrl+B, щоб максимально збільшити простір редактора. Ваше налаштування зберігається між сеансами.

Останні файли

MarkForge зберігає список 10 останніх відкритих файлів. Доступ через Файл → Останні файли.

  • Натисніть на будь-який запис, щоб негайно повторно відкрити файл.
  • Якщо файл більше не існує на диску, він автоматично видаляється зі списку.
  • Виберіть Очистити останні файли вгорі підменю, щоб очистити всю історію.
  • Список зберігається між сеансами.

Перетягування

Ви можете перетягувати файли з файлового менеджера на редактор. MarkForge відображає діалог для вибору способу обробки кожного типу файлу:

  • Файли зображень (.png, .jpg, .gif тощо) — оберіть вставку за оригінальним шляхом або скопіювати до папки assets/ й вставити відносне посилання.
  • Файли Markdown (.md, .markdown) — Відкрити файл з оригінального місця або Копіювати до папки (скопіювати в ту саму директорію, де знаходиться поточний документ, та відкрити копію).
  • PDF-файли — Імпортувати … (стандартний потік імпорту з діалогом збереження) або Імпортувати до папки (автоматично імпортує в директорію поточного документа).
  • Будь-який інший файл — вставляє посилання Markdown [ім'я файлу](шлях/до/файлу).
Примітка

Для операцій копіювання або імпорту в папку поточного документа файл повинен бути відкритий і збережений, щоб MarkForge знав цільову директорію.

Режим фокусування F11

Натисніть F11 або виберіть Вигляд → Режим фокусування, щоб увійти в режим письма без відволікань. MarkForge приховує дерево файлів, структуру документа, живий попередній перегляд і рядок стану, залишаючи редактор на весь екран.

Натисніть F11 ще раз, щоб вийти: всі панелі й рядок стану відновлюються точно так, як були до цього.

Порада

Режим фокусування працює незалежно від індивідуальних перемикачів панелей (Ctrl+B, Ctrl+Shift+P).

7. Довідник Markdown

MarkForge підтримує стандартний CommonMark Markdown плюс такі розширення: extra (таблиці, виноски, скорочення), codehilite (блоки коду із підсвічуванням синтаксису), toc (зміст), sane_lists і закреслення.

Заголовки

Markdown
# Heading 1 ## Heading 2 ### Heading 3 #### Heading 4 ##### Heading 5 ###### Heading 6
Відтворення

Heading 1

Heading 2

Heading 3

Виділення та вбудоване форматування

Markdown
**bold text** *italic text* ~~strikethrough~~ `inline code` **_bold italic_**
Відтворення
bold text
italic text
strikethrough
inline code
bold italic

Списки

Невпорядкований список:

Markdown
- First item - Second item - Nested item - Third item
Відтворення
  • First item
  • Second item
    • Nested item
  • Third item

Впорядкований список:

Markdown
1. First item 2. Second item 3. Third item
Відтворення
  1. First item
  2. Second item
  3. Third item

Список завдань (прапорці):

Markdown
- [x] Done task - [ ] Pending task - [ ] Another task
Відтворення
  • ✅ Done task
  • ☐ Pending task
  • ☐ Another task

Посилання та зображення

Markdown
[Link text](https://example.com) [Tooltip link](https://example.com "My tooltip") ![Alt text](image.png) ![Alt text](https://example.com/img.png)
Відтворення
Link text
Tooltip link

[image: Alt text]

Таблиці (GFM)

Markdown
| Left | Center | Right | |:-----|:------:|------:| | A | B | C | | D | E | F |
Відтворення
LeftCenterRight
ABC
DEF

Вирівнювання стовпців визначається положенням двокрапки у рядку розділювача: :--- ліворуч, :---: по центру, ---: праворуч.

Блоки коду

Markdown
```python def hello(name): print(f"Hello, {name}!") ```
Відтворення
def hello(name):
    print(f"Hello, {name}!")

Вкажіть будь-який ідентифікатор мови після відкриваючих потрійних зворотних лапок для підсвічування синтаксису: python, javascript, bash, sql, json тощо (за допомогою Pygments).

Інші елементи

Блочні цитати

Markdown

Відтворення
This is a blockquote. It can span multiple lines.

Виноски

Markdown
Some text[^1] with a footnote. [^1]: The footnote content.
Відтворення
Some text[1] with a footnote.

Скорочення

Markdown
The HTML spec is long. *[HTML]: HyperText Markup Language
Відтворення
The HTML spec is long.

Зміст

Розмістіть [TOC] будь-де в документі, щоб вставити автоматично згенерований зміст на основі заголовків документа.

Горизонтальна лінія

Markdown

Відтворення

8. Математичні формули

MarkForge відтворює математичну нотацію LaTeX за допомогою MathJax. Під час першого використання потрібне підключення до інтернету (MathJax завантажується з CDN).

Вбудована математика

Оберніть вираз одинарними знаками долара, щоб відобразити його вбудовано в абзаці:

The famous equation $E = mc^2$ was published in 1905.

Результат: The famous equation E = mc² was published in 1905.

Блочна математика (відображувана)

Оберніть вираз подвійними знаками долара, щоб відобразити його відцентрованим на окремому рядку:

$$\int_0^\infty e^{-x}\, dx = 1$$

Однорядкова блочна математика

Також можна використати одинарний $ на окремому рядку для відкриття та закриття блоку відображуваної математики:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
Порада

MathJax підтримує повний математичний режим LaTeX, зокрема дроби, матриці, грецькі літери, оператори та власні макроси. Зверніться до документації MathJax для повного синтаксису.

9. Діалоги вставки

MarkForge надає керовані діалоги для вставки поширених конструкцій Markdown. Усі діалоги вставляють текст у поточну позицію курсору в редакторі.

Вставити посилання Ctrl+K

Відкрити через Вставка → Посилання… або натиснути Ctrl+K.

Діалог містить три поля:

  • Текст відображення — видима мітка посилання (наприклад, Натисніть тут).
  • URL — адреса призначення (наприклад, https://example.com).
  • Заголовок (необов'язково) — підказка, що відображається під час наведення курсору на посилання.

Натисніть Вставити, щоб помістити [Текст відображення](URL "Заголовок") в редактор.

Вставити зображення Ctrl+Shift+K

Відкрити через Вставка → Зображення… або натиснути Ctrl+Shift+K.

Діалог містить два поля:

  • Альтернативний текст — короткий опис для програм зчитування з екрана.
  • URL / Шлях — віддалений URL або локальний шлях до файлу. Використовуйте кнопку перегляду для вибору локального файлу зображення.

Натисніть Вставити, щоб помістити ![Альтернативний текст](URL) в редактор.

Вставити таблицю Ctrl+Shift+T

Відкрити через Вставка → Таблиця… або натиснути Ctrl+Shift+T.

Налаштуйте структуру таблиці:

  • Рядки даних — кількість рядків без урахування рядка заголовка.
  • Стовпці — кількість стовпців (до максимально підтримуваної кількості).
  • Назви стовпців — введіть заголовок для кожного стовпця.
  • Вирівнювання — виберіть «Ліворуч», «По центру» або «Праворуч» для кожного стовпця окремо.

Живий попередній перегляд Markdown оновлюється внизу діалогу. Натисніть Вставити, щоб помістити таблицю в редактор.

Вставити PlantUML Ctrl+Shift+U

Відкрити через Вставка → PlantUML… або натиснути Ctrl+Shift+U.

Виберіть один із 12 шаблонів діаграм у спадному списку:

  • Діаграма послідовності
  • Діаграма класів
  • Діаграма діяльності
  • Діаграма станів
  • Діаграма компонентів
  • Діаграма варіантів використання
  • Діаграма Ганта
  • Ментальна карта
  • WBS (Структура декомпозиції робіт)
  • Часова діаграма
  • Діаграма розгортання
  • JSON

Вибраний шаблон заповнює редактор коду PlantUML. Відредагуйте код за потреби, а потім натисніть Вставити. Діаграма огортається в огороджений блок коду та відтворюється через онлайн-сервіс plantuml.com у попередньому перегляді. Потрібне підключення до інтернету.

Примітка

Попередній перегляд діаграми автоматично з'являється в головному вікні після вставки. Для відтворення використовується публічний сервер plantuml.com; локальна установка PlantUML не потрібна.

Вставити Mermaid Ctrl+Shift+M

Відкрити через Вставка → Mermaid… або натиснути Ctrl+Shift+M.

Виберіть один із 9 шаблонів діаграм:

  • Блок-схема
  • Діаграма послідовності
  • Діаграма класів
  • Діаграма станів
  • Діаграма Ганта
  • Кругова діаграма
  • Ментальна карта
  • ER-діаграма
  • Граф Git

Відредагуйте код Mermaid у текстовому полі та натисніть Вставити. Діаграма огортається в огороджений блок коду mermaid та відтворюється в попередньому перегляді через kroki.io. Потрібне підключення до інтернету.

10. Функції PDF

Експортувати як PDF Ctrl+Shift+E

Виберіть Файл → Експортувати як PDF… або натисніть Ctrl+Shift+E. MarkForge відтворює поточний документ у файл PDF за допомогою того самого конвеєра, що й живий попередній перегляд, але з білим фоном і CSS, оптимізованим для друку. З'явиться діалог збереження файлу, де ви зможете вибрати місце виведення та ім'я файлу.

Порада

Переключіть попередній перегляд на світлу тему перед експортом у PDF для найбільш чистого виведення, готового до друку.

Експортувати як HTML

Виберіть Файл → Експортувати як HTML…. MarkForge рендерить поточний документ у автономний HTML-файл із вбудованим CSS та локальними зображеннями, перетвореними на URI даних Base64, тому файл відкривається в будь-якому браузері без підключення до інтернету.

Експорт через Pandoc (DOCX, EPUB, LaTeX)

Виберіть Файл → Експортувати як DOCX (Pandoc)…, Експортувати як EPUB (Pandoc)… або Експортувати як LaTeX (Pandoc)…. Ці параметри вимагають встановлення pandoc та його наявності в PATH. Якщо pandoc не знайдено, з'являється діалогове вікно з попередженням.

Імпортувати PDF Ctrl+Shift+I

Виберіть Файл → Імпортувати PDF… або натисніть Ctrl+Shift+I. MarkForge перетворює будь-який файл PDF у Markdown за допомогою бібліотеки pymupdf4llm. Перетворення розумно зберігає структуру документа:

  • Заголовки та абзаци
  • Таблиці
  • Списки
  • Блоки коду там, де вони виявлені

Перетворення виконується у фоновому потоці, тому інтерфейс залишається чуйним. Після завершення MarkForge запропонує вибрати місце збереження отриманого файлу .md, а потім завантажить його в редакторі.

Примітка

Імпорт PDF потребує пакета pymupdf4llm. Якщо він не встановлений, MarkForge запропонує встановити його: pip install pymupdf4llm

11. Інтеграція з Git

MarkForge може відкривати файли Markdown безпосередньо з розміщених репозиторіїв Git, дозволяти їх редагувати та відправляти зміни назад — без відкриття термінала. Також підтримуються розширені операції, такі як amend і squash.

Підтримувані платформи

ПлатформаПриклад шаблону URL
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.mycompany.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.mycompany.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

Платформа визначається автоматично зі структури URL, а не лише за іменем хоста — тому самостійно розгорнуті екземпляри також працюють автоматично.

Відкриття файлу з Git

  1. Виберіть Файл → Відкрити з Git… або натисніть Ctrl+Shift+G.
  2. Вставте URL файлу, який потрібно відкрити (скопіюйте його з браузера файлів GitHub або Bitbucket).
  3. MarkForge перевіряє URL і показує рядок попереднього перегляду: платформа · власник/репозиторій · шлях/до/файлу.md.
  4. За потреби змініть гілку, тег або хеш commit у полі Ref.
  5. Натисніть OK — репозиторій клонується у фоновому режимі, і файл відкривається в редакторі.

Поки відкрито файл під управлінням Git, у заголовку дерева файлів відображається [GIT] назва-репозиторію зеленим кольором.

Порада

Публічні репозиторії працюють без будь-яких налаштованих облікових даних. Для приватних репозиторіїв налаштуйте свої облікові дані в Вигляд → Налаштування → Автентифікація Git перед відкриттям.

Збереження (Commit & Push)

Коли відкрито файл під управлінням Git, натискання Ctrl+S відкриває діалог Git Commit & Push замість звичайного збереження файлу. Діалог містить:

  • Повідомлення commit — обов'язкове. Введіть змістовний опис ваших змін.
  • Змінити попередній commit (amend) — доступно, починаючи з другого збереження, при використанні автентифікації SSH або git-binary. Замінює попередній commit замість створення нового; попередньо заповнює попереднє повідомлення commit; автоматично використовує force-push (--force-with-lease). Змінювати можна лише ті commits, що були зроблені MarkForge під час поточного сеансу.
  • Push до:
    • Поточна гілка — push безпосередньо до гілки, з якої було зроблено клон.
    • Нова гілка — створює нову гілку з назвою, яку ви вказуєте.
  • Створити Pull Request — доступно, коли вибрано «Нова гілка». Автоматично відкриває pull request після push. Відображає поля для заголовка PR і цільової гілки.

Squash Commits Ctrl+Shift+Q

Доступно лише з автентифікацією SSH та HTTPS (git binary).

Виберіть Файл → Git Squash… або натисніть Ctrl+Shift+Q. Діалог Squash Commits відображає таблицю всіх commits поточної гілки, яких ще немає у вибраній базовій гілці.

  • Поле базової гілки — еталонна гілка (наприклад, main, master, develop). Натисніть Перезавантажити після її зміни.
  • Вибрати всі нові commits — відмічає всі перелічені commits одразу.
  • Повідомлення squash commit — єдине повідомлення, яке замінить усі вибрані commits.
  • Squash & Push — переписує історію гілки та виконує force-push до віддаленого репозиторію.

Необхідно вибрати щонайменше 2 commits, які мають утворювати суміжний діапазон, починаючи з найновішого commit (HEAD).

Режими автентифікації

Налаштуйте метод автентифікації в Вигляд → Налаштування → Автентифікація Git.

HTTPS (вбудований)

Працює на всіх платформах без будь-якого додаткового встановленого програмного забезпечення. Використовує REST API платформи безпосередньо. Локальний бінарний файл git не потрібний.

Обмеження

Функції amend і squash недоступні в цьому режимі, оскільки повна локальна історія git не підтримується.

ПлатформаТип токена
GitHubПерсональний токен доступу (ghp_…) — потребує область repo
GitHub EnterpriseПерсональний токен доступу — аналогічно GitHub
Bitbucket CloudПароль застосунку (Обліковий запис → Паролі застосунків)
Bitbucket ServerПерсональний токен доступу або токен HTTP-доступу

HTTPS (git binary)

Використовує системний виконуваний файл git для всіх операцій — повне локальне клонування, commit і push. Потрібен Git for Windows (або будь-який бінарний файл git у вашому PATH). Підтримує amend і squash. Використовуйте ті самі поля імені користувача та токена, що й режим HTTPS (вбудований). Також необхідно вказати ваше Ім'я та Електронну пошту для ідентифікації commit.

SSH (файл ключа)

Виберіть SSH (файл ключа) та вкажіть шлях до вашого приватного ключа SSH і необов'язкову парольну фразу. Цей режим використовує paramiko для чистого Python SSH-з'єднання (системний клієнт ssh не потрібний). Якщо paramiko не встановлено, MarkForge використовує системний клієнт ssh.

pip install paramiko

Автентифікація SSH підтримує amend і squash.

Зберігання облікових даних

Токени, парольні фрази SSH і паролі проксі зберігаються у сховищі ключів ОС (GNOME Keyring / KWallet на Linux, Keychain на macOS, Windows Credential Manager на Windows) — не у файлах конфігурації у відкритому тексті.

HTTP Проксі

Якщо ваша мережа потребує HTTP проксі для доступу до серверів Git, налаштуйте його в Вигляд → Налаштування → HTTP Проксі:

  • URL проксі — наприклад, http://proxy.company.com:8080
  • Ім'я користувача проксі — залиште порожнім, якщо автентифікація не потрібна.
  • Пароль проксі — залиште порожнім, якщо автентифікація не потрібна.

Залиште всі поля проксі порожніми, щоб використовувати системні налаштування проксі за замовчуванням.

Закриття

Коли ви закриваєте MarkForge, поки відкрито файл під управлінням Git, MarkForge запитає, чи слід видалити тимчасовий каталог клону. Видалення каталогу звільняє місце на диску; збереження дозволяє вам переглянути клоновані файли після цього.

12. Налаштування
