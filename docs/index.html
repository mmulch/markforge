MarkForge User Manual v1.0

1. Introduction

MarkForge is a modern, feature-rich Markdown editor built with Python and PyQt6. It combines a distraction-free writing experience with a real-time side-by-side preview, full Git workflow integration, diagram support, math rendering, and PDF import/export — all in one clean, dual-pane desktop application.

Whether you are writing technical documentation, project READMEs, or publishing content directly to a GitHub repository, MarkForge handles the full editing-and-publishing lifecycle without requiring a terminal.

Live PreviewGitHub-styled HTML rendered side-by-side as you type
Syntax HighlightingMarkdown-aware colour scheme, dark & light
Git IntegrationOpen, edit, commit, and push to GitHub or Bitbucket
DiagramsPlantUML (12 templates) and Mermaid (9 templates)
Math FormulasLaTeX notation rendered with MathJax
PDF ImportConvert any PDF to Markdown automatically
PDF ExportPrint-optimised PDF from any document
MultilingualEnglish and German (Deutsch) user interface

2. Installation

Linux (Debian / Ubuntu)

Download the .deb package from the Releases page on GitHub, then install it with:

sudo apt install ./markforge_*.deb

The package automatically installs all required system-level dependencies (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments). No virtual environment is required.

Windows

Download MarkForge-Setup.exe from the Releases page and run the installer. The setup wizard guides you through the installation and creates start menu shortcuts automatically.

Note

The Windows installer bundles the Python runtime and all dependencies. No separate Python installation is required.

From Source

Requirements: Python 3.8 or later, CMake 3.16 or later.

Using CMake (recommended):

# 1. Clone the repository
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Configure the build (creates a local virtual environment)
cmake -B build

# 3. Install Python dependencies into the local venv
cmake --build build --target install_deps

# 4. Run the application
cmake --build build --target run

Direct Python launch (without CMake):

pip install -r requirements.txt
python src/main.py
Tip

For SSH-based Git operations, also install paramiko: pip install paramiko

3. Interface Overview

Layout

The MarkForge window is divided into three resizable panes separated by draggable splitters:

PanePositionDescription
File TreeLeftBrowse .md and image files in the current folder. Toggle with Ctrl+B.
EditorCentrePlain-text editor with Markdown syntax highlighting and line numbers.
PreviewRightLive HTML rendering of the current document. Toggle with Ctrl+Shift+P.

Drag any splitter handle to resize panes. Pane sizes are remembered between sessions.

Toolbar

The toolbar below the menu bar provides one-click access to the most common actions: New, Open, Save, Insert Link, Insert Image, Insert Table, Insert PlantUML, Insert Mermaid, Toggle Preview, and Settings. All toolbar actions duplicate menu items and keyboard shortcuts.

Status Bar

The status bar at the bottom of the window always shows two pieces of information:

  • Word count — the total number of words in the document (updates live as you type).
  • Cursor position — the current line and column number, e.g. Line 12, Col 5.

Menu Bar

The menu bar contains five top-level menus:

MenuContents
FileNew, Open, Open Folder, Import PDF, Open from Git, Git Squash, Save, Save As, Export as PDF, Quit
EditUndo, Redo, Cut, Copy, Paste
InsertLink, Image, PlantUML, Mermaid, Table
ViewShow file tree, Show preview, Word wrap, Settings
HelpMarkdown reference, About

Window Title

The window title shows the name of the currently open file. When you make unsaved changes, an asterisk (*) is appended to the filename, e.g. README.md *. When no file is open the title shows Untitled.

4. The Editor

The centre pane hosts a full-featured plain-text editor tuned for Markdown authoring.

Syntax Highlighting

MarkForge applies Markdown-aware syntax highlighting to the document as you type. Headings, emphasis markers, links, code spans, fenced code blocks, and other Markdown constructs are coloured distinctly. The highlighting adapts automatically when you switch between the Light and Dark editor themes.

Line Numbers

A gutter on the left side of the editor displays line numbers. The current line is highlighted with a subtle background colour to help you track your position in long documents.

Word Wrap

Word wrap is enabled by default. Long lines wrap to fit the visible editor width. Toggle word wrap at any time via View → Word wrap. The preference is saved between sessions.

Editor Themes

Two built-in editor themes are available:

  • Dark (default) — dark background with high-contrast syntax colours.
  • Light — white background suitable for bright environments.

Switch themes in View → Settings → Editor theme. The change takes effect immediately without a restart.

Placeholder Text

When no file is open, the editor is disabled and shows the message: "Open or create a Markdown file to start editing." The editor activates as soon as you create a new file or open an existing one.

Editing Operations

All standard text editing operations are available:

  • Ctrl+Z — Undo
  • Ctrl+Y — Redo
  • Ctrl+X / Ctrl+C / Ctrl+V — Cut / Copy / Paste

5. Live Preview

The right pane displays a fully rendered HTML version of your Markdown document, updated automatically as you type.

Rendering

The preview uses a GitHub-inspired stylesheet for clean, readable output. It supports all Markdown extensions including tables, task lists, footnotes, code highlighting, math, and diagrams. The rendering pipeline processes your text through the Python markdown library with the extra, codehilite, toc, and sane_lists extensions.

Scroll Position Preservation

When you edit the document the preview updates after a short 300 ms debounce delay. Before refreshing the HTML, MarkForge reads the current scroll position from the web view and restores it after rendering, so the preview does not jump to the top while you are working in the middle of a document.

Preview Themes

Two preview themes are available:

  • Dark (default) — dark background matching the overall application theme.
  • Light — white background for a print-like reading experience.

Switch via View → Settings → Preview theme. The change takes effect immediately.

Toggling the Preview

Press Ctrl+Shift+P or use View → Show preview to hide or show the preview panel. Hiding the preview gives the editor more screen space.

Note

Math formulas (MathJax) and Mermaid diagrams require an internet connection on first use, as the libraries are loaded from a CDN. PlantUML diagrams are always fetched from plantuml.com and require internet access.

6. File Operations

New File

Press Ctrl+N or choose File → New. If the current document has unsaved changes, MarkForge will prompt you to save before proceeding. A new empty document is opened and the editor is activated.

Open File

Press Ctrl+O or choose File → Open… to open a file browser dialog. Markdown files (.md, .markdown), text files (.txt), and all other files are selectable. The file tree will update to show the folder containing the opened file.

Open Folder

Choose File → Open Folder… (Ctrl+Shift+O) to select a directory. The file tree then shows all .md and image files in that directory. Click any file in the tree to open it in the editor.

Save

Press Ctrl+S or choose File → Save. If the file was opened from a Git repository, a Git commit & push dialog will appear instead (see Saving via Git). For local files, the file is written to disk immediately and the asterisk in the title bar disappears.

Save As

Press Ctrl+Shift+S or choose File → Save As… to save the current document to a new location or with a new name.

File Tree

The left panel displays a flat list of .md files and images in the current root directory. Sub-directories containing no directly relevant files are hidden to reduce clutter. Click a file to open it. The tree header shows [GIT] reponame in green when a Git-managed file is open.

Tip

Toggle the file tree with Ctrl+B to maximise editor space. Your preference is saved between sessions.

8. Math Formulas

MarkForge renders LaTeX math notation using MathJax. An internet connection is required on first use (MathJax is loaded from a CDN).

Inline Math

Wrap an expression in single dollar signs to render it inline within a paragraph:

The famous equation $E = mc^2$ was published in 1905.

Result: The famous equation E = mc² was published in 1905.

Display (Block) Math

Wrap the expression in double dollar signs to render it centred on its own line:

$$\int_0^\infty e^{-x}\, dx = 1$$

Single-Line Block Math

You can also use a lone $ on its own line to open and close a display-math block:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
Tip

MathJax supports the full LaTeX math mode, including fractions, matrices, Greek letters, operators, and custom macros. Refer to the MathJax documentation for the complete syntax.

9. Insert Dialogs

MarkForge provides guided dialogs for inserting common Markdown constructs. All dialogs insert text at the current cursor position in the editor.

Insert Link Ctrl+K

Open via Insert → Link… or press Ctrl+K.

The dialog has three fields:

  • Display text — the visible link label (e.g. Click here).
  • URL — the destination address (e.g. https://example.com).
  • Title (optional) — a tooltip shown when the user hovers over the link.

Click Insert to place [Display text](URL "Title") in the editor.

Insert Image Ctrl+Shift+K

Open via Insert → Image… or press Ctrl+Shift+K.

The dialog has two fields:

  • Alt Text — a short description for screen readers.
  • URL / Path — a remote URL or a local file path. Use the browse button to pick a local image file.

Click Insert to place ![Alt text](URL) in the editor.

Insert Table Ctrl+Shift+T

Open via Insert → Table… or press Ctrl+Shift+T.

Configure your table structure:

  • Data rows — number of rows excluding the header row.
  • Columns — number of columns (up to the maximum supported).
  • Column names — enter a title for each column.
  • Alignment — choose Left, Center, or Right for each column independently.

A live Markdown preview updates at the bottom of the dialog. Click Insert to place the table in the editor.

Insert PlantUML Ctrl+Shift+U

Open via Insert → PlantUML… or press Ctrl+Shift+U.

Choose from 12 diagram templates in the dropdown:

  • Sequence diagram
  • Class diagram
  • Activity diagram
  • State diagram
  • Component diagram
  • Use case diagram
  • Gantt diagram
  • Mind map
  • WBS (Work breakdown)
  • Timing diagram
  • Deployment diagram
  • JSON

The selected template populates the PlantUML code editor. Edit the code as needed, then click Insert. The diagram is wrapped in a fenced code block and rendered via the plantuml.com online service in the preview. An internet connection is required.

Note

The diagram preview appears automatically in the main window after inserting. The rendering uses the public plantuml.com server; no local PlantUML installation is needed.

Insert Mermaid Ctrl+Shift+M

Open via Insert → Mermaid… or press Ctrl+Shift+M.

Choose from 9 diagram templates:

  • Flowchart
  • Sequence diagram
  • Class diagram
  • State diagram
  • Gantt
  • Pie chart
  • Mindmap
  • ER diagram
  • Git graph

Edit the Mermaid code in the text area and click Insert. The diagram is wrapped in a fenced mermaid code block and rendered in the preview via kroki.io. An internet connection is required.

10. PDF Features

Export as PDF Ctrl+Shift+E

Choose File → Export as PDF… or press Ctrl+Shift+E. MarkForge renders the current document to a PDF file using the same pipeline as the live preview, but with a white background and print-optimised CSS. A file save dialog appears so you can choose the output location and filename.

Tip

Switch the preview to the Light theme before exporting to PDF for the cleanest print-ready output.

Import PDF Ctrl+Shift+I

Choose File → Import PDF… or press Ctrl+Shift+I. MarkForge converts any PDF file to Markdown using the pymupdf4llm library. The conversion intelligently preserves document structure:

  • Headings and paragraphs
  • Tables
  • Lists
  • Code blocks where detected

The conversion runs in a background thread so the UI stays responsive. When complete, MarkForge prompts you for a save location for the resulting .md file and then loads it in the editor.

Note

PDF import requires the pymupdf4llm package. If it is not installed, MarkForge will prompt you to install it: pip install pymupdf4llm

11. Git Integration

MarkForge can open Markdown files directly from hosted Git repositories, allow you to edit them, and push changes back — without ever opening a terminal. Advanced operations like amend and squash are also supported.

Supported Platforms

PlatformExample URL pattern
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.mycompany.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.mycompany.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

The platform is auto-detected from the URL structure, not just the hostname — so self-hosted instances work automatically.

Opening a File from Git

  1. Choose File → Open from Git… or press Ctrl+Shift+G.
  2. Paste the URL of the file you want to open (copy it from the GitHub or Bitbucket file browser).
  3. MarkForge validates the URL and shows a preview line: platform · owner/repo · path/to/file.md.
  4. Optionally override the branch, tag, or commit hash in the Ref field.
  5. Click OK — the repository is cloned in the background and the file opens in the editor.

While a Git-managed file is open, the file tree header shows [GIT] reponame in green.

Tip

Public repositories work without any credentials configured. For private repositories, configure your credentials in View → Settings → Git Authentication before opening.

Saving (Commit & Push)

When a Git-managed file is open, pressing Ctrl+S opens the Git Commit & Push dialog instead of the normal file save. The dialog contains:

  • Commit message — required. Enter a meaningful description of your changes.
  • Amend previous commit — available from the second save onwards when using SSH or git-binary authentication. Replaces the previous commit instead of creating a new one; pre-fills the previous commit message; automatically uses force-push (--force-with-lease). Only commits made by MarkForge in the current session can be amended.
  • Push to:
    • Current branch — pushes directly to the branch you cloned from.
    • New branch — creates a new branch with a name you specify.
  • Create Pull Request — available when "New branch" is selected. Automatically opens a pull request after pushing. Shows fields for PR title and target branch.

Squash Commits Ctrl+Shift+Q

Available with SSH and HTTPS (git binary) authentication only.

Choose File → Git Squash… or press Ctrl+Shift+Q. The Squash Commits dialog shows a table of all commits on the current branch that are not yet in the chosen base branch.

  • Base branch field — the reference branch (e.g. main, master, develop). Click Reload after changing it.
  • Select all new commits — checks all listed commits at once.
  • Squash commit message — the single message that will replace all selected commits.
  • Squash & Push — rewrites the branch history and force-pushes to the remote.

You must select at least 2 commits, and they must form a contiguous range starting from the most recent commit (HEAD).

Authentication Modes

Configure your authentication method in View → Settings → Git Authentication.

HTTPS (embedded)

Works on all platforms without any additional software installed. Uses the platform REST API directly. No local git binary is required.

Limitation

The amend and squash features are not available in this mode because no full local git history is maintained.

PlatformToken type
GitHubPersonal access token (ghp_…) — requires repo scope
GitHub EnterprisePersonal access token — same as GitHub
Bitbucket CloudApp password (Account → App passwords)
Bitbucket ServerPersonal access token or HTTP access token

HTTPS (git binary)

Uses the system git executable for all operations — full local clone, commit, and push. Requires Git for Windows (or any git binary on your PATH). Supports amend and squash. Use the same username and token fields as the HTTPS (embedded) mode. You must also provide your Name and Email for commit identity.

SSH (key file)

Select SSH (key file) and provide the path to your private SSH key and an optional passphrase. This mode uses paramiko for a pure-Python SSH connection (no system ssh client required). If paramiko is not installed, MarkForge falls back to the system ssh client.

pip install paramiko

SSH authentication supports amend and squash.

HTTP Proxy

If your network requires an HTTP proxy to reach Git servers, configure it in View → Settings → HTTP Proxy:

  • Proxy URL — e.g. http://proxy.company.com:8080
  • Proxy username — leave empty if no authentication is required.
  • Proxy password — leave empty if no authentication is required.

Leave all proxy fields empty to use the system's default proxy settings.

Closing

When you close MarkForge while a Git-managed file is open, MarkForge will ask whether to delete the temporary clone directory. Deleting the directory frees disk space; keeping it allows you to inspect the cloned files afterwards.
