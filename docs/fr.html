Manuel utilisateur MarkForge v1.8.0

1. Introduction

MarkForge est un éditeur Markdown moderne et riche en fonctionnalités, construit avec Python et PyQt6. Il combine une expérience d'écriture sans distraction avec un aperçu en temps réel côte à côte, une intégration complète du workflow Git, la prise en charge des diagrammes, le rendu mathématique et l'import/export PDF — le tout dans une application de bureau à double panneau épurée.

Que vous rédigiez de la documentation technique, des fichiers README de projets ou que vous publiiez du contenu directement dans un dépôt GitHub, MarkForge gère l'intégralité du cycle de vie édition-publication sans nécessiter un terminal.

2. Installation

Linux (Debian / Ubuntu)

Téléchargez le paquet .deb depuis la page Releases sur GitHub, puis installez-le avec :

sudo apt install ./markforge_*.deb

Le paquet installe automatiquement toutes les dépendances système requises (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments). Aucun environnement virtuel n'est nécessaire.

Windows

Téléchargez MarkForge-Setup.exe depuis la page Releases et lancez l'installateur. L'assistant d'installation vous guide tout au long du processus et crée automatiquement des raccourcis dans le menu Démarrer.

Remarque

L'installateur Windows inclut l'environnement d'exécution Python et toutes les dépendances. Aucune installation Python séparée n'est requise.

Depuis les sources

Prérequis : Python 3.8 ou ultérieur, CMake 3.16 ou ultérieur.

Avec CMake (recommandé) :

# 1. Clone the repository
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Configure the build (creates a local virtual environment)
cmake -B build

# 3. Install Python dependencies into the local venv
cmake --build build --target install_deps

# 4. Run the application
cmake --build build --target run

Lancement direct avec Python (sans CMake) :

pip install -r requirements.txt
python src/main.py
Astuce

Pour les opérations Git via SSH, installez également paramiko : pip install paramiko

3. Présentation de l'interface

Disposition

La fenêtre de MarkForge est divisée en trois panneaux redimensionnables séparés par des séparateurs déplaçables :

PanneauPositionDescription
Plan + ArborescenceGaucheLe panneau gauche comporte deux sections empilées : le Plan du document (arbre des titres) en haut, et l'Arborescence de fichiers (explorateur de fichiers) en bas. Activez/désactivez le plan via Affichage → Afficher le plan, l'arborescence via Ctrl+B.
ÉditeurCentreÉditeur de texte brut avec coloration syntaxique Markdown et numéros de ligne.
AperçuDroiteRendu HTML en direct du document actuel. Activez/désactivez avec Ctrl+Shift+P.

Faites glisser n'importe quel séparateur pour redimensionner les panneaux. Les tailles des panneaux sont mémorisées entre les sessions.

Plan du document

Le panneau Plan du document se trouve en haut de la barre latérale gauche et liste tous les titres (H1–H6) du document actuel sous forme d'arbre navigable.

  • Cliquez sur n'importe quel titre pour déplacer le curseur de l'éditeur à cette ligne.
  • Le plan se met à jour automatiquement pendant la saisie.
  • Activez/désactivez la visibilité via Affichage → Afficher le plan. La préférence est sauvegardée entre les sessions.

Barre d'outils

La barre d'outils sous la barre de menus fournit un accès en un clic aux actions les plus courantes : Nouveau, Ouvrir, Enregistrer, Insérer un lien, Insérer une image, Insérer un tableau, Insérer PlantUML, Insérer Mermaid, Basculer l'aperçu et Paramètres. Toutes les actions de la barre d'outils dupliquent les éléments de menu et les raccourcis clavier.

Barre d'état

La barre d'état en bas de la fenêtre affiche toujours :

  • Nombre de mots & temps de lecture — nombre de mots et temps de lecture estimé (ex. 342 mots · 2 min de lecture), mis à jour en direct.
  • Progression de l'objectif de mots — si un objectif est défini via Afficher → Définir l'objectif de mots, une barre de progression compacte apparaît.
  • Position du curseur — ligne et colonne actuelles, ex. Ligne 12, Col 5.

Barre de menus

La barre de menus contient cinq menus de premier niveau :

Titre de la fenêtre

Le titre de la fenêtre affiche le nom du fichier actuellement ouvert. Lorsque vous effectuez des modifications non enregistrées, un astérisque (*) est ajouté au nom du fichier, par exemple README.md *. Lorsqu'aucun fichier n'est ouvert, le titre affiche Untitled.

4. L'éditeur

Le panneau central héberge un éditeur de texte brut complet, optimisé pour la rédaction en Markdown.

Coloration syntaxique

MarkForge applique une coloration syntaxique adaptée au Markdown au document pendant la saisie. Les titres, les marqueurs d'emphase, les liens, les fragments de code, les blocs de code délimités et autres constructions Markdown sont colorés de façon distincte. La coloration s'adapte automatiquement lors du changement entre les thèmes d'éditeur Clair et Sombre.

Numéros de ligne

Une marge sur le côté gauche de l'éditeur affiche les numéros de ligne. La ligne courante est mise en surbrillance avec une couleur de fond subtile pour vous aider à suivre votre position dans les longs documents.

Marqueurs de différences en ligne

Lorsqu'un fichier géré par Git est ouvert, la gouttière de l'éditeur affiche des barres colorées à côté des lignes qui diffèrent de la dernière version validée :

  • Vert — lignes ajoutées
  • Orange — lignes modifiées
  • Rouge — blocs supprimés (affichés comme un fin marqueur au-dessus du point de suppression)

Les marqueurs se mettent à jour en temps réel pendant la saisie et se réinitialisent après chaque commit & push réussi.

Retour à la ligne

Le retour à la ligne est activé par défaut. Les longues lignes sont ajustées à la largeur visible de l'éditeur. Activez/désactivez le retour à la ligne à tout moment via Affichage → Retour à la ligne. La préférence est sauvegardée entre les sessions.

Thèmes de l'éditeur

Deux thèmes d'éditeur intégrés sont disponibles :

  • Sombre (par défaut) — fond sombre avec des couleurs de syntaxe à fort contraste.
  • Clair — fond blanc adapté aux environnements lumineux.

Changez de thème dans Affichage → Paramètres → Thème de l'éditeur. Le changement prend effet immédiatement sans redémarrage.

Texte d'espace réservé

Lorsqu'aucun fichier n'est ouvert, l'éditeur est désactivé et affiche le message : "Open or create a Markdown file to start editing." L'éditeur s'active dès que vous créez un nouveau fichier ou en ouvrez un existant.

Opérations d'édition

Toutes les opérations d'édition de texte standard sont disponibles :

  • Ctrl+Z — Annuler
  • Ctrl+Y — Rétablir
  • Ctrl+X / Ctrl+C / Ctrl+V — Couper / Copier / Coller

Rechercher & Remplacer

Appuyez sur Ctrl+F pour ouvrir la barre de recherche, ou Ctrl+H pour l'ouvrir avec la ligne de remplacement visible. La barre apparaît au-dessus de l'éditeur.

  • Saisissez dans le champ de recherche — toutes les correspondances sont instantanément mises en surbrillance dans l'éditeur.
  • Naviguez entre les correspondances avec les boutons ↑ / ↓, ou appuyez sur Entrée (suivant) / Shift+Entrée (précédent) dans le champ de recherche.
  • Le compteur de correspondances affiche la position actuelle (par exemple 3 / 12).
  • Aa — activer/désactiver la recherche sensible à la casse.
  • [W] — activer/désactiver la correspondance de mots entiers.
  • Utilisez Remplacer pour remplacer la correspondance actuelle, ou Tout remplacer pour remplacer toutes les occurrences à la fois.
  • Appuyez sur Échap ou cliquez sur ✕ pour fermer la barre et effacer toutes les surbrillances.

Si du texte est sélectionné dans l'éditeur lorsque vous ouvrez la barre, le champ de recherche est pré-rempli avec la sélection (sélections d'une seule ligne uniquement).

Vérification orthographique

MarkForge peut souligner les mots mal orthographiés avec une ligne ondulée rouge pendant la saisie. La vérification orthographique est désactivée par défaut.

  • Activez via Affichage → Vérification orthographique (bascule).
  • Sélectionnez la langue du dictionnaire via Affichage → Langue de vérification orthographique. Langues disponibles : English, Deutsch, Español, Français, Italiano, Nederlands, Português, العربية, فارسی (via pyspellchecker) et Tiếng Việt*, Svenska*, Українська*, ಕನ್ನಡ*, हिंदी* (via enchant/Hunspell).
  • Les mots dans les fragments de code en ligne (`…`) et les URL ne sont jamais soulignés.
  • Les abréviations EN MAJUSCULES (HTML, API, URL…) sont ignorées.
  • La préférence de vérification orthographique et la langue sélectionnée sont sauvegardées entre les sessions.
  • Nécessite la bibliothèque pyspellchecker. Si elle n'est pas installée, la bascule est silencieusement ignorée.
  • Les langues marquées avec * nécessitent la bibliothèque pyenchant et les dictionnaires Hunspell système (par exemple hunspell-vi sur Debian/Ubuntu).

5. Aperçu en direct

Le panneau droit affiche une version HTML entièrement rendue de votre document Markdown, mise à jour automatiquement pendant la saisie.

Rendu

L'aperçu utilise une feuille de style inspirée de GitHub pour un rendu propre et lisible. Il prend en charge toutes les extensions Markdown, notamment les tableaux, les listes de tâches, les notes de bas de page, la coloration syntaxique du code, les mathématiques et les diagrammes. Le pipeline de rendu traite votre texte via la bibliothèque Python markdown avec les extensions extra, codehilite, toc et sane_lists.

Préservation de la position de défilement

Lorsque vous modifiez le document, l'aperçu se met à jour après un court délai de debounce de 300 ms. Avant d'actualiser le HTML, MarkForge lit la position de défilement actuelle depuis la vue web et la restaure après le rendu, de sorte que l'aperçu ne saute pas au début pendant que vous travaillez au milieu d'un document.

Thèmes d'aperçu

Deux thèmes d'aperçu sont disponibles :

  • Sombre (par défaut) — fond sombre correspondant au thème général de l'application.
  • Clair — fond blanc pour une expérience de lecture semblable à l'impression.

Changez via Affichage → Paramètres → Thème d'aperçu. Le changement prend effet immédiatement.

Basculer l'aperçu

Appuyez sur Ctrl+Shift+P ou utilisez Affichage → Afficher l'aperçu pour masquer ou afficher le panneau d'aperçu. Masquer l'aperçu donne à l'éditeur plus d'espace à l'écran.

Remarque

Les formules mathématiques (MathJax) et les diagrammes Mermaid nécessitent une connexion Internet lors de la première utilisation, car les bibliothèques sont chargées depuis un CDN. Les diagrammes PlantUML sont toujours récupérés depuis plantuml.com et nécessitent un accès Internet.

6. Opérations sur les fichiers

Nouveau fichier

Appuyez sur Ctrl+N ou choisissez Fichier → Nouveau. Si le document actuel a des modifications non enregistrées, MarkForge vous demandera d'enregistrer avant de continuer. Un nouveau document vide est ouvert et l'éditeur est activé.

Ouvrir un fichier

Appuyez sur Ctrl+O ou choisissez Fichier → Ouvrir… pour ouvrir une boîte de dialogue de navigation dans les fichiers. Les fichiers Markdown (.md, .markdown), les fichiers texte (.txt) et tous les autres fichiers sont sélectionnables. L'arborescence de fichiers se met à jour pour afficher le dossier contenant le fichier ouvert.

Ouvrir un dossier

Choisissez Fichier → Ouvrir un dossier… (Ctrl+Shift+O) pour sélectionner un répertoire. L'arborescence de fichiers affiche alors tous les fichiers .md et les images dans ce répertoire. Cliquez sur n'importe quel fichier dans l'arborescence pour l'ouvrir dans l'éditeur.

Enregistrer

Appuyez sur Ctrl+S ou choisissez Fichier → Enregistrer. Si le fichier a été ouvert depuis un dépôt Git, une boîte de dialogue Git commit & push apparaît à la place (voir Enregistrer via Git). Pour les fichiers locaux, le fichier est immédiatement écrit sur le disque et l'astérisque dans la barre de titre disparaît.

Enregistrer sous

Appuyez sur Ctrl+Shift+S ou choisissez Fichier → Enregistrer sous… pour enregistrer le document actuel à un nouvel emplacement ou avec un nouveau nom.

Arborescence de fichiers

Le panneau gauche affiche une liste plate des fichiers .md et des images dans le répertoire racine actuel. Les sous-répertoires ne contenant aucun fichier directement pertinent sont masqués pour réduire l'encombrement. Cliquez sur un fichier pour l'ouvrir. L'en-tête de l'arborescence affiche [GIT] reponame en vert lorsqu'un fichier géré par Git est ouvert.

Astuce

Basculez l'arborescence de fichiers avec Ctrl+B pour maximiser l'espace de l'éditeur. Votre préférence est sauvegardée entre les sessions.

Fichiers récents

MarkForge conserve une liste des 10 derniers fichiers que vous avez ouverts. Accédez-y via Fichier → Fichiers récents.

  • Cliquez sur n'importe quelle entrée pour rouvrir immédiatement le fichier.
  • Si un fichier n'existe plus sur le disque, il est automatiquement retiré de la liste.
  • Choisissez Effacer les fichiers récents en haut du sous-menu pour effacer tout l'historique.
  • La liste est sauvegardée entre les sessions.

Glisser & Déposer

Vous pouvez faire glisser des fichiers depuis votre gestionnaire de fichiers vers l'éditeur. MarkForge affiche une boîte de dialogue pour choisir comment traiter chaque type de fichier :

  • Fichiers image (.png, .jpg, .gif, etc.) — choisissez d'insérer avec le chemin d'origine ou de copier dans le dossier assets/ et d'insérer un lien relatif.
  • Fichiers Markdown (.md, .markdown) — Ouvrir le fichier depuis son emplacement d'origine ou Copier dans le dossier (copie dans le même répertoire que le document actuel et ouvre la copie).
  • Fichiers PDF — Importer … (flux d'import standard avec boîte de dialogue de sauvegarde) ou Importer dans le dossier (importe automatiquement dans le répertoire du document actuel).
  • Tout autre fichier — insère un lien Markdown [nom de fichier](chemin/vers/fichier).
Remarque

Pour les opérations qui copient ou importent dans le dossier du document actuel, un fichier doit être ouvert et enregistré pour que MarkForge connaisse le répertoire cible.

Mode concentration F11

Appuyez sur F11 ou choisissez Affichage → Mode concentration pour entrer dans le mode d'écriture sans distractions. MarkForge masque l'arborescence des fichiers, le plan du document, l'aperçu en direct et la barre d'état, laissant l'éditeur occuper toute la fenêtre.

Appuyez à nouveau sur F11 pour quitter : tous les panneaux et la barre d'état sont restaurés exactement comme ils étaient avant.

Astuce

Le mode concentration fonctionne indépendamment des boutons de bascule des panneaux individuels (Ctrl+B, Ctrl+Shift+P).

7. Référence Markdown

MarkForge prend en charge le Markdown CommonMark standard plus les extensions suivantes : extra (tableaux, notes de bas de page, abréviations), codehilite (blocs de code avec coloration syntaxique), toc (table des matières), sane_lists et barré.

Titres

Markdown
# Heading 1 ## Heading 2 ### Heading 3 #### Heading 4 ##### Heading 5 ###### Heading 6
Rendu

Heading 1

Heading 2

Heading 3

Emphase & Mise en forme en ligne

Markdown
**bold text** *italic text* ~~strikethrough~~ `inline code` **_bold italic_**
Rendu
bold text
italic text
strikethrough
inline code
bold italic

Listes

Liste non ordonnée :

Markdown
- First item - Second item - Nested item - Third item
Rendu
  • First item
  • Second item
    • Nested item
  • Third item

Liste ordonnée :

Markdown
1. First item 2. Second item 3. Third item
Rendu
  1. First item
  2. Second item
  3. Third item

Liste de tâches (cases à cocher) :

Markdown
- [x] Done task - [ ] Pending task - [ ] Another task
Rendu
  • ✅ Done task
  • ☐ Pending task
  • ☐ Another task

Liens & Images

Markdown
[Link text](https://example.com) [Tooltip link](https://example.com "My tooltip") ![Alt text](image.png) ![Alt text](https://example.com/img.png)
Rendu
Link text
Tooltip link

[image: Alt text]

Tableaux (GFM)

Markdown
| Left | Center | Right | |:-----|:------:|------:| | A | B | C | | D | E | F |
Rendu
LeftCenterRight
ABC
DEF

L'alignement des colonnes est contrôlé par la position des deux-points dans la ligne de séparation : :--- gauche, :---: centre, ---: droite.

Blocs de code

Markdown
```python def hello(name): print(f"Hello, {name}!") ```
Rendu
def hello(name):
    print(f"Hello, {name}!")

Spécifiez n'importe quel identifiant de langage après le triple accent grave d'ouverture pour la coloration syntaxique : python, javascript, bash, sql, json, etc. (propulsé par Pygments).

Autres éléments

Citations

Markdown
> This is a blockquote. > It can span multiple lines.
Rendu
This is a blockquote. It can span multiple lines.

Notes de bas de page

Markdown
Some text[^1] with a footnote. [^1]: The footnote content.
Rendu
Some text[1] with a footnote.

Abréviations

Markdown
The HTML spec is long. *[HTML]: HyperText Markup Language
Rendu
The HTML spec is long.

Table des matières

Placez [TOC] n'importe où dans le document pour insérer une table des matières auto-générée basée sur les titres du document.

Ligne horizontale

Markdown
--- or ***
Rendu

8. Formules mathématiques

MarkForge rend la notation mathématique LaTeX en utilisant MathJax. Une connexion Internet est requise lors de la première utilisation (MathJax est chargé depuis un CDN).

Mathématiques en ligne

Entourez une expression de signes dollar simples pour la rendre en ligne dans un paragraphe :

The famous equation $E = mc^2$ was published in 1905.

Résultat : La célèbre équation E = mc² a été publiée en 1905.

Mathématiques en bloc

Entourez l'expression de signes dollar doubles pour la rendre centrée sur sa propre ligne :

$$\int_0^\infty e^{-x}\, dx = 1$$

Mathématiques en bloc sur une seule ligne

Vous pouvez également utiliser un $ seul sur sa propre ligne pour ouvrir et fermer un bloc de mathématiques en affichage :

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
Astuce

MathJax prend en charge le mode mathématique LaTeX complet, notamment les fractions, les matrices, les lettres grecques, les opérateurs et les macros personnalisées. Consultez la documentation MathJax pour la syntaxe complète.

9. Boîtes de dialogue d'insertion

MarkForge fournit des boîtes de dialogue guidées pour insérer des constructions Markdown courantes. Toutes les boîtes de dialogue insèrent du texte à la position actuelle du curseur dans l'éditeur.

Insérer un lien Ctrl+K

Ouvrez via Insérer → Lien… ou appuyez sur Ctrl+K.

La boîte de dialogue comporte trois champs :

  • Texte d'affichage — l'étiquette de lien visible (par exemple Cliquez ici).
  • URL — l'adresse de destination (par exemple https://example.com).
  • Titre (facultatif) — une info-bulle affichée lorsque l'utilisateur survole le lien.

Cliquez sur Insérer pour placer [Texte d'affichage](URL "Titre") dans l'éditeur.

Insérer une image Ctrl+Shift+K

Ouvrez via Insérer → Image… ou appuyez sur Ctrl+Shift+K.

La boîte de dialogue comporte deux champs :

  • Texte alternatif — une courte description pour les lecteurs d'écran.
  • URL / Chemin — une URL distante ou un chemin de fichier local. Utilisez le bouton de navigation pour sélectionner un fichier image local.

Cliquez sur Insérer pour placer ![Texte alternatif](URL) dans l'éditeur.

Insérer un tableau Ctrl+Shift+T

Ouvrez via Insérer → Tableau… ou appuyez sur Ctrl+Shift+T.

Configurez la structure de votre tableau :

  • Lignes de données — nombre de lignes hors ligne d'en-tête.
  • Colonnes — nombre de colonnes (jusqu'au maximum pris en charge).
  • Noms des colonnes — saisissez un titre pour chaque colonne.
  • Alignement — choisissez Gauche, Centre ou Droite pour chaque colonne indépendamment.

Un aperçu Markdown en direct se met à jour en bas de la boîte de dialogue. Cliquez sur Insérer pour placer le tableau dans l'éditeur.

Insérer PlantUML Ctrl+Shift+U

Ouvrez via Insérer → PlantUML… ou appuyez sur Ctrl+Shift+U.

Choisissez parmi 12 modèles de diagrammes dans le menu déroulant :

  • Diagramme de séquence
  • Diagramme de classes
  • Diagramme d'activité
  • Diagramme d'état
  • Diagramme de composants
  • Diagramme de cas d'utilisation
  • Diagramme de Gantt
  • Carte mentale
  • WBS (Décomposition du travail)
  • Diagramme de synchronisation
  • Diagramme de déploiement
  • JSON

Le modèle sélectionné remplit l'éditeur de code PlantUML. Modifiez le code selon vos besoins, puis cliquez sur Insérer. Le diagramme est encapsulé dans un bloc de code délimité et rendu via le service en ligne plantuml.com dans l'aperçu. Une connexion Internet est requise.

Remarque

L'aperçu du diagramme apparaît automatiquement dans la fenêtre principale après l'insertion. Le rendu utilise le serveur public plantuml.com ; aucune installation locale de PlantUML n'est nécessaire.

Insérer Mermaid Ctrl+Shift+M

Ouvrez via Insérer → Mermaid… ou appuyez sur Ctrl+Shift+M.

Choisissez parmi 9 modèles de diagrammes :

  • Organigramme
  • Diagramme de séquence
  • Diagramme de classes
  • Diagramme d'état
  • Gantt
  • Graphique circulaire
  • Carte mentale
  • Diagramme ER
  • Graphe Git

Modifiez le code Mermaid dans la zone de texte et cliquez sur Insérer. Le diagramme est encapsulé dans un bloc de code mermaid délimité et rendu dans l'aperçu via kroki.io. Une connexion Internet est requise.

10. Fonctionnalités PDF

Exporter en PDF Ctrl+Shift+E

Choisissez Fichier → Exporter en PDF… ou appuyez sur Ctrl+Shift+E. MarkForge rend le document actuel dans un fichier PDF en utilisant le même pipeline que l'aperçu en direct, mais avec un fond blanc et un CSS optimisé pour l'impression. Une boîte de dialogue d'enregistrement de fichier apparaît pour que vous puissiez choisir l'emplacement de sortie et le nom du fichier.

Astuce

Passez l'aperçu au thème Clair avant d'exporter en PDF pour obtenir la sortie la plus propre prête à imprimer.

Exporter en HTML

Choisissez Fichier → Exporter en HTML…. MarkForge rend le document actuel en un fichier HTML autonome avec tout le CSS intégré et les images locales converties en URI de données Base64, permettant d'ouvrir le fichier dans n'importe quel navigateur sans connexion internet.

Exporter via Pandoc (DOCX, EPUB, LaTeX)

Choisissez Fichier → Exporter en DOCX (Pandoc)…, Exporter en EPUB (Pandoc)… ou Exporter en LaTeX (Pandoc)…. Ces options nécessitent que pandoc soit installé et disponible dans le PATH. Si pandoc n'est pas trouvé, un dialogue d'avertissement s'affiche.

Importer PDF Ctrl+Shift+I

Choisissez Fichier → Importer PDF… ou appuyez sur Ctrl+Shift+I. MarkForge convertit n'importe quel fichier PDF en Markdown en utilisant la bibliothèque pymupdf4llm. La conversion préserve intelligemment la structure du document :

  • Titres et paragraphes
  • Tableaux
  • Listes
  • Blocs de code là où ils sont détectés

La conversion s'exécute dans un thread d'arrière-plan afin que l'interface reste réactive. Une fois terminée, MarkForge vous invite à choisir un emplacement d'enregistrement pour le fichier .md résultant, puis le charge dans l'éditeur.

Remarque

L'import PDF nécessite le paquet pymupdf4llm. S'il n'est pas installé, MarkForge vous invitera à l'installer : pip install pymupdf4llm

11. Intégration Git

MarkForge peut ouvrir des fichiers Markdown directement depuis des dépôts Git hébergés, vous permettre de les modifier et de renvoyer les changements — sans jamais ouvrir un terminal. Les opérations avancées comme amend et squash sont également prises en charge.

Plateformes prises en charge

PlateformeExemple de modèle d'URL
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.mycompany.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.mycompany.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

La plateforme est automatiquement détectée à partir de la structure de l'URL, pas seulement du nom d'hôte — les instances auto-hébergées fonctionnent donc automatiquement.

Ouvrir un fichier depuis Git

  1. Choisissez Fichier → Ouvrir depuis Git… ou appuyez sur Ctrl+Shift+G.
  2. Collez l'URL du fichier que vous souhaitez ouvrir (copiez-la depuis le navigateur de fichiers GitHub ou Bitbucket).
  3. MarkForge valide l'URL et affiche une ligne d'aperçu : plateforme · propriétaire/dépôt · chemin/vers/fichier.md.
  4. Remplacez éventuellement la branche, le tag ou le hash de commit dans le champ Ref.
  5. Cliquez sur OK — le dépôt est cloné en arrière-plan et le fichier s'ouvre dans l'éditeur.

Pendant qu'un fichier géré par Git est ouvert, l'en-tête de l'arborescence affiche [GIT] reponame en vert.

Astuce

Les dépôts publics fonctionnent sans aucune accréditation configurée. Pour les dépôts privés, configurez vos accréditations dans Affichage → Paramètres → Authentification Git avant d'ouvrir.

Enregistrer (Commit & Push)

Lorsqu'un fichier géré par Git est ouvert, appuyer sur Ctrl+S ouvre la boîte de dialogue Git Commit & Push au lieu de l'enregistrement normal. La boîte de dialogue contient :

  • Message de commit — obligatoire. Saisissez une description significative de vos modifications.
  • Modifier le commit précédent — disponible à partir du deuxième enregistrement lors de l'utilisation de l'authentification SSH ou git-binary. Remplace le commit précédent au lieu d'en créer un nouveau ; pré-remplit le message de commit précédent ; utilise automatiquement force-push (--force-with-lease). Seuls les commits effectués par MarkForge dans la session actuelle peuvent être modifiés.
  • Pousser vers :
    • Branche actuelle — pousse directement vers la branche depuis laquelle vous avez cloné.
    • Nouvelle branche — crée une nouvelle branche avec un nom que vous spécifiez.
  • Créer une Pull Request — disponible lorsque "Nouvelle branche" est sélectionnée. Ouvre automatiquement une pull request après le push. Affiche des champs pour le titre de la PR et la branche cible.

Squash de commits Ctrl+Shift+Q

Disponible uniquement avec l'authentification SSH et HTTPS (git binary).

Choisissez Fichier → Git Squash… ou appuyez sur Ctrl+Shift+Q. La boîte de dialogue Squash de commits affiche un tableau de tous les commits sur la branche actuelle qui ne sont pas encore dans la branche de base choisie.

  • Champ de branche de base — la branche de référence (par exemple main, master, develop). Cliquez sur Recharger après l'avoir modifiée.
  • Sélectionner tous les nouveaux commits — coche tous les commits listés à la fois.
  • Message de commit squash — le message unique qui remplacera tous les commits sélectionnés.
  • Squash & Push — réécrit l'historique de la branche et effectue un force-push vers le distant.

Vous devez sélectionner au moins 2 commits, et ils doivent former une plage contiguë commençant par le commit le plus récent (HEAD).

Changer de branche

Lorsqu'un fichier géré par Git est ouvert, la barre d'état affiche le nom de la branche actuelle en vert. Cliquez dessus pour ouvrir le dialogue Changer de branche.

  • Récupérer le dernier — retélécharger les dernières modifications pour la branche actuelle.
  • Filtre — tapez pour rechercher dans la liste des branches distantes.
  • Changer — sélectionnez une branche différente et cliquez sur Changer (ou double-cliquez) pour l'activer. Le fichier se recharge automatiquement avec le contenu de la nouvelle branche.

Les trois méthodes d'authentification sont prises en charge : HTTPS (API), HTTPS (git binary) et SSH.

Modes d'authentification

Configurez votre méthode d'authentification dans Affichage → Paramètres → Authentification Git.

HTTPS (intégré)

Fonctionne sur toutes les plateformes sans aucun logiciel supplémentaire installé. Utilise directement l'API REST de la plateforme. Aucun binaire git local n'est requis.

Limitation

Les fonctions amend et squash ne sont pas disponibles dans ce mode car aucun historique git local complet n'est maintenu.

PlateformeType de jeton
GitHubJeton d'accès personnel (ghp_…) — nécessite la portée repo
GitHub EnterpriseJeton d'accès personnel — identique à GitHub
Bitbucket CloudMot de passe d'application (Compte → Mots de passe d'application)
Bitbucket ServerJeton d'accès personnel ou jeton d'accès HTTP

HTTPS (git binary)

Utilise l'exécutable git du système pour toutes les opérations — clone local complet, commit et push. Nécessite Git for Windows (ou tout binaire git dans votre PATH). Prend en charge amend et squash. Utilisez les mêmes champs de nom d'utilisateur et de jeton que le mode HTTPS (intégré). Vous devez également fournir votre Nom et votre E-mail pour l'identité du commit.

SSH (fichier de clé)

Sélectionnez SSH (fichier de clé) et fournissez le chemin vers votre clé SSH privée et une phrase secrète facultative. Ce mode utilise paramiko pour une connexion SSH en Python pur (aucun client ssh système n'est requis). Si paramiko n'est pas installé, MarkForge se replie sur le client ssh système.

pip install paramiko

L'authentification SSH prend en charge amend et squash.

Stockage des identifiants

Les jetons, les phrases secrètes SSH et les mots de passe proxy sont stockés dans le trousseau de clés du système d'exploitation (GNOME Keyring / KWallet sous Linux, Keychain sous macOS, Windows Credential Manager sous Windows) — et non dans des fichiers de configuration en texte clair.

Proxy HTTP

Si votre réseau nécessite un proxy HTTP pour atteindre les serveurs Git, configurez-le dans Affichage → Paramètres → Proxy HTTP :

  • URL du proxy — par exemple http://proxy.company.com:8080
  • Nom d'utilisateur du proxy — laissez vide si aucune authentification n'est requise.
  • Mot de passe du proxy — laissez vide si aucune authentification n'est requise.

Laissez tous les champs proxy vides pour utiliser les paramètres proxy par défaut du système.

Fermer

Lorsque vous fermez MarkForge alors qu'un fichier géré par Git est ouvert, MarkForge vous demande si vous souhaitez supprimer le répertoire de clone temporaire. La suppression du répertoire libère de l'espace disque ; le conserver vous permet d'inspecter les fichiers clonés par la suite.
