MarkForge Gebruikershandleiding v1.8.0

1. Inleiding

MarkForge is een moderne, veelzijdige Markdown-editor gebouwd met Python en PyQt6. Het combineert een afleidingsvrije schrijfervaring met een realtime zij-aan-zij-voorvertoning, volledige Git-workflowintegratie, diagramondersteuning, wiskundig renderen en PDF-import/export — allemaal in één overzichtelijke bureaubladtoepassing met twee panelen.

Of u nu technische documentatie schrijft, project-README's maakt of inhoud rechtstreeks naar een GitHub-repository publiceert, MarkForge beheert de volledige bewerkings- en publicatielevenscyclus zonder een terminal te vereisen.

Live voorvertoningGitHub-gestijlde HTML naast elkaar weergegeven tijdens het typen
SyntaxisaccentueringMarkdown-bewust kleurenschema, donker & licht
Git-integratieOpenen, bewerken, committen en pushen naar GitHub of Bitbucket
DiagrammenPlantUML (12 sjablonen) en Mermaid (9 sjablonen)
Wiskundige formulesLaTeX-notatie weergegeven met MathJax
PDF importerenElk PDF automatisch naar Markdown converteren
PDF exporterenAfdrukgeoptimaliseerde PDF van elk document
HTML exporterenZelfstandige HTML met ingesloten CSS en afbeeldingen
Pandoc exporterenDOCX, EPUB, LaTeX via pandoc (optioneel)
MeertaligGebruikersinterface in het Engels en Duits

2. Installatie

Linux (Debian / Ubuntu)

Download het .deb-pakket van de Releases-pagina op GitHub en installeer het met:

sudo apt install ./markforge_*.deb

Het pakket installeert automatisch alle vereiste systeemafhankelijkheden (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments). Er is geen virtuele omgeving vereist.

Windows

Download MarkForge-Setup.exe van de Releases-pagina en voer het installatieprogramma uit. De installatiewizard begeleidt u door de installatie en maakt automatisch snelkoppelingen in het startmenu aan.

Opmerking

Het Windows-installatieprogramma bevat de Python-runtime en alle afhankelijkheden. Er is geen afzonderlijke Python-installatie vereist.

Vanuit broncode

Vereisten: Python 3.8 of hoger, CMake 3.16 of hoger.

Met CMake (aanbevolen):

# 1. Repository klonen
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. De build configureren (maakt een lokale virtuele omgeving aan)
cmake -B build

# 3. Python-afhankelijkheden in de lokale venv installeren
cmake --build build --target install_deps

# 4. De toepassing starten
cmake --build build --target run

Direct Python-start (zonder CMake):

pip install -r requirements.txt
python src/main.py
Tip

Voor SSH-gebaseerde Git-bewerkingen installeert u ook paramiko: pip install paramiko

Het paneel Documentoverzicht bevindt zich bovenaan de linker zijbalk en toont alle koppen (H1–H6) in het huidige document als een navigeerbare structuur.

  • Klik op een kop om de editorcursor naar die regel te verplaatsen.
  • Het overzicht wordt automatisch bijgewerkt tijdens het typen.
  • Zichtbaarheid in-/uitschakelen via Beeld → Overzicht tonen. De voorkeur wordt opgeslagen tussen sessies.

Alle werkbalkacties dupliceren menu-items en sneltoetsen.

Statusbalk

  • Cursorpositie — huidige regel en kolom, bijv. Regel 12, Kolom 5.

De venstertitel toont de naam van het momenteel geopende bestand. Wanneer u niet-opgeslagen wijzigingen aanbrengt, wordt een asterisk (*) aan de bestandsnaam toegevoegd, bijv. README.md *. Wanneer er geen bestand is geopend, toont de titel Naamloos.

4. De editor

Het middelste paneel bevat een volwaardige teksteditor afgestemd op het schrijven van Markdown.

Syntaxisaccentuering

MarkForge past Markdown-bewuste syntaxisaccentuering toe op het document tijdens het typen. Koppen, nadrukmarkeringen, koppelingen, code-spans, afgebakende codeblokken en andere Markdown-constructies worden met onderscheidende kleuren weergegeven. De accentuering past zich automatisch aan wanneer u wisselt tussen de lichte en donkere editorthema's.

Regelnummers

Een goot aan de linkerkant van de editor toont regelnummers. De huidige regel is gemarkeerd met een subtiele achtergrondkleur om u te helpen uw positie in lange documenten bij te houden.

Tekstterugloop

Tekstterugloop is standaard ingeschakeld. Lange regels lopen terug om in de zichtbare editorbreedte te passen. Schakel tekstterugloop op elk moment in/uit via Beeld → Tekstterugloop. De voorkeur wordt opgeslagen tussen sessies.

Editorthema's

Er zijn twee ingebouwde editorthema's beschikbaar:

  • Donker (standaard) — donkere achtergrond met hoog-contrast syntaxiskleuren.
  • Licht — witte achtergrond geschikt voor lichte omgevingen.

Wissel van thema in Beeld → Instellingen → Editorthema. De wijziging heeft direct effect zonder opnieuw opstarten.

Plaatsaanduidingstekst

Wanneer er geen bestand is geopend, is de editor uitgeschakeld en toont het bericht: "Open of maak een Markdown-bestand aan om te beginnen met bewerken." De editor wordt geactiveerd zodra u een nieuw bestand aanmaakt of een bestaand bestand opent.

Bewerkingsbewerkingen

Alle standaard teksteditorbewerkingen zijn beschikbaar:

  • Ctrl+Z — Ongedaan maken
  • Ctrl+Y — Opnieuw uitvoeren
  • Ctrl+X / Ctrl+C / Ctrl+V — Knippen / Kopiëren / Plakken

Zoeken & Vervangen

Druk op Ctrl+F om de zoekbalk te openen, of Ctrl+H om deze te openen met de vervangrij zichtbaar. De balk verschijnt boven de editor.

  • Typ in het zoekveld — alle overeenkomsten worden direct gemarkeerd in de editor.
  • Navigeer tussen overeenkomsten met de knoppen ↑ / ↓, of druk op Enter (volgende) / Shift+Enter (vorige) in het zoekveld.
  • De overeenkomstteller toont de huidige positie (bijv. 3 / 12).
  • Aa — hoofdlettergevoelig zoeken in-/uitschakelen.
  • [W] — gehele woordovereenkomst in-/uitschakelen.
  • Gebruik Vervangen om de huidige overeenkomst te vervangen, of Alles vervangen om alle gevallen tegelijk te vervangen.
  • Druk op Escape of klik op ✕ om de balk te sluiten en alle markeringen te verwijderen.

Als er tekst is geselecteerd in de editor wanneer u de balk opent, wordt het zoekveld vooraf ingevuld met de selectie (alleen enkellijnige selecties).

Spellingcontrole

MarkForge kan verkeerd gespelde woorden onderstrepen met een rode golvende lijn tijdens het typen. Spellingcontrole is standaard uitgeschakeld.

  • Inschakelen via Beeld → Spellingcontrole (wisselknop).
  • De woordenboektaal selecteren via Beeld → Spellingcontrole-taal. Beschikbare talen: English, Deutsch, Español, Français, Italiano, Nederlands, Português, عربي, فارسی, Tiếng Việt, Svenska, Українська, ಕನ್ನಡ, हिंदी. Talen EN/DE/ES/FR/IT/NL/PT/AR/FA gebruiken pyspellchecker; VI/SV/UK/KN/HI gebruiken enchant/Hunspell.
  • Woorden in inline code-spans (`…`) en URL's worden nooit onderstreept.
  • HOOFDLETTERAFKORTINGEN (HTML, API, URL…) worden overgeslagen.
  • De spellingcontrole-voorkeur en geselecteerde taal worden opgeslagen tussen sessies.
  • Vereist de bibliotheek pyspellchecker. Als deze niet is geïnstalleerd, wordt de wisselknop stilzwijgend genegeerd.

5. Live voorvertoning

Het rechter paneel toont een volledig weergegeven HTML-versie van uw Markdown-document, automatisch bijgewerkt tijdens het typen.

Weergave

De voorvertoning gebruikt een door GitHub geïnspireerd stijlblad voor een schone, leesbare uitvoer. Het ondersteunt alle Markdown-extensies inclusief tabellen, takenlijsten, voetnoten, code-accentuering, wiskunde en diagrammen. De weergave-pipeline verwerkt uw tekst via de Python-bibliotheek markdown met de extensies extra, codehilite, toc en sane_lists.

Behoud van scrolpositie

Wanneer u het document bewerkt, wordt de voorvertoning bijgewerkt na een korte vertraging van 300 ms. Voordat de HTML wordt vernieuwd, leest MarkForge de huidige scrolpositie uit de webweergave en herstelt deze na het weergeven, zodat de voorvertoning niet naar boven springt terwijl u in het midden van een document werkt.

Voorvertoning-thema's

Er zijn twee voorvertoning-thema's beschikbaar:

  • Donker (standaard) — donkere achtergrond passend bij het algehele toepassingsthema.
  • Licht — witte achtergrond voor een drukklare leeservaring.

Wissel via Beeld → Instellingen → Voorvertoning-thema. De wijziging heeft direct effect.

De voorvertoning in-/uitschakelen

Druk op Ctrl+Shift+P of gebruik Beeld → Voorvertoning tonen om het voorvertoning-paneel te verbergen of weer te geven. Het verbergen van de voorvertoning geeft de editor meer schermruimte.

Opmerking

Wiskundige formules (MathJax) en Mermaid-diagrammen vereisen een internetverbinding bij het eerste gebruik, aangezien de bibliotheken worden geladen van een CDN. PlantUML-diagrammen worden altijd opgehaald van plantuml.com en vereisen internettoegang.

6. Bestandsbewerkingen

Nieuw bestand

Druk op Ctrl+N of kies Bestand → Nieuw. Als het huidige document niet-opgeslagen wijzigingen heeft, vraagt MarkForge u om op te slaan voordat u verdergaat. Een nieuw leeg document wordt geopend en de editor wordt geactiveerd.

Bestand openen

Druk op Ctrl+O of kies Bestand → Openen… om een bestandsverkenner-dialoogvenster te openen. Markdown-bestanden (.md, .markdown), tekstbestanden (.txt) en alle andere bestanden zijn selecteerbaar. De bestandsstructuur wordt bijgewerkt om de map met het geopende bestand te tonen.

Map openen

Kies Bestand → Map openen… (Ctrl+Shift+O) om een map te selecteren. De bestandsstructuur toont dan alle .md-bestanden en afbeeldingen in die map. Klik op een bestand in de structuur om het in de editor te openen.

Opslaan

Druk op Ctrl+S of kies Bestand → Opslaan. Als het bestand vanuit een Git-repository is geopend, verschijnt er een Git commit & push-dialoogvenster (zie Opslaan via Git). Voor lokale bestanden wordt het bestand direct naar schijf geschreven en de asterisk in de titelbalk verdwijnt.

Opslaan als

Druk op Ctrl+Shift+S of kies Bestand → Opslaan als… om het huidige document op een nieuwe locatie of met een nieuwe naam op te slaan.

Bestandsstructuur

Het linker paneel toont een vlakke lijst van .md-bestanden en afbeeldingen in de huidige hoofdmap. Submappen zonder direct relevante bestanden zijn verborgen om overbodige informatie te verminderen. Klik op een bestand om het te openen. De koptekst van de structuur toont [GIT] reponaam in groen wanneer een door Git beheerd bestand is geopend.

Tip

Schakel de bestandsstructuur in/uit met Ctrl+B om de editorruimte te maximaliseren. Uw voorkeur wordt opgeslagen tussen sessies.

Recente bestanden

MarkForge houdt een lijst bij van de laatste 10 geopende bestanden. Toegang via Bestand → Recente bestanden.

  • Klik op een item om het bestand direct opnieuw te openen.
  • Als een bestand niet meer op schijf bestaat, wordt het automatisch van de lijst verwijderd.
  • Kies Recente bestanden wissen bovenaan het submenu om de volledige geschiedenis te wissen.
  • De lijst wordt opgeslagen tussen sessies.

MarkForge ondersteunt standaard CommonMark Markdown plus de volgende extensies: extra (tabellen, voetnoten, afkortingen), codehilite (syntaxisgeaccentueerde codeblokken), toc (inhoudsopgave), sane_lists en doorhalen.

Kop 2

Kop 3

8. Wiskundige formules

MarkForge geeft LaTeX wiskundige notatie weer met MathJax. Bij het eerste gebruik is een internetverbinding vereist (MathJax wordt geladen van een CDN).

Inline wiskunde

Omsluit een uitdrukking met enkele dollartekens om deze inline in een alinea weer te geven:

De beroemde vergelijking $E = mc^2$ werd gepubliceerd in 1905.

Resultaat: De beroemde vergelijking E = mc² werd gepubliceerd in 1905.

Blokwiskunde (gecentreerd)

Omsluit de uitdrukking met dubbele dollartekens om deze gecentreerd op een eigen regel weer te geven:

$$\int_0^\infty e^{-x}\, dx = 1$$

Enkele-regel blokwiskunde

U kunt ook een enkel $ op een eigen regel gebruiken om een weergave-wiskundeblok te openen en te sluiten:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
Tip

MathJax ondersteunt de volledige LaTeX wiskundemodus, inclusief breuken, matrices, Griekse letters, operatoren en aangepaste macro's. Raadpleeg de MathJax-documentatie voor de volledige syntaxis.

9. Invoegdialoogvensters

MarkForge biedt begeleide dialoogvensters voor het invoegen van veelgebruikte Markdown-constructies. Alle dialoogvensters voegen tekst in op de huidige cursorpositie in de editor.

Koppeling invoegen Ctrl+K

Openen via Invoegen → Koppeling… of druk op Ctrl+K.

Het dialoogvenster heeft drie velden:

  • Weergavetekst — het zichtbare koppelingslabel (bijv. Klik hier).
  • URL — het doeladres (bijv. https://example.com).
  • Titel (optioneel) — een tooltip die wordt weergegeven wanneer de gebruiker over de koppeling beweegt.

Klik op Invoegen om [Weergavetekst](URL "Titel") in de editor te plaatsen.

Afbeelding invoegen Ctrl+Shift+K

Openen via Invoegen → Afbeelding… of druk op Ctrl+Shift+K.

Het dialoogvenster heeft twee velden:

  • Alt-tekst — een korte beschrijving voor schermlezers.
  • URL / Pad — een externe URL of een lokaal bestandspad. Gebruik de bladerknop om een lokaal afbeeldingsbestand te selecteren.

Klik op Invoegen om ![Alt-tekst](URL) in de editor te plaatsen.

Tabel invoegen Ctrl+Shift+T

Openen via Invoegen → Tabel… of druk op Ctrl+Shift+T.

Configureer uw tabelstructuur:

  • Gegevensrijen — aantal rijen exclusief de koptekstrij.
  • Kolommen — aantal kolommen (tot het ondersteunde maximum).
  • Kolomnamen — voer een titel in voor elke kolom.
  • Uitlijning — kies Links, Midden of Rechts voor elke kolom afzonderlijk.

Een live Markdown-voorvertoning wordt onderaan het dialoogvenster bijgewerkt. Klik op Invoegen om de tabel in de editor te plaatsen.

PlantUML invoegen Ctrl+Shift+U

Openen via Invoegen → PlantUML… of druk op Ctrl+Shift+U.

Kies uit 12 diagramsjablonen in de vervolgkeuzelijst:

  • Sequentiediagram
  • Klassediagram
  • Activiteitsdiagram
  • Toestandsdiagram
  • Componentendiagram
  • Use-case-diagram
  • Gantt-diagram
  • Mindmap
  • WBS (Work breakdown)
  • Tijddiagram
  • Implementatiediagram
  • JSON

Het geselecteerde sjabloon vult de PlantUML-code-editor. Bewerk de code naar behoefte en klik vervolgens op Invoegen. Het diagram wordt ingepakt in een afgebakend codeblok en weergegeven via de online service plantuml.com in de voorvertoning. Een internetverbinding is vereist.

Opmerking

De diagramvoorvertoning verschijnt automatisch in het hoofdvenster na het invoegen. De weergave gebruikt de openbare plantuml.com-server; er is geen lokale PlantUML-installatie nodig.

Mermaid invoegen Ctrl+Shift+M

Openen via Invoegen → Mermaid… of druk op Ctrl+Shift+M.

Kies uit 9 diagramsjablonen:

  • Stroomdiagram
  • Sequentiediagram
  • Klassediagram
  • Toestandsdiagram
  • Gantt
  • Cirkeldiagram
  • Mindmap
  • ER-diagram
  • Git-grafiek

Bewerk de Mermaid-code in het tekstgebied en klik op Invoegen. Het diagram wordt ingepakt in een afgebakend mermaid-codeblok en weergegeven in de voorvertoning via kroki.io. Een internetverbinding is vereist.

10. PDF-functies

Exporteren als PDF Ctrl+Shift+E

Kies Bestand → Exporteren als PDF… of druk op Ctrl+Shift+E. MarkForge geeft het huidige document weer als PDF-bestand met behulp van dezelfde pipeline als de live voorvertoning, maar met een witte achtergrond en afdrukgeoptimaliseerde CSS. Er verschijnt een opslaandialoogvenster zodat u de uitvoerlocatie en bestandsnaam kunt kiezen.

Tip

Schakel de voorvertoning over naar het Licht-thema voordat u naar PDF exporteert voor de schoonste afdrukklare uitvoer.

Exporteren als HTML

Kies Bestand → Exporteren als HTML…. MarkForge rendert het huidige document naar een zelfstandig HTML-bestand met alle CSS inline en lokale afbeeldingen omgezet naar Base64 data-URI's, zodat het bestand in elke browser kan worden geopend zonder internetverbinding.

Exporteren via Pandoc (DOCX, EPUB, LaTeX)

Kies Bestand → Exporteren als DOCX (Pandoc)…, Exporteren als EPUB (Pandoc)… of Exporteren als LaTeX (Pandoc)…. Deze opties vereisen dat pandoc is geïnstalleerd en beschikbaar is in het PATH. Als pandoc niet wordt gevonden, verschijnt er een waarschuwingsdialoog.

PDF importeren Ctrl+Shift+I

Kies Bestand → PDF importeren… of druk op Ctrl+Shift+I. MarkForge converteert elk PDF-bestand naar Markdown met de bibliotheek pymupdf4llm. De conversie behoudt de documentstructuur intelligent:

  • Koppen en alinea's
  • Tabellen
  • Lijsten
  • Codeblokken waar gedetecteerd

De conversie wordt uitgevoerd in een achtergrondthread zodat de gebruikersinterface responsief blijft. Na voltooiing vraagt MarkForge u om een opslaglocatie voor het resulterende .md-bestand en laadt het vervolgens in de editor.

Opmerking

PDF importeren vereist het pakket pymupdf4llm. Als het niet is geïnstalleerd, vraagt MarkForge u het te installeren: pip install pymupdf4llm

11. Git-integratie

MarkForge kan Markdown-bestanden rechtstreeks openen vanuit gehoste Git-repositories, u ze laten bewerken en wijzigingen terugsturen — zonder ooit een terminal te openen. Geavanceerde bewerkingen zoals amend en squash worden ook ondersteund.

Het platform wordt automatisch gedetecteerd vanuit de URL-structuur, niet alleen de hostnaam — dus zelf-gehoste instanties werken automatisch.

Een bestand openen vanuit Git

  1. Kies Bestand → Bestand openen vanuit Git… of druk op Ctrl+Shift+G.
  2. Plak de URL van het bestand dat u wilt openen (kopieer het vanuit de GitHub of Bitbucket bestandsverkenner).
  3. MarkForge valideert de URL en toont een voorvertoningsregel: platform · owner/repo · pad/naar/bestand.md.
  4. Overschrijf optioneel de branch, tag of commit-hash in het veld Ref.
  5. Klik op OK — de repository wordt op de achtergrond gekloond en het bestand opent in de editor.

Terwijl een door Git beheerd bestand is geopend, toont de koptekst van de bestandsstructuur [GIT] reponaam in groen.

Tip

Openbare repositories werken zonder geconfigureerde inloggegevens. Configureer uw inloggegevens voor privé-repositories in Beeld → Instellingen → Git-authenticatie voordat u opent.

Opslaan (Commit & Push)

Wanneer een door Git beheerd bestand is geopend, opent u met Ctrl+S het dialoogvenster Git Commit & Push in plaats van het normale bestandsopslaan. Het dialoogvenster bevat:

  • Commit-bericht — vereist. Voer een zinvolle beschrijving van uw wijzigingen in.
  • Vorige commit wijzigen — beschikbaar vanaf de tweede opslag bij gebruik van SSH of git-binary-authenticatie. Vervangt de vorige commit in plaats van een nieuwe aan te maken; vult het vorige commit-bericht vooraf in; gebruikt automatisch force-push (--force-with-lease). Alleen commits gemaakt door MarkForge in de huidige sessie kunnen worden gewijzigd.
  • Pushen naar:
    • Huidige branch — pusht rechtstreeks naar de branch waarvan u hebt gekloond.
    • Nieuwe branch — maakt een nieuwe branch aan met een naam die u opgeeft.
  • Pull Request aanmaken — beschikbaar wanneer "Nieuwe branch" is geselecteerd. Opent automatisch een pull request na het pushen. Toont velden voor de PR-titel en de doelbranch.

Commits samenvoegen Ctrl+Shift+Q

Alleen beschikbaar met SSH en HTTPS (git binary)-authenticatie.

Kies Bestand → Git Squash… of druk op Ctrl+Shift+Q. Het Squash-dialoogvenster toont een tabel met alle commits op de huidige branch die nog niet in de gekozen basisbranch staan.

  • Basisbranchveld — de referentiebranch (bijv. main, master, develop). Klik op Herladen nadat u dit hebt gewijzigd.
  • Alle nieuwe commits selecteren — vinkt alle vermelde commits tegelijk aan.
  • Samengevoegd commit-bericht — het enkele bericht dat alle geselecteerde commits vervangt.
  • Samenvoegen & Pushen — herschrijft de branch-geschiedenis en force-pusht naar de remote.

U moet ten minste 2 commits selecteren, en ze moeten een aaneengesloten bereik vormen vanaf de meest recente commit (HEAD).

Authenticatiemodi

Configureer uw authenticatiemethode in Beeld → Instellingen → Git-authenticatie.

HTTPS (ingebouwd)

Werkt op alle platforms zonder extra software geïnstalleerd. Gebruikt de platform REST API rechtstreeks. Er is geen lokale git-binary vereist.

Beperking

De amend- en squash-functies zijn niet beschikbaar in deze modus omdat er geen volledige lokale git-geschiedenis wordt bijgehouden.

PlatformToken-type
GitHubPersoonlijk toegangstoken (ghp_…) — vereist repo-scope
GitHub EnterprisePersoonlijk toegangstoken — hetzelfde als GitHub
Bitbucket CloudApp-wachtwoord (Account → App-wachtwoorden)
Bitbucket ServerPersoonlijk toegangstoken of HTTP-toegangstoken

HTTPS (git binary)

Gebruikt het systeem git-uitvoerbaar bestand voor alle bewerkingen — volledige lokale kloon, commit en push. Vereist Git voor Windows (of een willekeurige git-binary in uw PATH). Ondersteunt amend en squash. Gebruik dezelfde gebruikersnaam- en tokenvelden als de HTTPS (ingebouwde) modus. U moet ook uw Naam en E-mail opgeven voor de commit-identiteit.

SSH (sleutelbestand)

Selecteer SSH (sleutelbestand) en geef het pad naar uw privé SSH-sleutel en een optionele wachtzin op. Deze modus gebruikt paramiko voor een pure-Python SSH-verbinding (geen systeem-ssh-client vereist). Als paramiko niet is geïnstalleerd, valt MarkForge terug op de systeem-ssh-client.

pip install paramiko

SSH-authenticatie ondersteunt amend en squash.

Opslag van inloggegevens

Tokens, SSH-wachtzinnen en proxywachtwoorden worden opgeslagen in de OS-sleutelhanger (GNOME Keyring / KWallet op Linux, Keychain op macOS, Windows Referentiebeheer op Windows) — niet in platte tekstconfiguratiebestanden.

HTTP-proxy

Als uw netwerk een HTTP-proxy vereist om Git-servers te bereiken, configureert u dit in Beeld → Instellingen → HTTP-proxy:

  • Proxy-URL — bijv. http://proxy.bedrijf.com:8080
  • Proxy-gebruikersnaam — leeg laten als er geen authenticatie vereist is.
  • Proxywachtwoord — leeg laten als er geen authenticatie vereist is.

Laat alle proxyvelden leeg om de standaard proxy-instellingen van het systeem te gebruiken.

Sluiten

Wanneer u MarkForge sluit terwijl een door Git beheerd bestand is geopend, vraagt MarkForge of u de tijdelijke kloonmap wilt verwijderen. Het verwijderen van de map maakt schijfruimte vrij; het bewaren stelt u in staat de gekloonde bestanden daarna te inspecteren.
