راهنمای کاربر MarkForge v1.7.1

۱. مقدمه

MarkForge یک ویرایشگر Markdown مدرن و پربار است که با Python و PyQt6 ساخته شده است. این برنامه تجربه نوشتاری بدون حواس‌پرتی را با پیش‌نمایش زنده کنار هم، یکپارچه‌سازی کامل با گردش‌کار Git، پشتیبانی از نمودار، رندرینگ ریاضی و وارد/صادر کردن PDF — همه در یک برنامه دسکتاپ تمیز با دو پانل — ترکیب می‌کند.

چه در حال نوشتن مستندات فنی، README پروژه‌ها، یا انتشار محتوا مستقیماً به مخزن GitHub باشید، MarkForge چرخه کامل ویرایش و انتشار را بدون نیاز به ترمینال مدیریت می‌کند.

پیش‌نمایش زندهHTML با سبک GitHub در کنار هم هنگام تایپ نمایش داده می‌شود
برجسته‌سازی نحوطرح رنگی آشنا با Markdown، تاریک و روشن
یکپارچه‌سازی Gitباز کردن، ویرایش، commit و push به GitHub یا Bitbucket
نمودارهاPlantUML (۱۲ قالب) و Mermaid (۹ قالب)
فرمول‌های ریاضینماد LaTeX با MathJax رندر می‌شود
وارد کردن PDFتبدیل خودکار هر PDF به Markdown
صادر کردن PDFPDF بهینه‌شده برای چاپ از هر سندی
چندزبانهرابط کاربری به زبان‌های انگلیسی و آلمانی

۲. نصب

Linux (Debian / Ubuntu)

بسته .deb را از صفحه Releases در GitHub دانلود کرده و با این دستور نصب کنید:

sudo apt install ./markforge_*.deb

این بسته به طور خودکار تمام وابستگی‌های سطح سیستم لازم را نصب می‌کند (python3-pyqt6، python3-pyqt6.qtwebengine، python3-markdown، python3-pygments). هیچ محیط مجازی لازم نیست.

Windows

فایل MarkForge-Setup.exe را از صفحه Releases دانلود کرده و نصاب را اجرا کنید. راهنمای نصب شما را در طول فرآیند نصب هدایت می‌کند و به طور خودکار میانبرهای منوی Start ایجاد می‌کند.

توجه

نصاب Windows شامل runtime Python و تمام وابستگی‌ها است. نصب جداگانه Python لازم نیست.

از کد منبع

پیش‌نیازها: Python 3.8 یا بالاتر، CMake 3.16 یا بالاتر.

با استفاده از CMake (توصیه می‌شود):

# 1. کلون کردن مخزن
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. پیکربندی ساخت (یک محیط مجازی محلی ایجاد می‌کند)
cmake -B build

# 3. نصب وابستگی‌های Python در venv محلی
cmake --build build --target install_deps

# 4. اجرای برنامه
cmake --build build --target run

اجرای مستقیم Python (بدون CMake):

pip install -r requirements.txt
python src/main.py
نکته

برای عملیات Git مبتنی بر SSH، paramiko را نیز نصب کنید: pip install paramiko

۳. مرور رابط کاربری

چیدمان

پنجره MarkForge به سه پانل قابل تغییر اندازه تقسیم شده است که با تقسیم‌کننده‌های کشیدنی از هم جدا می‌شوند:

پانلموقعیتتوضیحات
ساختار + درخت فایلچپپانل چپ دارای دو بخش تو در تو است: ساختار سند (درخت سرتیترها) در بالا، و درخت فایل (مرورگر فایل) در پایین. برای تغییر وضعیت ساختار از نمایش → نمایش ساختار، برای درخت فایل از Ctrl+B استفاده کنید.
ویرایشگرمرکزویرایشگر متن ساده با برجسته‌سازی نحو Markdown و شماره خط.
پیش‌نمایشراسترندرینگ زنده HTML سند جاری. با Ctrl+Shift+P تغییر وضعیت دهید.

هر دستگیره تقسیم‌کننده را بکشید تا اندازه پانل‌ها تغییر کند. اندازه پانل‌ها بین جلسات ذخیره می‌شود.

ساختار سند

پانل ساختار سند در بالای نوار کناری چپ قرار دارد و تمام سرتیترها (H1–H6) در سند جاری را به صورت یک درخت قابل ناوبری نشان می‌دهد.

  • روی هر سرتیتر کلیک کنید تا مکان‌نما ویرایشگر به آن خط برود.
  • ساختار به طور خودکار هنگام تایپ به‌روزرسانی می‌شود.
  • قابلیت مشاهده را از طریق نمایش → نمایش ساختار تغییر دهید. این تنظیم بین جلسات ذخیره می‌شود.

نوار ابزار

نوار ابزار زیر نوار منو با یک کلیک به رایج‌ترین عملکردها دسترسی می‌دهد: جدید، باز کردن، ذخیره، درج پیوند، درج تصویر، درج جدول، درج PlantUML، درج Mermaid، تغییر وضعیت پیش‌نمایش و تنظیمات. تمام عملکردهای نوار ابزار، موارد منو و میانبرهای صفحه‌کلید را تکرار می‌کنند.

نوار وضعیت

نوار وضعیت در پایین پنجره همیشه دو اطلاعات را نشان می‌دهد:

  • تعداد کلمات — تعداد کل کلمات در سند (هنگام تایپ به صورت زنده به‌روزرسانی می‌شود).
  • موقعیت مکان‌نما — شماره خط و ستون جاری، مثلاً خط ۱۲، ستون ۵.

نوار منو

نوار منو شامل پنج منوی سطح بالا است:

منومحتوا
فایلجدید، باز کردن، باز کردن پوشه، فایل‌های اخیر، وارد کردن PDF، باز کردن از Git، Git Squash، ذخیره، ذخیره به عنوان، صادر کردن به عنوان PDF، خروج
ویرایشبرگشت، بازگشت، برش، کپی، چسباندن
درجپیوند، تصویر، PlantUML، Mermaid، جدول
نمایشنمایش درخت فایل، نمایش ساختار، نمایش پیش‌نمایش، شکستن خط، بررسی غلط‌نما، زبان بررسی غلط‌نما، تنظیمات
راهنمامرجع Markdown، درباره

عنوان پنجره

عنوان پنجره نام فایل باز شده را نشان می‌دهد. هنگامی که تغییرات ذخیره نشده‌ای ایجاد می‌کنید، یک ستاره (*) به نام فایل اضافه می‌شود، مثلاً README.md *. وقتی هیچ فایلی باز نیست، عنوان بی‌عنوان را نشان می‌دهد.

۴. ویرایشگر

پانل مرکزی یک ویرایشگر متن ساده کامل‌ویژگی برای نوشتن Markdown را در خود جای داده است.

برجسته‌سازی نحو

MarkForge برجسته‌سازی نحو آشنا با Markdown را هنگام تایپ روی سند اعمال می‌کند. سرتیترها، نشانگرهای تأکید، پیوندها، span‌های کد، بلوک‌های کد محصور و سایر سازه‌های Markdown با رنگ‌های متمایز نمایش داده می‌شوند. برجسته‌سازی هنگام تغییر بین تم‌های ویرایشگر روشن و تاریک به طور خودکار تنظیم می‌شود.

شماره خط

یک حاشیه در سمت چپ ویرایشگر شماره خط‌ها را نشان می‌دهد. خط جاری با یک رنگ پس‌زمینه ظریف برجسته می‌شود تا به شما کمک کند موقعیت خود را در اسناد طولانی دنبال کنید.

شکستن خط

شکستن خط به طور پیش‌فرض فعال است. خط‌های طولانی تا جایی که در عرض قابل مشاهده ویرایشگر جا شوند شکسته می‌شوند. شکستن خط را در هر زمان از طریق نمایش → شکستن خط تغییر دهید. این تنظیم بین جلسات ذخیره می‌شود.

تم‌های ویرایشگر

دو تم ویرایشگر داخلی در دسترس است:

  • تاریک (پیش‌فرض) — پس‌زمینه تاریک با رنگ‌های نحو با کنتراست بالا.
  • روشن — پس‌زمینه سفید مناسب برای محیط‌های روشن.

تم‌ها را در نمایش → تنظیمات → تم ویرایشگر تغییر دهید. تغییر بلافاصله بدون راه‌اندازی مجدد اعمال می‌شود.

متن جانشین

وقتی هیچ فایلی باز نیست، ویرایشگر غیرفعال است و پیام زیر را نشان می‌دهد: "یک فایل Markdown باز کنید یا ایجاد کنید تا شروع به ویرایش کنید." ویرایشگر به محض ایجاد یا باز کردن یک فایل فعال می‌شود.

عملیات ویرایش

تمام عملیات ویرایش متن استاندارد در دسترس است:

  • Ctrl+Z — برگشت
  • Ctrl+Y — بازگشت
  • Ctrl+X / Ctrl+C / Ctrl+V — برش / کپی / چسباندن

یافتن & جایگزینی

برای باز کردن نوار یافتن Ctrl+F فشار دهید، یا Ctrl+H برای باز کردن آن با ردیف جایگزینی قابل مشاهده. این نوار بالای ویرایشگر ظاهر می‌شود.

  • در قسمت جستجو تایپ کنید — تمام تطابق‌ها فوراً در ویرایشگر برجسته می‌شوند.
  • با دکمه‌های ↑ / ↓ بین تطابق‌ها ناوبری کنید، یا در قسمت جستجو Enter (بعدی) / Shift+Enter (قبلی) فشار دهید.
  • شمارنده تطابق موقعیت جاری را نشان می‌دهد (مثلاً ۳ / ۱۲).
  • Aa — تغییر وضعیت جستجوی حساس به بزرگی و کوچکی حروف.
  • [W] — تغییر وضعیت تطابق کلمه کامل.
  • از جایگزینی برای جایگزینی تطابق جاری، یا جایگزینی همه برای جایگزینی تمام موارد به یکباره استفاده کنید.
  • برای بستن نوار و پاک کردن تمام برجسته‌سازی‌ها Escape فشار دهید یا روی ✕ کلیک کنید.

اگر هنگام باز کردن نوار متنی در ویرایشگر انتخاب شده باشد، قسمت جستجو با انتخاب پر می‌شود (فقط برای انتخاب‌های تک‌خطی).

بررسی غلط‌نما

MarkForge می‌تواند کلمات غلط‌نویسی‌شده را هنگام تایپ با خط موج‌دار قرمز زیر خط بکشد. بررسی غلط‌نما به طور پیش‌فرض غیرفعال است.

  • از طریق نمایش → بررسی غلط‌نما (تغییر وضعیت) فعال کنید.
  • زبان لغت‌نامه را از طریق نمایش → زبان بررسی غلط‌نما انتخاب کنید. زبان‌های موجود: English, Deutsch, Español, Français, Italiano, Nederlands, Português, عربي, فارسی, Tiếng Việt, Svenska, Українська, ಕನ್ನಡ, हिंदी. زبان‌های EN/DE/ES/FR/IT/NL/PT/AR/FA از pyspellchecker استفاده می‌کنند؛ VI/SV/UK/KN/HI از enchant/Hunspell استفاده می‌کنند.
  • کلمات درون span‌های کد inline (`…`) و URL‌ها هرگز زیر خط نمی‌روند.
  • مخفف‌های با حروف بزرگ (HTML، API، URL…) نادیده گرفته می‌شوند.
  • تنظیمات بررسی غلط‌نما و زبان انتخاب‌شده بین جلسات ذخیره می‌شوند.
  • نیازمند کتابخانه pyspellchecker است. اگر نصب نشده باشد، تغییر وضعیت بی‌صدا نادیده گرفته می‌شود.

۵. پیش‌نمایش زنده

پانل راست یک نسخه HTML کاملاً رندر شده از سند Markdown شما را نمایش می‌دهد که به طور خودکار هنگام تایپ به‌روزرسانی می‌شود.

رندرینگ

پیش‌نمایش از یک stylesheet الهام‌گرفته از GitHub برای خروجی تمیز و خوانا استفاده می‌کند. از تمام پسوندهای Markdown شامل جداول، فهرست وظایف، یادداشت‌های پانویس، برجسته‌سازی کد، ریاضی و نمودارها پشتیبانی می‌کند. pipeline رندرینگ متن شما را از طریق کتابخانه Python markdown با پسوندهای extra، codehilite، toc و sane_lists پردازش می‌کند.

حفظ موقعیت اسکرول

هنگامی که سند را ویرایش می‌کنید، پیش‌نمایش پس از یک تأخیر کوتاه ۳۰۰ میلی‌ثانیه به‌روزرسانی می‌شود. قبل از تازه‌سازی HTML، MarkForge موقعیت اسکرول جاری را از نمای وب می‌خواند و آن را پس از رندرینگ بازیابی می‌کند، به طوری که پیش‌نمایش هنگام کار در وسط سند به بالا نمی‌پرد.

تم‌های پیش‌نمایش

دو تم پیش‌نمایش در دسترس است:

  • تاریک (پیش‌فرض) — پس‌زمینه تاریک که با تم کلی برنامه مطابقت دارد.
  • روشن — پس‌زمینه سفید برای تجربه خواندن شبیه به چاپ.

از طریق نمایش → تنظیمات → تم پیش‌نمایش تغییر دهید. تغییر بلافاصله اعمال می‌شود.

تغییر وضعیت پیش‌نمایش

Ctrl+Shift+P فشار دهید یا از نمایش → نمایش پیش‌نمایش برای پنهان یا نمایش پانل پیش‌نمایش استفاده کنید. پنهان کردن پیش‌نمایش فضای بیشتری برای ویرایشگر ایجاد می‌کند.

توجه

فرمول‌های ریاضی (MathJax) و نمودارهای Mermaid در اولین استفاده نیاز به اتصال اینترنت دارند، زیرا کتابخانه‌ها از یک CDN بارگذاری می‌شوند. نمودارهای PlantUML همیشه از plantuml.com دریافت می‌شوند و نیاز به دسترسی به اینترنت دارند.

۶. عملیات فایل

فایل جدید

Ctrl+N فشار دهید یا فایل → جدید را انتخاب کنید. اگر سند جاری تغییرات ذخیره نشده داشته باشد، MarkForge قبل از ادامه از شما می‌خواهد ذخیره کنید. یک سند جدید و خالی باز می‌شود و ویرایشگر فعال می‌شود.

باز کردن فایل

Ctrl+O فشار دهید یا فایل → باز کردن… را انتخاب کنید تا یک دیالوگ مرورگر فایل باز شود. فایل‌های Markdown (.md، .markdown)، فایل‌های متنی (.txt) و تمام فایل‌های دیگر قابل انتخاب هستند. درخت فایل برای نشان دادن پوشه حاوی فایل باز شده به‌روزرسانی می‌شود.

باز کردن پوشه

فایل → باز کردن پوشه… (Ctrl+Shift+O) را انتخاب کنید تا یک دایرکتوری انتخاب شود. سپس درخت فایل تمام فایل‌های .md و تصاویر در آن دایرکتوری را نشان می‌دهد. روی هر فایل در درخت کلیک کنید تا در ویرایشگر باز شود.

ذخیره

Ctrl+S فشار دهید یا فایل → ذخیره را انتخاب کنید. اگر فایل از یک مخزن Git باز شده باشد، به جای ذخیره عادی یک دیالوگ Git commit & push ظاهر می‌شود (به ذخیره از طریق Git مراجعه کنید). برای فایل‌های محلی، فایل بلافاصله روی دیسک نوشته می‌شود و ستاره در نوار عنوان ناپدید می‌شود.

ذخیره به عنوان

Ctrl+Shift+S فشار دهید یا فایل → ذخیره به عنوان… را انتخاب کنید تا سند جاری در مکان جدید یا با نام جدیدی ذخیره شود.

درخت فایل

پانل چپ یک لیست تخت از فایل‌های .md و تصاویر در دایرکتوری ریشه جاری نشان می‌دهد. زیردایرکتوری‌هایی که فایل‌های مستقیماً مرتبط ندارند پنهان می‌شوند. روی یک فایل کلیک کنید تا باز شود. هدر درخت هنگامی که یک فایل مدیریت‌شده توسط Git باز است [GIT] نام‌مخزن را به رنگ سبز نشان می‌دهد.

نکته

درخت فایل را با Ctrl+B تغییر وضعیت دهید تا فضای ویرایشگر به حداکثر برسد. این تنظیم بین جلسات ذخیره می‌شود.

فایل‌های اخیر

MarkForge فهرستی از ۱۰ فایل آخری که باز کرده‌اید نگه می‌دارد. از طریق فایل → فایل‌های اخیر به آنها دسترسی داشته باشید.

  • روی هر ورودی کلیک کنید تا فایل بلافاصله دوباره باز شود.
  • اگر یک فایل دیگر روی دیسک وجود نداشته باشد، به طور خودکار از فهرست حذف می‌شود.
  • پاک کردن فایل‌های اخیر را در بالای زیرمنو انتخاب کنید تا کل تاریخچه پاک شود.
  • فهرست بین جلسات ذخیره می‌شود.

۸. فرمول‌های ریاضی

MarkForge نماد ریاضی LaTeX را با استفاده از MathJax رندر می‌کند. در اولین استفاده نیاز به اتصال اینترنت است (MathJax از CDN بارگذاری می‌شود).

ریاضی درونخطی

یک عبارت را در علامت‌های دلار تکی محصور کنید تا به صورت درونخطی در یک پاراگراف رندر شود:

The famous equation $E = mc^2$ was published in 1905.

نتیجه: The famous equation E = mc² was published in 1905.

ریاضی بلوکی (نمایشی)

عبارت را در علامت‌های دلار دوگانه محصور کنید تا در خط خودش به صورت مرکزی رندر شود:

$$\int_0^\infty e^{-x}\, dx = 1$$

ریاضی بلوکی تک‌خطی

می‌توانید از یک $ مجزا در خط خودش برای باز و بستن یک بلوک ریاضی نمایشی استفاده کنید:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
نکته

MathJax از حالت ریاضی LaTeX کامل پشتیبانی می‌کند، شامل کسرها، ماتریس‌ها، حروف یونانی، عملگرها و ماکروهای سفارشی. برای نحو کامل به مستندات MathJax مراجعه کنید.

۹. دیالوگ‌های درج

MarkForge دیالوگ‌های هدایت‌شده برای درج سازه‌های متداول Markdown ارائه می‌دهد. تمام دیالوگ‌ها متن را در موقعیت مکان‌نما جاری در ویرایشگر درج می‌کنند.

درج پیوند Ctrl+K

از طریق درج → پیوند… باز کنید یا Ctrl+K فشار دهید.

دیالوگ سه فیلد دارد:

  • متن نمایشی — برچسب پیوند قابل مشاهده (مثلاً اینجا کلیک کنید).
  • URL — آدرس مقصد (مثلاً https://example.com).
  • عنوان (اختیاری) — یک tooltip که هنگام hover کاربر روی پیوند نشان داده می‌شود.

روی درج کلیک کنید تا [متن نمایشی](URL "عنوان") در ویرایشگر قرار گیرد.

درج تصویر Ctrl+Shift+K

از طریق درج → تصویر… باز کنید یا Ctrl+Shift+K فشار دهید.

دیالوگ دو فیلد دارد:

  • متن جایگزین — توضیح کوتاه برای screen reader‌ها.
  • URL / مسیر — یک URL از راه دور یا مسیر فایل محلی. از دکمه مرور برای انتخاب یک فایل تصویر محلی استفاده کنید.

روی درج کلیک کنید تا ![متن جایگزین](URL) در ویرایشگر قرار گیرد.

درج جدول Ctrl+Shift+T

از طریق درج → جدول… باز کنید یا Ctrl+Shift+T فشار دهید.

ساختار جدول خود را پیکربندی کنید:

  • ردیف‌های داده — تعداد ردیف‌ها به استثنای ردیف هدر.
  • ستون‌ها — تعداد ستون‌ها (تا حداکثر پشتیبانی‌شده).
  • نام‌های ستون — یک عنوان برای هر ستون وارد کنید.
  • تراز — چپ، مرکز یا راست را برای هر ستون به طور مستقل انتخاب کنید.

یک پیش‌نمایش Markdown زنده در پایین دیالوگ به‌روزرسانی می‌شود. روی درج کلیک کنید تا جدول در ویرایشگر قرار گیرد.

درج PlantUML Ctrl+Shift+U

از طریق درج → PlantUML… باز کنید یا Ctrl+Shift+U فشار دهید.

از ۱۲ قالب نمودار در منوی کشویی انتخاب کنید:

  • نمودار توالی
  • نمودار کلاس
  • نمودار فعالیت
  • نمودار حالت
  • نمودار مؤلفه
  • نمودار مورد استفاده
  • نمودار گانت
  • نقشه ذهنی
  • WBS (ساختار شکست کار)
  • نمودار زمان‌بندی
  • نمودار استقرار
  • JSON

قالب انتخاب‌شده ویرایشگر کد PlantUML را پر می‌کند. کد را به دلخواه ویرایش کنید، سپس روی درج کلیک کنید. نمودار در یک بلوک کد محصور پیچیده می‌شود و از طریق سرویس آنلاین plantuml.com در پیش‌نمایش رندر می‌شود. اتصال اینترنت لازم است.

توجه

پیش‌نمایش نمودار پس از درج به طور خودکار در پنجره اصلی ظاهر می‌شود. رندرینگ از سرور عمومی plantuml.com استفاده می‌کند؛ نصب محلی PlantUML لازم نیست.

درج Mermaid Ctrl+Shift+M

از طریق درج → Mermaid… باز کنید یا Ctrl+Shift+M فشار دهید.

از ۹ قالب نمودار انتخاب کنید:

  • نمودار جریان
  • نمودار توالی
  • نمودار کلاس
  • نمودار حالت
  • گانت
  • نمودار دایره‌ای
  • نقشه ذهنی
  • نمودار ER
  • نمودار Git

کد Mermaid را در ناحیه متنی ویرایش کنید و روی درج کلیک کنید. نمودار در یک بلوک کد mermaid محصور پیچیده می‌شود و از طریق kroki.io در پیش‌نمایش رندر می‌شود. اتصال اینترنت لازم است.

۱۰. ویژگی‌های PDF

صادر کردن به عنوان PDF Ctrl+Shift+E

فایل → صادر کردن به عنوان PDF… را انتخاب کنید یا Ctrl+Shift+E فشار دهید. MarkForge سند جاری را با استفاده از همان pipeline پیش‌نمایش زنده به یک فایل PDF رندر می‌کند، اما با پس‌زمینه سفید و CSS بهینه‌شده برای چاپ. یک دیالوگ ذخیره ظاهر می‌شود تا مکان خروجی و نام فایل را انتخاب کنید.

نکته

قبل از صادر کردن به PDF، پیش‌نمایش را به تم روشن تغییر دهید تا خروجی تمیزترین حالت آماده برای چاپ را داشته باشد.

وارد کردن PDF Ctrl+Shift+I

فایل → وارد کردن PDF… را انتخاب کنید یا Ctrl+Shift+I فشار دهید. MarkForge هر فایل PDF را با استفاده از کتابخانه pymupdf4llm به Markdown تبدیل می‌کند. تبدیل ساختار سند را به طور هوشمند حفظ می‌کند:

  • سرتیترها و پاراگراف‌ها
  • جداول
  • فهرست‌ها
  • بلوک‌های کد در صورت تشخیص

تبدیل در یک thread پس‌زمینه اجرا می‌شود تا رابط کاربری پاسخگو بماند. پس از اتمام، MarkForge از شما می‌خواهد مکان ذخیره برای فایل .md حاصل را مشخص کنید و سپس آن را در ویرایشگر بارگذاری می‌کند.

توجه

وارد کردن PDF نیازمند بسته pymupdf4llm است. اگر نصب نشده باشد، MarkForge از شما می‌خواهد آن را نصب کنید: pip install pymupdf4llm

۱۱. یکپارچه‌سازی Git

MarkForge می‌تواند فایل‌های Markdown را مستقیماً از مخازن Git میزبانی‌شده باز کند، به شما اجازه ویرایش آنها را بدهد و تغییرات را بازگرداند — بدون اینکه نیازی به باز کردن ترمینال باشد. عملیات پیشرفته مانند amend و squash نیز پشتیبانی می‌شود.

پلتفرم‌های پشتیبانی‌شده

پلتفرمنمونه الگوی URL
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.mycompany.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.mycompany.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

پلتفرم به طور خودکار از ساختار URL، نه فقط نام میزبان، تشخیص داده می‌شود — بنابراین نمونه‌های خودمیزبان به طور خودکار کار می‌کنند.

باز کردن فایل از Git

  1. فایل → باز کردن از Git… را انتخاب کنید یا Ctrl+Shift+G فشار دهید.
  2. URL فایلی را که می‌خواهید باز کنید جای‌گذاری کنید (آن را از مرورگر فایل GitHub یا Bitbucket کپی کنید).
  3. MarkForge URL را تأیید می‌کند و یک خط پیش‌نمایش نشان می‌دهد: پلتفرم · owner/repo · مسیر/به/فایل.md.
  4. به صورت اختیاری شاخه، تگ یا هش commit را در فیلد Ref جایگزین کنید.
  5. روی OK کلیک کنید — مخزن در پس‌زمینه کلون می‌شود و فایل در ویرایشگر باز می‌شود.

در حالی که یک فایل مدیریت‌شده توسط Git باز است، هدر درخت فایل [GIT] نام‌مخزن را به رنگ سبز نشان می‌دهد.

نکته

مخازن عمومی بدون پیکربندی هیچ اعتباری کار می‌کنند. برای مخازن خصوصی، قبل از باز کردن، اعتبارنامه‌های خود را در نمایش → تنظیمات → احراز هویت Git پیکربندی کنید.

ذخیره (Commit & Push)

وقتی یک فایل مدیریت‌شده توسط Git باز است، فشار دادن Ctrl+S دیالوگ Git Commit & Push را به جای ذخیره معمول فایل باز می‌کند. دیالوگ شامل موارد زیر است:

  • پیام commit — اجباری. یک توضیح معنادار از تغییرات خود وارد کنید.
  • اصلاح commit قبلی — از ذخیره دوم به بعد هنگام استفاده از احراز هویت SSH یا git-binary در دسترس است. commit قبلی را به جای ایجاد یک commit جدید جایگزین می‌کند؛ پیام commit قبلی را از پیش پر می‌کند؛ به طور خودکار از force-push (--force-with-lease) استفاده می‌کند. فقط commit‌هایی که توسط MarkForge در جلسه جاری ایجاد شده‌اند می‌توانند اصلاح شوند.
  • Push به:
    • شاخه جاری — مستقیماً به شاخه‌ای که از آن کلون کرده‌اید push می‌کند.
    • شاخه جدید — یک شاخه جدید با نامی که مشخص می‌کنید ایجاد می‌کند.
  • ایجاد Pull Request — وقتی "شاخه جدید" انتخاب شده در دسترس است. پس از push به طور خودکار یک pull request باز می‌کند. فیلدهایی برای عنوان PR و شاخه هدف نشان می‌دهد.

Squash Commits Ctrl+Shift+Q

فقط با احراز هویت SSH و HTTPS (git binary) در دسترس است.

فایل → Git Squash… را انتخاب کنید یا Ctrl+Shift+Q فشار دهید. دیالوگ Squash جدولی از تمام commit‌های روی شاخه جاری که هنوز در شاخه پایه انتخاب‌شده نیستند نشان می‌دهد.

  • فیلد شاخه پایه — شاخه مرجع (مثلاً main، master، develop). پس از تغییر آن روی بارگذاری مجدد کلیک کنید.
  • انتخاب تمام commit‌های جدید — تمام commit‌های فهرست‌شده را به یکباره علامت‌گذاری می‌کند.
  • پیام commit squash — پیام واحدی که جایگزین تمام commit‌های انتخاب‌شده می‌شود.
  • Squash & Push — تاریخچه شاخه را بازنویسی می‌کند و force-push به remote می‌کند.

باید حداقل ۲ commit انتخاب کنید، و آنها باید یک محدوده پیوسته از جدیدترین commit (HEAD) تشکیل دهند.

حالت‌های احراز هویت

روش احراز هویت خود را در نمایش → تنظیمات → احراز هویت Git پیکربندی کنید.

HTTPS (تعبیه‌شده)

بر روی تمام پلتفرم‌ها بدون نصب نرم‌افزار اضافی کار می‌کند. مستقیماً از REST API پلتفرم استفاده می‌کند. نیازی به git binary محلی نیست.

محدودیت

ویژگی‌های amend و squash در این حالت در دسترس نیستند زیرا هیچ تاریخچه git محلی کامل نگه‌داری نمی‌شود.

پلتفرمنوع token
GitHubToken دسترسی شخصی (ghp_…) — نیاز به دسترسی repo دارد
GitHub EnterpriseToken دسترسی شخصی — مانند GitHub
Bitbucket CloudApp password (حساب → App passwords)
Bitbucket ServerToken دسترسی شخصی یا token دسترسی HTTP

HTTPS (git binary)

از فایل اجرایی git سیستم برای تمام عملیات استفاده می‌کند — کلون محلی کامل، commit و push. نیاز به Git for Windows (یا هر فایل git binary در PATH شما) دارد. از amend و squash پشتیبانی می‌کند. از همان فیلدهای نام کاربری و token مانند حالت HTTPS (تعبیه‌شده) استفاده کنید. باید نام و ایمیل خود را نیز برای هویت commit ارائه دهید.

SSH (فایل کلید)

SSH (فایل کلید) را انتخاب کنید و مسیر کلید SSH خصوصی و عبارت عبور اختیاری را ارائه دهید. این حالت از paramiko برای اتصال SSH کاملاً Python-based استفاده می‌کند (نیازی به client ssh سیستم نیست). اگر paramiko نصب نشده باشد، MarkForge به client ssh سیستم باز می‌گردد.

pip install paramiko

احراز هویت SSH از amend و squash پشتیبانی می‌کند.

ذخیره اعتبارنامه‌ها

Token‌ها، عبارات عبور SSH و رمزهای عبور proxy در keyring سیستم‌عامل ذخیره می‌شوند (GNOME Keyring / KWallet در Linux، Keychain در macOS، Windows Credential Manager در Windows) — نه در فایل‌های پیکربندی متن ساده.

پراکسی HTTP

اگر شبکه شما برای دسترسی به سرورهای Git نیاز به پراکسی HTTP دارد، آن را در نمایش → تنظیمات → پراکسی HTTP پیکربندی کنید:

  • URL پراکسی — مثلاً http://proxy.company.com:8080
  • نام کاربری پراکسی — اگر احراز هویت لازم نیست خالی بگذارید.
  • رمز عبور پراکسی — اگر احراز هویت لازم نیست خالی بگذارید.

تمام فیلدهای پراکسی را خالی بگذارید تا از تنظیمات پراکسی پیش‌فرض سیستم استفاده شود.

بستن

وقتی MarkForge را می‌بندید در حالی که یک فایل مدیریت‌شده توسط Git باز است، MarkForge می‌پرسد که آیا می‌خواهید دایرکتوری کلون موقت را حذف کنید. حذف دایرکتوری فضای دیسک آزاد می‌کند؛ نگه‌داشتن آن به شما اجازه می‌دهد فایل‌های کلون‌شده را بعداً بررسی کنید.
