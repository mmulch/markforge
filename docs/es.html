Manual de usuario de MarkForge v1.8.0

1. Introducción

MarkForge es un editor Markdown moderno y repleto de funciones, construido con Python y PyQt6. Combina una experiencia de escritura sin distracciones con una vista previa en tiempo real lado a lado, integración completa de flujo de trabajo con Git, soporte de diagramas, renderizado matemático e importación/exportación de PDF, todo en una limpia aplicación de escritorio de doble panel.

Ya sea que esté escribiendo documentación técnica, READMEs de proyectos o publicando contenido directamente en un repositorio de GitHub, MarkForge maneja el ciclo de vida completo de edición y publicación sin necesidad de abrir una terminal.

2. Instalación

Linux (Debian / Ubuntu)

Descargue el paquete .deb desde la página de Releases en GitHub y luego instálelo con:

sudo apt install ./markforge_*.deb

El paquete instala automáticamente todas las dependencias del sistema necesarias (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments). No se requiere un entorno virtual.

Windows

Descargue MarkForge-Setup.exe desde la página de Releases y ejecute el instalador. El asistente de instalación le guía a través del proceso y crea accesos directos en el menú de inicio automáticamente.

Nota

El instalador de Windows incluye el entorno de ejecución de Python y todas las dependencias. No se requiere una instalación separada de Python.

Desde el código fuente

Requisitos: Python 3.8 o posterior, CMake 3.16 o posterior.

Usando CMake (recomendado):

# 1. Clone the repository
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Configure the build (creates a local virtual environment)
cmake -B build

# 3. Install Python dependencies into the local venv
cmake --build build --target install_deps

# 4. Run the application
cmake --build build --target run

Inicio directo con Python (sin CMake):

pip install -r requirements.txt
python src/main.py
Consejo

Para operaciones Git basadas en SSH, instale también paramiko: pip install paramiko

3. Descripción de la interfaz

Diseño

La ventana de MarkForge está dividida en tres paneles redimensionables separados por divisores arrastrables:

EditorCentroEditor de texto sin formato con resaltado de sintaxis Markdown y números de línea.
Vista previaDerechaRenderizado HTML en vivo del documento actual. Active/desactive con Ctrl+Shift+P.

Arrastre cualquier divisor para cambiar el tamaño de los paneles. Los tamaños de los paneles se recuerdan entre sesiones.

Esquema del documento

El panel de Esquema del documento se encuentra en la parte superior de la barra lateral izquierda y lista todos los encabezados (H1–H6) del documento actual como un árbol navegable.

  • Haga clic en cualquier encabezado para mover el cursor del editor a esa línea.
  • El esquema se actualiza automáticamente mientras escribe.
  • Active/desactive la visibilidad mediante Ver → Mostrar esquema. La preferencia se guarda entre sesiones.

Barra de herramientas

La barra de herramientas debajo de la barra de menú proporciona acceso con un clic a las acciones más comunes: Nuevo, Abrir, Guardar, Insertar enlace, Insertar imagen, Insertar tabla, Insertar PlantUML, Insertar Mermaid, Alternar vista previa y Configuración. Todas las acciones de la barra de herramientas duplican los elementos del menú y los atajos de teclado.

Barra de estado

La barra de estado en la parte inferior de la ventana siempre muestra:

  • Recuento de palabras & tiempo de lectura — recuento de palabras y tiempo de lectura estimado (p.ej. 342 palabras · 2 min de lectura), actualizado en tiempo real.
  • Progreso del objetivo de palabras — si se establece un objetivo mediante Ver → Establecer objetivo de palabras, aparece una barra de progreso compacta.
  • Posición del cursor — línea y columna actuales, p.ej. Línea 12, Col 5.

Barra de menú

La barra de menú contiene cinco menús de nivel superior:

Título de la ventana

El título de la ventana muestra el nombre del archivo actualmente abierto. Cuando realiza cambios sin guardar, se agrega un asterisco (*) al nombre del archivo, por ejemplo README.md *. Cuando no hay ningún archivo abierto, el título muestra Untitled.

4. El editor

El panel central aloja un editor de texto sin formato con todas las funciones, optimizado para la creación de Markdown.

Resaltado de sintaxis

MarkForge aplica resaltado de sintaxis con reconocimiento de Markdown al documento mientras escribe. Los encabezados, marcadores de énfasis, enlaces, fragmentos de código, bloques de código delimitados y otras construcciones de Markdown se colorean de forma diferenciada. El resaltado se adapta automáticamente al cambiar entre los temas de editor Claro y Oscuro.

Números de línea

Un margen en el lado izquierdo del editor muestra los números de línea. La línea actual se resalta con un color de fondo sutil para ayudarle a rastrear su posición en documentos largos.

Marcadores de diferencias en línea

Cuando un archivo gestionado por Git está abierto, el margen del editor muestra barras de colores junto a las líneas que difieren de la última versión confirmada:

  • Verde — líneas añadidas
  • Naranja — líneas modificadas
  • Rojo — bloques eliminados (se muestra como una marca delgada sobre el punto de eliminación)

Los marcadores se actualizan en tiempo real mientras escribes y se restablecen después de cada commit & push exitoso.

Ajuste de línea

El ajuste de línea está habilitado de forma predeterminada. Las líneas largas se ajustan para encajar en el ancho visible del editor. Active/desactive el ajuste de línea en cualquier momento mediante Ver → Ajuste de línea. La preferencia se guarda entre sesiones.

Temas del editor

Hay dos temas de editor integrados disponibles:

  • Oscuro (predeterminado) — fondo oscuro con colores de sintaxis de alto contraste.
  • Claro — fondo blanco adecuado para entornos brillantes.

Cambie los temas en Ver → Configuración → Tema del editor. El cambio surte efecto inmediatamente sin reiniciar.

Texto de marcador de posición

Cuando no hay ningún archivo abierto, el editor está desactivado y muestra el mensaje: "Open or create a Markdown file to start editing." El editor se activa en cuanto crea un nuevo archivo o abre uno existente.

Operaciones de edición

Todas las operaciones estándar de edición de texto están disponibles:

  • Ctrl+Z — Deshacer
  • Ctrl+Y — Rehacer
  • Ctrl+X / Ctrl+C / Ctrl+V — Cortar / Copiar / Pegar

Buscar & Reemplazar

Presione Ctrl+F para abrir la barra de búsqueda, o Ctrl+H para abrirla con la fila de reemplazo visible. La barra aparece encima del editor.

  • Escriba en el campo de búsqueda — todas las coincidencias se resaltan instantáneamente en el editor.
  • Navegue entre las coincidencias con los botones ↑ / ↓, o presione Enter (siguiente) / Shift+Enter (anterior) en el campo de búsqueda.
  • El contador de coincidencias muestra la posición actual (por ejemplo, 3 / 12).
  • Aa — alternar búsqueda sensible a mayúsculas/minúsculas.
  • [W] — alternar coincidencia de palabras completas.
  • Use Reemplazar para reemplazar la coincidencia actual, o Reemplazar todo para reemplazar todas las ocurrencias a la vez.
  • Presione Escape o haga clic en ✕ para cerrar la barra y borrar todos los resaltados.

Si hay texto seleccionado en el editor cuando abre la barra, el campo de búsqueda se rellena previamente con la selección (solo selecciones de una línea).

Corrección ortográfica

MarkForge puede subrayar las palabras mal escritas con una línea ondulada roja mientras escribe. La corrección ortográfica está desactivada de forma predeterminada.

  • Actívela mediante Ver → Corrección ortográfica (interruptor).
  • Seleccione el idioma del diccionario mediante Ver → Idioma de corrección ortográfica. Idiomas disponibles: English, Deutsch, Español, Français, Italiano, Nederlands, Português, العربية, فارسی (vía pyspellchecker) y Tiếng Việt*, Svenska*, Українська*, ಕನ್ನಡ*, हिंदी* (vía enchant/Hunspell).
  • Las palabras dentro de fragmentos de código en línea (`…`) y las URL nunca se subrayan.
  • Las abreviaturas EN MAYÚSCULAS (HTML, API, URL…) se omiten.
  • La preferencia de corrección ortográfica y el idioma seleccionado se guardan entre sesiones.
  • Requiere la biblioteca pyspellchecker. Si no está instalada, el interruptor se ignora silenciosamente.
  • Los idiomas marcados con * requieren la biblioteca pyenchant y los diccionarios del sistema Hunspell (por ejemplo, hunspell-vi en Debian/Ubuntu).

5. Vista previa en vivo

El panel derecho muestra una versión HTML completamente renderizada de su documento Markdown, actualizada automáticamente mientras escribe.

Renderizado

La vista previa utiliza una hoja de estilos inspirada en GitHub para una salida limpia y legible. Admite todas las extensiones de Markdown, incluidas tablas, listas de tareas, notas al pie, resaltado de código, matemáticas y diagramas. La canalización de renderizado procesa su texto a través de la biblioteca Python markdown con las extensiones extra, codehilite, toc y sane_lists.

Preservación de la posición de desplazamiento

Cuando edita el documento, la vista previa se actualiza después de un breve retardo de debounce de 300 ms. Antes de actualizar el HTML, MarkForge lee la posición de desplazamiento actual de la vista web y la restaura después del renderizado, de modo que la vista previa no salte al inicio mientras trabaja en medio de un documento.

Temas de vista previa

Hay dos temas de vista previa disponibles:

  • Oscuro (predeterminado) — fondo oscuro que coincide con el tema general de la aplicación.
  • Claro — fondo blanco para una experiencia de lectura similar a la impresión.

Cambie mediante Ver → Configuración → Tema de vista previa. El cambio surte efecto inmediatamente.

Alternar la vista previa

Presione Ctrl+Shift+P o use Ver → Mostrar vista previa para ocultar o mostrar el panel de vista previa. Ocultar la vista previa da al editor más espacio en la pantalla.

Nota

Las fórmulas matemáticas (MathJax) y los diagramas Mermaid requieren una conexión a internet en el primer uso, ya que las bibliotecas se cargan desde una CDN. Los diagramas PlantUML siempre se obtienen de plantuml.com y requieren acceso a internet.

6. Operaciones de archivo

Nuevo archivo

Presione Ctrl+N o elija Archivo → Nuevo. Si el documento actual tiene cambios sin guardar, MarkForge le pedirá que guarde antes de continuar. Se abre un nuevo documento vacío y se activa el editor.

Abrir archivo

Presione Ctrl+O o elija Archivo → Abrir… para abrir un diálogo de explorador de archivos. Se pueden seleccionar archivos Markdown (.md, .markdown), archivos de texto (.txt) y todos los demás archivos. El árbol de archivos se actualizará para mostrar la carpeta que contiene el archivo abierto.

Abrir carpeta

Elija Archivo → Abrir carpeta… (Ctrl+Shift+O) para seleccionar un directorio. El árbol de archivos mostrará entonces todos los archivos .md e imágenes en ese directorio. Haga clic en cualquier archivo del árbol para abrirlo en el editor.

Guardar

Presione Ctrl+S o elija Archivo → Guardar. Si el archivo se abrió desde un repositorio Git, aparecerá un diálogo de commit & push de Git en su lugar (consulte Guardar a través de Git). Para archivos locales, el archivo se escribe inmediatamente en el disco y el asterisco en la barra de título desaparece.

Guardar como

Presione Ctrl+Shift+S o elija Archivo → Guardar como… para guardar el documento actual en una nueva ubicación o con un nuevo nombre.

Árbol de archivos

El panel izquierdo muestra una lista plana de archivos .md e imágenes en el directorio raíz actual. Los subdirectorios que no contienen archivos directamente relevantes están ocultos para reducir el desorden. Haga clic en un archivo para abrirlo. El encabezado del árbol muestra [GIT] reponame en verde cuando hay un archivo administrado por Git abierto.

Consejo

Active/desactive el árbol de archivos con Ctrl+B para maximizar el espacio del editor. Su preferencia se guarda entre sesiones.

Archivos recientes

MarkForge mantiene una lista de los últimos 10 archivos que ha abierto. Acceda a ellos mediante Archivo → Archivos recientes.

  • Haga clic en cualquier entrada para reabrir el archivo inmediatamente.
  • Si un archivo ya no existe en el disco, se elimina de la lista automáticamente.
  • Elija Borrar archivos recientes en la parte superior del submenú para borrar todo el historial.
  • La lista se guarda entre sesiones.

Arrastrar & Soltar

Puede arrastrar archivos desde su gestor de archivos al editor. MarkForge muestra un diálogo para elegir cómo manejar cada tipo de archivo:

  • Archivos de imagen (.png, .jpg, .gif, etc.) — elija insertar usando la ruta original o copiar a la carpeta assets/ e insertar un enlace relativo.
  • Archivos Markdown (.md, .markdown) — Abrir el archivo desde su ubicación original o Copiar a la carpeta (copia al mismo directorio que el documento actual y abre la copia).
  • Archivos PDF — Importar … (flujo de importación estándar con diálogo de guardado) o Importar a la carpeta (importa automáticamente al directorio del documento actual).
  • Cualquier otro archivo — inserta un enlace Markdown [nombre](ruta/al/archivo).
Nota

Para las operaciones que copian o importan a la carpeta del documento actual, debe haber un archivo abierto y guardado para que MarkForge conozca el directorio de destino.

Modo enfoque F11

Pulse F11 o elija Ver → Modo enfoque para entrar en el modo de escritura sin distracciones. MarkForge oculta el árbol de archivos, el esquema del documento, la vista previa en vivo y la barra de estado, dejando el editor ocupando toda la ventana.

Pulse F11 de nuevo para salir: todos los paneles y la barra de estado se restauran exactamente como estaban antes.

Consejo

El modo enfoque funciona independientemente de los controles individuales de panel (Ctrl+B, Ctrl+Shift+P).

7. Referencia de Markdown

MarkForge admite el Markdown CommonMark estándar más las siguientes extensiones: extra (tablas, notas al pie, abreviaturas), codehilite (bloques de código resaltados con sintaxis), toc (tabla de contenidos), sane_lists y tachado.

Resultado

Heading 2

Heading 3

8. Fórmulas matemáticas

MarkForge renderiza la notación matemática LaTeX usando MathJax. Se requiere una conexión a internet en el primer uso (MathJax se carga desde una CDN).

Matemáticas en línea

Envuelva una expresión en signos de dólar simples para renderizarla en línea dentro de un párrafo:

The famous equation $E = mc^2$ was published in 1905.

Resultado: La famosa ecuación E = mc² fue publicada en 1905.

Matemáticas en bloque

Envuelva la expresión en signos de dólar dobles para renderizarla centrada en su propia línea:

$$\int_0^\infty e^{-x}\, dx = 1$$

Matemáticas en bloque de una sola línea

También puede usar un $ solitario en su propia línea para abrir y cerrar un bloque de matemáticas en pantalla:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
Consejo

MathJax admite el modo matemático LaTeX completo, incluidas fracciones, matrices, letras griegas, operadores y macros personalizados. Consulte la documentación de MathJax para la sintaxis completa.

9. Diálogos de inserción

MarkForge proporciona diálogos guiados para insertar construcciones comunes de Markdown. Todos los diálogos insertan texto en la posición actual del cursor en el editor.

Insertar enlace Ctrl+K

Abra mediante Insertar → Enlace… o presione Ctrl+K.

El diálogo tiene tres campos:

  • Texto de visualización — la etiqueta de enlace visible (por ejemplo, Haga clic aquí).
  • URL — la dirección de destino (por ejemplo, https://example.com).
  • Título (opcional) — un tooltip que se muestra cuando el usuario pasa el ratón sobre el enlace.

Haga clic en Insertar para colocar [Texto de visualización](URL "Título") en el editor.

Insertar imagen Ctrl+Shift+K

Abra mediante Insertar → Imagen… o presione Ctrl+Shift+K.

El diálogo tiene dos campos:

  • Texto alternativo — una descripción breve para lectores de pantalla.
  • URL / Ruta — una URL remota o una ruta de archivo local. Use el botón de navegación para seleccionar un archivo de imagen local.

Haga clic en Insertar para colocar ![Texto alternativo](URL) en el editor.

Insertar tabla Ctrl+Shift+T

Abra mediante Insertar → Tabla… o presione Ctrl+Shift+T.

Configure la estructura de su tabla:

  • Filas de datos — número de filas excluyendo la fila de encabezado.
  • Columnas — número de columnas (hasta el máximo admitido).
  • Nombres de columnas — ingrese un título para cada columna.
  • Alineación — elija Izquierda, Centro o Derecha para cada columna de forma independiente.

Una vista previa de Markdown en vivo se actualiza en la parte inferior del diálogo. Haga clic en Insertar para colocar la tabla en el editor.

Insertar PlantUML Ctrl+Shift+U

Abra mediante Insertar → PlantUML… o presione Ctrl+Shift+U.

Elija entre 12 plantillas de diagrama en el desplegable:

  • Diagrama de secuencia
  • Diagrama de clases
  • Diagrama de actividad
  • Diagrama de estado
  • Diagrama de componentes
  • Diagrama de casos de uso
  • Diagrama de Gantt
  • Mapa mental
  • WBS (Desglose del trabajo)
  • Diagrama de temporización
  • Diagrama de despliegue
  • JSON

La plantilla seleccionada rellena el editor de código PlantUML. Edite el código según sea necesario, luego haga clic en Insertar. El diagrama se envuelve en un bloque de código delimitado y se renderiza a través del servicio en línea plantuml.com en la vista previa. Se requiere una conexión a internet.

Nota

La vista previa del diagrama aparece automáticamente en la ventana principal después de insertar. El renderizado usa el servidor público plantuml.com; no se necesita ninguna instalación local de PlantUML.

Insertar Mermaid Ctrl+Shift+M

Abra mediante Insertar → Mermaid… o presione Ctrl+Shift+M.

Elija entre 9 plantillas de diagrama:

  • Diagrama de flujo
  • Diagrama de secuencia
  • Diagrama de clases
  • Diagrama de estado
  • Gantt
  • Gráfico circular
  • Mapa mental
  • Diagrama ER
  • Gráfico de Git

Edite el código Mermaid en el área de texto y haga clic en Insertar. El diagrama se envuelve en un bloque de código mermaid delimitado y se renderiza en la vista previa a través de kroki.io. Se requiere una conexión a internet.

10. Funciones PDF

Exportar como PDF Ctrl+Shift+E

Elija Archivo → Exportar como PDF… o presione Ctrl+Shift+E. MarkForge renderiza el documento actual a un archivo PDF usando el mismo proceso que la vista previa en vivo, pero con un fondo blanco y CSS optimizado para impresión. Aparece un diálogo de guardar archivo para que pueda elegir la ubicación de salida y el nombre del archivo.

Consejo

Cambie la vista previa al tema Claro antes de exportar a PDF para obtener la salida más limpia lista para imprimir.

Exportar como HTML

Elija Archivo → Exportar como HTML…. MarkForge renderiza el documento actual a un archivo HTML autocontenido con todo el CSS integrado y las imágenes locales convertidas en URI de datos Base64, de modo que el archivo se abre en cualquier navegador sin conexión a internet.

Exportar mediante Pandoc (DOCX, EPUB, LaTeX)

Elija Archivo → Exportar como DOCX (Pandoc)…, Exportar como EPUB (Pandoc)… o Exportar como LaTeX (Pandoc)…. Estas opciones requieren que pandoc esté instalado y disponible en el PATH. Si pandoc no se encuentra, se muestra un diálogo de advertencia.

Importar PDF Ctrl+Shift+I

Elija Archivo → Importar PDF… o presione Ctrl+Shift+I. MarkForge convierte cualquier archivo PDF a Markdown usando la biblioteca pymupdf4llm. La conversión preserva inteligentemente la estructura del documento:

  • Encabezados y párrafos
  • Tablas
  • Listas
  • Bloques de código donde se detecten

La conversión se ejecuta en un subproceso en segundo plano para que la interfaz de usuario permanezca receptiva. Cuando se completa, MarkForge le pide una ubicación de guardado para el archivo .md resultante y luego lo carga en el editor.

Nota

La importación de PDF requiere el paquete pymupdf4llm. Si no está instalado, MarkForge le pedirá que lo instale: pip install pymupdf4llm

11. Integración con Git

MarkForge puede abrir archivos Markdown directamente desde repositorios Git alojados, permitirle editarlos y enviar los cambios de vuelta, sin necesidad de abrir una terminal. Las operaciones avanzadas como amend y squash también son compatibles.

Plataformas compatibles

PlataformaEjemplo de patrón de URL
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.mycompany.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.mycompany.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

La plataforma se detecta automáticamente a partir de la estructura de URL, no solo del nombre de host, por lo que las instancias alojadas automáticamente funcionan de forma automática.

Abrir un archivo desde Git

  1. Elija Archivo → Abrir desde Git… o presione Ctrl+Shift+G.
  2. Pegue la URL del archivo que desea abrir (cópiela desde el explorador de archivos de GitHub o Bitbucket).
  3. MarkForge valida la URL y muestra una línea de vista previa: plataforma · propietario/repo · ruta/al/archivo.md.
  4. Opcionalmente, anule la rama, etiqueta o hash de commit en el campo Ref.
  5. Haga clic en OK — el repositorio se clona en segundo plano y el archivo se abre en el editor.

Mientras un archivo administrado por Git está abierto, el encabezado del árbol de archivos muestra [GIT] reponame en verde.

Consejo

Los repositorios públicos funcionan sin ninguna credencial configurada. Para repositorios privados, configure sus credenciales en Ver → Configuración → Autenticación Git antes de abrir.

Guardar (Commit & Push)

Cuando un archivo administrado por Git está abierto, presionar Ctrl+S abre el diálogo Git Commit & Push en lugar del guardado normal del archivo. El diálogo contiene:

  • Mensaje de commit — requerido. Ingrese una descripción significativa de sus cambios.
  • Enmendar commit anterior — disponible a partir del segundo guardado cuando se usa autenticación SSH o git-binary. Reemplaza el commit anterior en lugar de crear uno nuevo; rellena previamente el mensaje de commit anterior; usa automáticamente force-push (--force-with-lease). Solo se pueden enmendar los commits realizados por MarkForge en la sesión actual.
  • Push a:
    • Rama actual — empuja directamente a la rama desde la que clonó.
    • Nueva rama — crea una nueva rama con un nombre que especifique.
  • Crear Pull Request — disponible cuando se selecciona "Nueva rama". Abre automáticamente un pull request después de hacer push. Muestra campos para el título del PR y la rama de destino.

Squash de commits Ctrl+Shift+Q

Disponible solo con autenticación SSH y HTTPS (git binary).

Elija Archivo → Git Squash… o presione Ctrl+Shift+Q. El diálogo de Squash de commits muestra una tabla de todos los commits en la rama actual que aún no están en la rama base elegida.

  • Campo de rama base — la rama de referencia (por ejemplo, main, master, develop). Haga clic en Recargar después de cambiarlo.
  • Seleccionar todos los commits nuevos — marca todos los commits listados a la vez.
  • Mensaje de commit squash — el mensaje único que reemplazará todos los commits seleccionados.
  • Squash & Push — reescribe el historial de la rama y hace force-push al remoto.

Debe seleccionar al menos 2 commits, y deben formar un rango contiguo comenzando desde el commit más reciente (HEAD).

Cambiar rama

Cuando un archivo gestionado por Git está abierto, la barra de estado muestra el nombre de la rama actual en verde. Haz clic en él para abrir el diálogo Cambiar rama.

  • Obtener más reciente — volver a descargar los últimos cambios para la rama actual.
  • Filtro — escribe para buscar en la lista de ramas remotas.
  • Cambiar — selecciona una rama diferente y haz clic en Cambiar (o doble clic) para activarla. El archivo se recarga automáticamente con el contenido de la nueva rama.

Se admiten los tres métodos de autenticación: HTTPS (API), HTTPS (git binary) y SSH.

Modos de autenticación

Configure su método de autenticación en Ver → Configuración → Autenticación Git.

HTTPS (integrado)

Funciona en todas las plataformas sin ningún software adicional instalado. Usa la API REST de la plataforma directamente. No se requiere ningún binario git local.

Limitación

Las funciones de amend y squash no están disponibles en este modo porque no se mantiene ningún historial git local completo.

PlataformaTipo de token
GitHubToken de acceso personal (ghp_…) — requiere el alcance repo
GitHub EnterpriseToken de acceso personal — igual que GitHub
Bitbucket CloudContraseña de aplicación (Cuenta → Contraseñas de aplicación)
Bitbucket ServerToken de acceso personal o token de acceso HTTP

HTTPS (git binary)

Usa el ejecutable git del sistema para todas las operaciones: clonación local completa, commit y push. Requiere Git for Windows (o cualquier binario git en su PATH). Admite amend y squash. Use los mismos campos de nombre de usuario y token que el modo HTTPS (integrado). También debe proporcionar su Nombre y Correo electrónico para la identidad del commit.

SSH (archivo de clave)

Seleccione SSH (archivo de clave) y proporcione la ruta a su clave SSH privada y una frase de contraseña opcional. Este modo usa paramiko para una conexión SSH en Python puro (no se requiere cliente ssh del sistema). Si paramiko no está instalado, MarkForge recurre al cliente ssh del sistema.

pip install paramiko

La autenticación SSH admite amend y squash.

Almacenamiento de credenciales

Los tokens, frases de contraseña SSH y contraseñas de proxy se almacenan en el llavero del sistema operativo (GNOME Keyring / KWallet en Linux, Keychain en macOS, Windows Credential Manager en Windows), no en archivos de configuración en texto plano.

Proxy HTTP

Si su red requiere un proxy HTTP para llegar a los servidores Git, configúrelo en Ver → Configuración → Proxy HTTP:

  • URL del proxy — por ejemplo, http://proxy.company.com:8080
  • Nombre de usuario del proxy — déjelo vacío si no se requiere autenticación.
  • Contraseña del proxy — déjela vacía si no se requiere autenticación.

Deje todos los campos de proxy vacíos para usar la configuración de proxy predeterminada del sistema.

Cerrar

Cuando cierre MarkForge mientras un archivo administrado por Git está abierto, MarkForge le preguntará si desea eliminar el directorio de clonación temporal. Eliminar el directorio libera espacio en disco; conservarlo le permite inspeccionar los archivos clonados después.
