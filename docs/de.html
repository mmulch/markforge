MarkForge Benutzerhandbuch v1.6.1

1. Einleitung

MarkForge ist ein moderner, funktionsreicher Markdown-Editor, der mit Python und PyQt6 entwickelt wurde. Er vereint eine ablenkungsfreie Schreibumgebung mit einer Echtzeit-Seitenvorschau, vollständiger Git-Workflow-Integration, Diagrammunterstützung, Mathematik-Rendering sowie PDF-Import und -Export — alles in einer übersichtlichen Desktop-Anwendung.

Ob Sie technische Dokumentation, Projekt-READMEs oder Inhalte schreiben, die direkt in ein GitHub-Repository veröffentlicht werden sollen — MarkForge begleitet Sie vom Schreiben bis zum Veröffentlichen, ganz ohne Terminal.

SyntaxhervorhebungMarkdown-bewusstes Farbschema, dunkel & hell
Git-IntegrationÖffnen, bearbeiten, committen und pushen zu GitHub oder Bitbucket
DiagrammePlantUML (12 Vorlagen) und Mermaid (9 Vorlagen)
Mathematische FormelnLaTeX-Notation, gerendert mit MathJax
PDF-ImportBeliebige PDF-Dateien automatisch in Markdown umwandeln
PDF-ExportDruckoptimiertes PDF aus jedem Dokument erstellen
MehrsprachigDeutsch und Englische Benutzeroberfläche

2. Installation

Linux (Debian / Ubuntu)

Laden Sie das .deb-Paket von der Releases-Seite auf GitHub herunter und installieren Sie es mit:

sudo apt install ./markforge_*.deb

Das Paket installiert automatisch alle erforderlichen Systemabhängigkeiten (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments). Eine virtuelle Umgebung ist nicht erforderlich.

Windows

Laden Sie MarkForge-Setup.exe von der Releases-Seite herunter und starten Sie den Installer. Der Einrichtungsassistent führt Sie durch die Installation und erstellt automatisch Startmenü-Einträge.

Hinweis

Der Windows-Installer enthält die Python-Laufzeitumgebung und alle Abhängigkeiten. Eine separate Python-Installation ist nicht erforderlich.

Aus dem Quellcode

Voraussetzungen: Python 3.8 oder neuer, CMake 3.16 oder neuer.

Mit CMake (empfohlen):

# 1. Repository klonen
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Build konfigurieren (erstellt eine lokale virtuelle Umgebung)
cmake -B build

# 3. Python-Abhängigkeiten in die lokale venv installieren
cmake --build build --target install_deps

# 4. Anwendung starten
cmake --build build --target run

Direkter Python-Start (ohne CMake):

pip install -r requirements.txt
python src/main.py
Tipp

Für SSH-basierte Git-Operationen installieren Sie zusätzlich paramiko: pip install paramiko

3. Benutzeroberfläche

Layout

Das MarkForge-Fenster ist in drei Bereiche unterteilt, die durch verschiebbare Trennleisten voneinander getrennt sind:

BereichPositionBeschreibung
Gliederung + DateibaumLinksDas linke Panel enthält zwei gestapelte Bereiche: die Dokumentgliederung (Überschriftenbaum) oben und den Dateibaum (Dateibrowser) unten. Gliederung ein-/ausblenden via Ansicht → Gliederung anzeigen, Dateibaum mit Ctrl+B.
EditorMitteKlartexteditor mit Markdown-Syntaxhervorhebung und Zeilennummern.

Ziehen Sie eine beliebige Trennleiste, um die Bereiche in der Größe anzupassen. Die Einstellungen werden sitzungsübergreifend gespeichert.

Dokumentgliederung

Die Dokumentgliederung im linken Panel listet alle Überschriften (H1–H6) des aktuellen Dokuments als navigierbaren Baum auf.

  • Klicken Sie auf eine Überschrift, um den Editor-Cursor zu dieser Zeile zu springen.
  • Die Gliederung wird automatisch beim Tippen aktualisiert.
  • Sichtbarkeit ein-/ausblenden via Ansicht → Gliederung anzeigen. Die Einstellung wird sitzungsübergreifend gespeichert.

Werkzeugleiste

Die Werkzeugleiste unterhalb der Menüleiste bietet schnellen Zugriff auf die häufigsten Aktionen: Neu, Öffnen, Speichern, Link einfügen, Bild einfügen, Tabelle einfügen, PlantUML einfügen, Mermaid einfügen, Vorschau umschalten und Einstellungen. Alle Schaltflächen entsprechen Menüeinträgen und Tastenkürzel.

Statusleiste

Die Statusleiste am unteren Rand des Fensters zeigt stets zwei Informationen:

  • Wörteranzahl — die Gesamtzahl der Wörter im Dokument (wird beim Tippen aktualisiert).
  • Cursorposition — aktuelle Zeile und Spalte, z. B. Zeile 12, Spalte 5.

Menüleiste

Die Menüleiste enthält fünf Hauptmenüs:

Fenstertitel

Der Fenstertitel zeigt den Namen der aktuell geöffneten Datei. Bei ungespeicherten Änderungen wird ein Sternchen (*) an den Dateinamen angehängt, z. B. README.md *. Wenn keine Datei geöffnet ist, erscheint Unbenannt.

4. Der Editor

Der mittlere Bereich enthält einen vollwertigen Klartexteditor, der speziell für das Schreiben von Markdown optimiert ist.

Syntaxhervorhebung

MarkForge wendet beim Tippen eine Markdown-bewusste Syntaxhervorhebung auf das Dokument an. Überschriften, Betonungszeichen, Links, Code-Spans, abgegrenzte Codeblöcke und andere Markdown-Konstrukte werden in unterschiedlichen Farben dargestellt. Die Hervorhebung passt sich automatisch an, wenn Sie zwischen den hellen und dunklen Editor-Themes wechseln.

Zeilennummern

Ein Randbereich auf der linken Seite des Editors zeigt Zeilennummern an. Die aktuelle Zeile ist mit einem subtilen Hintergrund hervorgehoben, damit Sie Ihre Position in langen Dokumenten leichter verfolgen können.

Zeilenumbruch

Der Zeilenumbruch ist standardmäßig aktiviert. Lange Zeilen werden umgebrochen, um in die sichtbare Editorbreite zu passen. Schalten Sie den Zeilenumbruch jederzeit über Ansicht → Zeilenumbruch um. Diese Einstellung wird sitzungsübergreifend gespeichert.

Editor-Themes

Es stehen zwei integrierte Editor-Themes zur Verfügung:

  • Dunkel (Standard) — dunkler Hintergrund mit kontrastreichen Syntaxfarben.
  • Hell — weißer Hintergrund für helle Umgebungen.

Wechseln Sie das Theme über Ansicht → Einstellungen → Editor-Theme. Die Änderung wird sofort übernommen, ohne dass ein Neustart erforderlich ist.

Platzhaltertext

Wenn keine Datei geöffnet ist, ist der Editor deaktiviert und zeigt die Meldung: „Öffnen oder erstellen Sie eine Markdown-Datei, um mit der Bearbeitung zu beginnen." Der Editor wird aktiviert, sobald Sie eine neue Datei erstellen oder eine vorhandene öffnen.

Bearbeitungsoperationen

Alle Standard-Textbearbeitungsoperationen stehen zur Verfügung:

  • Ctrl+Z — Rückgängig
  • Ctrl+Y — Wiederholen
  • Ctrl+X / Ctrl+C / Ctrl+V — Ausschneiden / Kopieren / Einfügen

Suchen & Ersetzen

Drücken Sie Ctrl+F, um die Suchleiste zu öffnen, oder Ctrl+H, um sie mit der Ersetzen-Zeile anzuzeigen. Die Leiste erscheint oberhalb des Editors.

  • Tippen Sie im Suchfeld — alle Treffer werden sofort im Editor hervorgehoben.
  • Navigieren Sie mit den Schaltflächen ↑ / ↓ oder drücken Sie im Suchfeld Enter (nächstes) / Shift+Enter (vorheriges).
  • Der Trefferzähler zeigt die aktuelle Position an (z. B. 3 / 12).
  • Aa — Groß-/Kleinschreibung beachten.
  • [W] — Nur ganze Wörter suchen.
  • Mit Ersetzen wird der aktuelle Treffer ersetzt; Alle ersetzen ersetzt alle Vorkommen auf einmal.
  • Drücken Sie Escape oder klicken Sie auf ✕, um die Leiste zu schließen und alle Hervorhebungen zu entfernen.

Wenn beim Öffnen der Leiste Text im Editor markiert ist, wird das Suchfeld automatisch damit befüllt (nur einzeilige Auswahlen).

Rechtschreibprüfung

MarkForge kann falsch geschriebene Wörter mit einer roten Wellenlinie unterstreichen. Die Rechtschreibprüfung ist standardmäßig deaktiviert.

  • Aktivieren via Ansicht → Rechtschreibprüfung (Umschalter).
  • Wörterbuchsprache wählen via Ansicht → Rechtschreibprüfung Sprache. Verfügbare Sprachen: English, Deutsch, Español, Français, Italiano, Nederlands, Português, Русский, العربية, فارسی.
  • Wörter in Inline-Code-Spans (`…`) und URLs werden nie unterstrichen.
  • GROSSBUCHSTABEN-Abkürzungen (HTML, API, URL…) werden übersprungen.
  • Die Einstellung und die gewählte Sprache werden sitzungsübergreifend gespeichert.
  • Erfordert die Bibliothek pyspellchecker. Falls nicht installiert, wird der Schalter stillschweigend ignoriert.

5. Live-Vorschau

Der rechte Bereich zeigt eine vollständig gerenderte HTML-Version Ihres Markdown-Dokuments, die beim Tippen automatisch aktualisiert wird.

Rendering

Die Vorschau verwendet ein an GitHub angelehntes Stylesheet für eine übersichtliche, ansprechende Darstellung. Sie unterstützt alle Markdown-Erweiterungen: Tabellen, Aufgabenlisten, Fußnoten, Code-Hervorhebung, Formeln und Diagramme. Die Rendering-Pipeline verarbeitet Ihren Text über die Python-Bibliothek markdown mit den Erweiterungen extra, codehilite, toc und sane_lists.

Erhaltung der Scrollposition

Wenn Sie das Dokument bearbeiten, wird die Vorschau nach einer kurzen Verzögerung von 300 ms aktualisiert. Vor dem Neurendern des HTML liest MarkForge die aktuelle Scrollposition aus der Web-Ansicht und stellt sie nach dem Rendering wieder her, sodass die Vorschau nicht nach oben springt, während Sie in der Mitte eines langen Dokuments arbeiten.

Vorschau-Themes

Es stehen zwei Vorschau-Themes zur Verfügung:

  • Dunkel (Standard) — dunkler Hintergrund, passend zum Gesamtdesign der Anwendung.
  • Hell — weißer Hintergrund für ein druckähnliches Leseerlebnis.

Wechseln Sie über Ansicht → Einstellungen → Vorschau-Theme. Die Änderung wird sofort übernommen.

Vorschau ein-/ausblenden

Drücken Sie Ctrl+Shift+P oder verwenden Sie Ansicht → Vorschau anzeigen, um das Vorschau-Panel ein- oder auszublenden. Durch Ausblenden der Vorschau erhält der Editor mehr Platz.

Hinweis

Mathematische Formeln (MathJax) und Mermaid-Diagramme benötigen beim ersten Einsatz eine Internetverbindung, da die Bibliotheken von einem CDN geladen werden. PlantUML-Diagramme werden stets vom Server plantuml.com abgerufen und erfordern ebenfalls eine Internetverbindung.

6. Dateioperationen

Neue Datei

Drücken Sie Ctrl+N oder wählen Sie Datei → Neu. Falls das aktuelle Dokument ungespeicherte Änderungen enthält, fordert MarkForge Sie auf, diese zu speichern. Anschließend wird ein neues, leeres Dokument geöffnet und der Editor aktiviert.

Datei öffnen

Drücken Sie Ctrl+O oder wählen Sie Datei → Öffnen…, um einen Datei-Browser-Dialog zu öffnen. Markdown-Dateien (.md, .markdown), Textdateien (.txt) und alle anderen Dateien können ausgewählt werden. Der Dateibaum aktualisiert sich automatisch und zeigt den Ordner der geöffneten Datei an.

Ordner öffnen

Wählen Sie Datei → Ordner öffnen… (Ctrl+Shift+O), um ein Verzeichnis auszuwählen. Der Dateibaum zeigt dann alle .md- und Bilddateien in diesem Verzeichnis. Klicken Sie auf eine Datei im Baum, um sie im Editor zu öffnen.

Speichern

Drücken Sie Ctrl+S oder wählen Sie Datei → Speichern. Wenn die Datei aus einem Git-Repository geöffnet wurde, erscheint stattdessen der Git-Commit-&-Push-Dialog (siehe Speichern über Git). Bei lokalen Dateien wird die Datei sofort auf der Festplatte gespeichert und das Sternchen in der Titelleiste verschwindet.

Speichern unter

Drücken Sie Ctrl+Shift+S oder wählen Sie Datei → Speichern unter…, um das aktuelle Dokument an einem neuen Speicherort oder unter einem neuen Namen zu speichern.

Dateibaum

Das linke Panel zeigt eine flache Liste der .md-Dateien und Bilder im aktuellen Stammverzeichnis. Unterverzeichnisse ohne direkt relevante Dateien werden ausgeblendet, um die Übersicht zu verbessern. Klicken Sie auf eine Datei, um sie zu öffnen. Die Überschrift des Dateibaums zeigt [GIT] Reponame in Grün, wenn eine Git-verwaltete Datei geöffnet ist.

Tipp

Schalten Sie den Dateibaum mit Ctrl+B um, um dem Editor mehr Platz zu geben. Ihre Einstellung wird sitzungsübergreifend gespeichert.

Zuletzt geöffnet

MarkForge merkt sich die letzten 10 geöffneten Dateien. Zugriff über Datei → Zuletzt geöffnet.

  • Klicken Sie auf einen Eintrag, um die Datei sofort wieder zu öffnen.
  • Existiert eine Datei nicht mehr auf der Festplatte, wird sie automatisch aus der Liste entfernt.
  • Wählen Sie Zuletzt geöffnete löschen am Anfang des Untermenüs, um den gesamten Verlauf zu löschen.
  • Die Liste wird sitzungsübergreifend gespeichert.

8. Mathematische Formeln

MarkForge rendert LaTeX-Mathematiknotation mit MathJax. Beim ersten Einsatz ist eine Internetverbindung erforderlich, da MathJax von einem CDN geladen wird.

Inline-Formel

Schreiben Sie einen Ausdruck zwischen einfache Dollarzeichen, um ihn inline im Fließtext darzustellen:

Die berühmte Gleichung $E = mc^2$ wurde 1905 veröffentlicht.

Ergebnis: Die berühmte Gleichung E = mc² wurde 1905 veröffentlicht.

Block-Formel (zentriert)

Schreiben Sie den Ausdruck zwischen doppelte Dollarzeichen, um ihn zentriert in einer eigenen Zeile darzustellen:

$$\int_0^\infty e^{-x}\, dx = 1$$

Einzeiliger Block-Formel-Modus

Sie können auch ein einzelnes $ in einer eigenen Zeile verwenden, um einen Block-Formel-Bereich zu öffnen und zu schließen:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
Tipp

MathJax unterstützt den vollständigen LaTeX-Mathematikmodus, einschließlich Brüche, Matrizen, griechische Buchstaben, Operatoren und benutzerdefinierte Makros. Die vollständige Syntaxreferenz finden Sie in der MathJax-Dokumentation.

9. Einfügen-Dialoge

MarkForge bietet geführte Dialoge zum Einfügen häufiger Markdown-Konstrukte. Alle Dialoge fügen Text an der aktuellen Cursorposition im Editor ein.

Link einfügen Ctrl+K

  • Anzeigetext — die sichtbare Linkbeschriftung (z. B. Hier klicken).
  • URL — die Zieladresse (z. B. https://beispiel.de).
  • Titel (optional) — ein Tooltip, der beim Überfahren des Links mit der Maus erscheint.

Klicken Sie auf Einfügen, um [Anzeigetext](URL "Titel") in den Editor einzufügen.

Bild einfügen Ctrl+Shift+K

Öffnen über Einfügen → Bild… oder mit Ctrl+Shift+K.

Der Dialog hat zwei Felder:

  • Alt-Text — eine kurze Bildbeschreibung für Screenreader.
  • URL / Pfad — eine externe URL oder ein lokaler Dateipfad. Verwenden Sie die Durchsuchen-Schaltfläche, um eine lokale Bilddatei auszuwählen.

Klicken Sie auf Einfügen, um ![Alt-Text](URL) in den Editor einzufügen.

Öffnen über Einfügen → Tabelle… oder mit Ctrl+Shift+T.

Konfigurieren Sie die Tabellenstruktur:

  • Ausrichtung — wählen Sie Links, Zentriert oder Rechts für jede Spalte separat.

Am unteren Rand des Dialogs wird eine Live-Markdown-Vorschau angezeigt. Klicken Sie auf Einfügen, um die Tabelle in den Editor einzufügen.

10. PDF-Funktionen

Als PDF exportieren Ctrl+Shift+E

Wählen Sie Datei → Als PDF exportieren… oder drücken Sie Ctrl+Shift+E. MarkForge rendert das aktuelle Dokument in eine PDF-Datei, verwendet dabei dieselbe Pipeline wie die Live-Vorschau, jedoch mit weißem Hintergrund und druckoptimiertem CSS. Ein Datei-Speicher-Dialog erscheint, um den Ausgabeort und Dateinamen festzulegen.

Tipp

Wechseln Sie die Vorschau vor dem PDF-Export auf das helle Theme, um die sauberste druckfertige Ausgabe zu erhalten.

PDF importieren Ctrl+Shift+I

Wählen Sie Datei → PDF importieren… oder drücken Sie Ctrl+Shift+I. MarkForge konvertiert beliebige PDF-Dateien mit der Bibliothek pymupdf4llm in Markdown. Die Konvertierung erhält die Dokumentstruktur intelligent:

  • Überschriften und Absätze
  • Tabellen
  • Listen
  • Codeblöcke, sofern erkannt

Die Konvertierung läuft in einem Hintergrundthread, damit die Benutzeroberfläche reaktionsfähig bleibt. Nach dem Abschluss fordert MarkForge Sie auf, einen Speicherort für die resultierende .md-Datei anzugeben, und lädt diese dann im Editor.

Hinweis

Der PDF-Import erfordert das Paket pymupdf4llm. Falls es nicht installiert ist, zeigt MarkForge eine entsprechende Meldung: pip install pymupdf4llm

11. Git-Integration

MarkForge kann Markdown-Dateien direkt aus gehosteten Git-Repositories öffnen, das Bearbeiten ermöglichen und Änderungen zurückpushen — ohne jemals ein Terminal öffnen zu müssen. Erweiterte Operationen wie Amend (vorherigen Commit überschreiben) und Squash (Commits zusammenführen) werden ebenfalls unterstützt.

Unterstützte Plattformen

PlattformBeispiel-URL-Muster
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.meinefirma.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.meinefirma.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

Die Plattform wird automatisch anhand der URL-Struktur erkannt, nicht nur anhand des Hostnamens — Self-Hosted-Instanzen werden dadurch automatisch unterstützt.

Datei aus Git öffnen

  1. Wählen Sie Datei → Aus Git öffnen… oder drücken Sie Ctrl+Shift+G.
  2. Fügen Sie die URL der zu öffnenden Datei ein (kopiert aus dem GitHub- oder Bitbucket-Dateibrowser).
  3. MarkForge validiert die URL und zeigt eine Vorschauzeile: Plattform · owner/repo · pfad/zur/datei.md.
  4. Optional: Überschreiben Sie Branch, Tag oder Commit-Hash im Ref-Feld.
  5. Klicken Sie auf OK — das Repository wird im Hintergrund geklont und die Datei im Editor geöffnet.

Während eine Git-verwaltete Datei geöffnet ist, zeigt die Dateibaum-Überschrift [GIT] Reponame in Grün.

Tipp

Öffentliche Repositories funktionieren ohne konfigurierte Anmeldedaten. Für private Repositories konfigurieren Sie Ihre Zugangsdaten zuerst unter Ansicht → Einstellungen → Git-Authentifizierung.

Speichern (Commit & Push)

Wenn eine Git-verwaltete Datei geöffnet ist, öffnet Ctrl+S den Dialog Git Commit & Push statt des normalen Speicherns. Der Dialog enthält:

  • Commit-Nachricht — erforderlich. Geben Sie eine aussagekräftige Beschreibung Ihrer Änderungen ein.
  • Vorherigen Commit überschreiben — ab dem zweiten Speichern verfügbar, wenn SSH- oder git-binary-Authentifizierung verwendet wird. Ersetzt den vorherigen Commit anstatt einen neuen zu erstellen; füllt die vorherige Commit-Nachricht vor; verwendet automatisch Force-Push (--force-with-lease). Nur Commits, die MarkForge in der aktuellen Sitzung erstellt hat, können überschrieben werden.
  • Pushen nach:
    • Aktueller Branch — pusht direkt in den Branch, von dem geklont wurde.
    • Neuer Branch — erstellt einen neuen Branch mit einem von Ihnen angegebenen Namen.
  • Pull Request erstellen — verfügbar, wenn „Neuer Branch" ausgewählt ist. Öffnet nach dem Push automatisch einen Pull Request. Zeigt Felder für PR-Titel und Zielbranch.

Commits zusammenführen Ctrl+Shift+Q

Nur mit SSH- und HTTPS (git-Programm)-Authentifizierung verfügbar.

Wählen Sie Datei → Git Squash… oder drücken Sie Ctrl+Shift+Q. Der Squash-Dialog zeigt eine Tabelle aller Commits auf dem aktuellen Branch, die noch nicht im ausgewählten Basis-Branch enthalten sind.

  • Basis-Branch-Feld — der Referenz-Branch (z. B. main, master, develop). Klicken Sie auf Neu laden, nachdem Sie ihn geändert haben.
  • Alle neuen Commits auswählen — markiert alle aufgelisteten Commits auf einmal.
  • Zusammengeführte Commit-Nachricht — die einzelne Nachricht, die alle ausgewählten Commits ersetzt.
  • Zusammenführen & Pushen — schreibt die Branch-History neu und pusht mit Force zum Remote.

Es müssen mindestens 2 Commits ausgewählt werden, und diese müssen eine zusammenhängende Reihe ab dem neuesten Commit (HEAD) bilden.

Authentifizierungsmodi

Konfigurieren Sie Ihre Authentifizierungsmethode unter Ansicht → Einstellungen → Git-Authentifizierung.

HTTPS (eingebettet)

Funktioniert auf allen Plattformen ohne zusätzliche Software. Verwendet direkt die Plattform-REST-API. Kein lokales git-Binary erforderlich.

Einschränkung

Die Funktionen „Vorherigen Commit überschreiben" und „Commits zusammenführen" sind in diesem Modus nicht verfügbar, da keine vollständige lokale Git-History gepflegt wird.

PlattformToken-Typ
GitHubPersonal access token (ghp_…) — benötigt repo-Scope
GitHub EnterprisePersonal access token — wie GitHub
Bitbucket CloudApp-Passwort (Konto → App-Passwörter)
Bitbucket ServerPersonal access token oder HTTP access token

HTTPS (git-Programm)

Verwendet das systemseitige git-Programm für alle Operationen — vollständiger lokaler Klon, Commit und Push. Erfordert Git für Windows (oder ein beliebiges git-Binary im PATH). Unterstützt Amend und Squash. Verwenden Sie dieselben Benutzername- und Token-Felder wie beim HTTPS (eingebettet)-Modus. Sie müssen außerdem Ihren Namen und Ihre E-Mail für die Commit-Identität angeben.

SSH (Schlüsseldatei)

Wählen Sie SSH (Schlüsseldatei) und geben Sie den Pfad zu Ihrem privaten SSH-Schlüssel und optional eine Passphrase an. Dieser Modus verwendet paramiko für eine rein Python-basierte SSH-Verbindung (kein System-SSH-Client erforderlich). Wenn paramiko nicht installiert ist, fällt MarkForge auf den System-ssh-Client zurück.

pip install paramiko

SSH-Authentifizierung unterstützt Amend und Squash.

Speicherung von Anmeldedaten

Tokens, SSH-Passphrasen und Proxy-Passwörter werden im OS-Schlüsselbund gespeichert (GNOME Keyring / KWallet unter Linux, Keychain unter macOS, Windows Credential Manager unter Windows) — nicht in Klartextkonfigurationsdateien.

HTTP-Proxy

Falls Ihr Netzwerk einen HTTP-Proxy benötigt, um Git-Server zu erreichen, konfigurieren Sie ihn unter Ansicht → Einstellungen → HTTP-Proxy:

  • Proxy-URL — z. B. http://proxy.firma.de:8080
  • Proxy-Benutzername — leer lassen, wenn keine Authentifizierung erforderlich ist.
  • Proxy-Passwort — leer lassen, wenn keine Authentifizierung erforderlich ist.

Lassen Sie alle Proxy-Felder leer, um die Standard-Proxy-Einstellungen des Systems zu verwenden.

Schliessen

Wenn Sie MarkForge schliessen, während eine Git-verwaltete Datei geöffnet ist, fragt MarkForge, ob das temporäre Klon-Verzeichnis gelöscht werden soll. Das Löschen des Verzeichnisses gibt Speicherplatz frei; das Behalten ermöglicht es Ihnen, die geklonten Dateien anschließend zu inspizieren.
