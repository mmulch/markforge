Hướng dẫn sử dụng MarkForge v1.8.0

1. Giới thiệu

MarkForge là trình soạn thảo Markdown hiện đại, giàu tính năng, được xây dựng bằng Python và PyQt6. Ứng dụng kết hợp trải nghiệm viết không phân tâm với tính năng xem trước song song theo thời gian thực, tích hợp đầy đủ quy trình làm việc với Git, hỗ trợ sơ đồ, hiển thị công thức toán học, cũng như nhập/xuất PDF — tất cả trong một ứng dụng desktop hai ngăn gọn gàng.

Dù bạn đang viết tài liệu kỹ thuật, tệp README cho dự án, hay xuất bản nội dung trực tiếp lên kho lưu trữ GitHub, MarkForge xử lý toàn bộ vòng đời soạn thảo và xuất bản mà không cần mở terminal.

2. Cài đặt

Linux (Debian / Ubuntu)

Tải gói .deb từ trang Releases trên GitHub, sau đó cài đặt bằng lệnh:

sudo apt install ./markforge_*.deb

Gói sẽ tự động cài đặt tất cả các phụ thuộc cần thiết ở cấp hệ thống (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments). Không cần môi trường ảo.

Windows

Tải MarkForge-Setup.exe từ trang Releases và chạy trình cài đặt. Trình hướng dẫn cài đặt sẽ dẫn bạn qua các bước và tự động tạo các lối tắt trong menu Start.

Lưu ý

Trình cài đặt Windows đã bao gồm sẵn Python runtime và tất cả các phụ thuộc. Không cần cài đặt Python riêng.

Từ mã nguồn

Yêu cầu: Python 3.8 trở lên, CMake 3.16 trở lên.

Sử dụng CMake (khuyến nghị):

# 1. Clone the repository
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Configure the build (creates a local virtual environment)
cmake -B build

# 3. Install Python dependencies into the local venv
cmake --build build --target install_deps

# 4. Run the application
cmake --build build --target run

Khởi chạy Python trực tiếp (không cần CMake):

pip install -r requirements.txt
python src/main.py
Mẹo

Để sử dụng các thao tác Git qua SSH, hãy cài thêm paramiko: pip install paramiko

3. Tổng quan giao diện

Bố cục

Cửa sổ MarkForge được chia thành ba ngăn có thể thay đổi kích thước, ngăn cách nhau bằng các thanh kéo:

Kéo bất kỳ thanh phân cách nào để thay đổi kích thước ngăn. Kích thước ngăn được ghi nhớ giữa các phiên làm việc.

Dàn ý tài liệu

Bảng Dàn ý tài liệu hiển thị tất cả các tiêu đề (H1–H6) dưới dạng cây có thể điều hướng trong thanh bên trái.

  • Nhấp vào bất kỳ tiêu đề nào để đưa con trỏ trình soạn thảo đến dòng đó.
  • Dàn ý tự động cập nhật khi bạn gõ.
  • Bật/tắt hiển thị qua Xem → Hiển thị dàn ý. Tùy chọn được lưu giữa các phiên làm việc.

Thanh công cụ

Thanh công cụ bên dưới thanh menu cung cấp truy cập một lần nhấp vào các thao tác phổ biến nhất: Mới, Mở, Lưu, Chèn liên kết, Chèn hình ảnh, Chèn bảng, Chèn PlantUML, Chèn Mermaid, Bật/tắt xem trước và Cài đặt. Tất cả hành động thanh công cụ đều trùng với các mục menu và phím tắt bàn phím.

Thanh trạng thái

Thanh trạng thái ở dưới cùng cửa sổ luôn hiển thị:

  • Số từ & thời gian đọc — số từ và thời gian đọc ước tính (ví dụ: 342 từ · 2 phút đọc), cập nhật khi bạn gõ.
  • Tiến độ mục tiêu từ — nếu đặt số từ mục tiêu qua Xem → Đặt Mục Tiêu Từ, thanh tiến độ nhỏ gọn sẽ xuất hiện.
  • Vị trí con trỏ — số dòng và cột hiện tại, ví dụ: Dòng 12, Cột 5.

Tiêu đề cửa sổ

Tiêu đề cửa sổ hiển thị tên tệp đang mở. Khi bạn thực hiện thay đổi chưa được lưu, một dấu hoa thị (*) sẽ được thêm vào sau tên tệp, ví dụ README.md *. Khi không có tệp nào đang mở, tiêu đề hiển thị Untitled.

4. Trình soạn thảo

Ngăn giữa chứa trình soạn thảo văn bản thuần đầy đủ tính năng, được tối ưu hóa cho việc viết Markdown.

Tô sáng cú pháp

MarkForge áp dụng tô sáng cú pháp nhận biết Markdown cho tài liệu khi bạn gõ. Các tiêu đề, ký hiệu nhấn mạnh, liên kết, đoạn mã nội tuyến, khối mã có hàng rào và các cấu trúc Markdown khác được tô màu riêng biệt. Tô sáng tự động thích nghi khi bạn chuyển đổi giữa giao diện trình soạn thảo Sáng và Tối.

Số dòng

Một lề bên trái của trình soạn thảo hiển thị số dòng. Dòng hiện tại được tô sáng bằng màu nền tinh tế để giúp bạn theo dõi vị trí trong các tài liệu dài.

Xuống dòng tự động

Xuống dòng tự động được bật theo mặc định. Các dòng dài sẽ xuống dòng vừa với chiều rộng hiển thị của trình soạn thảo. Bật/tắt xuống dòng tự động bất cứ lúc nào qua Xem → Xuống dòng tự động. Tùy chọn này được lưu giữa các phiên làm việc.

Giao diện trình soạn thảo

Có hai giao diện trình soạn thảo tích hợp sẵn:

  • Tối (mặc định) — nền tối với màu cú pháp tương phản cao.
  • Sáng — nền trắng phù hợp với môi trường sáng.

Chuyển đổi giao diện trong Xem → Cài đặt → Giao diện trình soạn thảo. Thay đổi có hiệu lực ngay lập tức mà không cần khởi động lại.

Văn bản giữ chỗ

Khi không có tệp nào đang mở, trình soạn thảo bị tắt và hiển thị thông báo: "Open or create a Markdown file to start editing." Trình soạn thảo sẽ kích hoạt ngay khi bạn tạo tệp mới hoặc mở tệp hiện có.

Các thao tác chỉnh sửa

Tất cả các thao tác chỉnh sửa văn bản tiêu chuẩn đều khả dụng:

  • Ctrl+Z — Hoàn tác
  • Ctrl+Y — Làm lại
  • Ctrl+X / Ctrl+C / Ctrl+V — Cắt / Sao chép / Dán

Tìm & Thay thế

Nhấn Ctrl+F để mở thanh tìm kiếm, hoặc Ctrl+H để mở cùng với hàng thay thế. Thanh xuất hiện phía trên trình soạn thảo.

  • Nhập vào ô tìm kiếm — tất cả kết quả khớp được tô sáng ngay lập tức trong trình soạn thảo.
  • Điều hướng giữa các kết quả bằng nút ↑ / ↓, hoặc nhấn Enter (tiếp theo) / Shift+Enter (trước đó) trong ô tìm kiếm.
  • Bộ đếm kết quả hiển thị vị trí hiện tại (ví dụ 3 / 12).
  • Aa — bật/tắt tìm kiếm phân biệt chữ hoa/thường.
  • [W] — bật/tắt khớp toàn bộ từ.
  • Dùng Thay thế để thay thế kết quả hiện tại, hoặc Thay thế tất cả để thay thế mọi lần xuất hiện cùng lúc.
  • Nhấn Escape hoặc nhấp ✕ để đóng thanh và xóa tất cả tô sáng.

Nếu có văn bản được chọn trong trình soạn thảo khi mở thanh, ô tìm kiếm sẽ được điền sẵn với nội dung đó (chỉ áp dụng cho lựa chọn một dòng).

Kiểm tra chính tả

MarkForge có thể gạch chân các từ sai chính tả bằng đường gợn sóng màu đỏ khi bạn gõ. Kiểm tra chính tả bị tắt theo mặc định.

  • Bật/tắt qua Xem → Kiểm tra chính tả (công tắc).
  • Chọn ngôn ngữ từ điển qua Xem → Ngôn ngữ kiểm tra chính tả. Ngôn ngữ có sẵn: English, Deutsch, Español, Français, Italiano, Nederlands, Português, العربية, فارسی (qua pyspellchecker) và Tiếng Việt*, Svenska*, Українська*, ಕನ್ನಡ*, हिंदी* (qua enchant/Hunspell).
  • Các từ bên trong đoạn mã nội tuyến (`…`) và URL không bao giờ bị gạch chân.
  • Các từ viết tắt VIẾT HOA (HTML, API, URL…) được bỏ qua.
  • Tùy chọn kiểm tra chính tả và ngôn ngữ được lưu giữa các phiên làm việc.
  • Yêu cầu thư viện pyspellchecker. Nếu chưa cài đặt, công tắc sẽ bị bỏ qua mà không có thông báo.
  • Các ngôn ngữ có dấu * yêu cầu thư viện pyenchant và từ điển Hunspell của hệ thống (ví dụ: hunspell-vi trên Debian/Ubuntu).

5. Xem trước trực tiếp

Ngăn bên phải hiển thị phiên bản HTML được kết xuất đầy đủ của tài liệu Markdown, tự động cập nhật khi bạn gõ.

Kết xuất

Phần xem trước sử dụng bảng kiểu lấy cảm hứng từ GitHub để có đầu ra rõ ràng, dễ đọc. Nó hỗ trợ tất cả các phần mở rộng Markdown bao gồm bảng, danh sách tác vụ, chú thích cuối trang, tô sáng mã, toán học và sơ đồ. Quy trình kết xuất xử lý văn bản của bạn qua thư viện Python markdown với các phần mở rộng extra, codehilite, toc và sane_lists.

Giữ nguyên vị trí cuộn

Khi bạn chỉnh sửa tài liệu, bản xem trước cập nhật sau độ trễ debounce ngắn 300 mili giây. Trước khi làm mới HTML, MarkForge đọc vị trí cuộn hiện tại từ chế độ xem web và khôi phục nó sau khi kết xuất, vì vậy bản xem trước không nhảy lên đầu trong khi bạn đang làm việc ở giữa tài liệu.

Giao diện xem trước

Có hai giao diện xem trước:

  • Tối (mặc định) — nền tối phù hợp với giao diện tổng thể của ứng dụng.
  • Sáng — nền trắng cho trải nghiệm đọc như in ấn.

Chuyển đổi qua Xem → Cài đặt → Giao diện xem trước. Thay đổi có hiệu lực ngay lập tức.

Bật/tắt xem trước

Nhấn Ctrl+Shift+P hoặc dùng Xem → Hiển thị xem trước để ẩn hoặc hiện ngăn xem trước. Ẩn xem trước sẽ cho trình soạn thảo nhiều không gian màn hình hơn.

Lưu ý

Công thức toán học (MathJax) và sơ đồ Mermaid yêu cầu kết nối internet khi sử dụng lần đầu vì các thư viện được tải từ CDN. Sơ đồ PlantUML luôn được tải từ plantuml.com và yêu cầu truy cập internet.

6. Thao tác với tệp

Tệp mới

Nhấn Ctrl+N hoặc chọn Tệp → Mới. Nếu tài liệu hiện tại có thay đổi chưa lưu, MarkForge sẽ nhắc bạn lưu trước khi tiếp tục. Một tài liệu trống mới sẽ được mở và trình soạn thảo được kích hoạt.

Mở tệp

Nhấn Ctrl+O hoặc chọn Tệp → Mở… để mở hộp thoại trình duyệt tệp. Các tệp Markdown (.md, .markdown), tệp văn bản (.txt) và tất cả các tệp khác đều có thể chọn. Cây tệp sẽ cập nhật để hiển thị thư mục chứa tệp đã mở.

Mở thư mục

Chọn Tệp → Mở thư mục… (Ctrl+Shift+O) để chọn một thư mục. Cây tệp sau đó hiển thị tất cả các tệp .md và hình ảnh trong thư mục đó. Nhấp vào bất kỳ tệp nào trong cây để mở nó trong trình soạn thảo.

Lưu

Nhấn Ctrl+S hoặc chọn Tệp → Lưu. Nếu tệp được mở từ kho lưu trữ Git, hộp thoại commit & push Git sẽ xuất hiện thay thế (xem Lưu qua Git). Đối với các tệp cục bộ, tệp được ghi ngay lập tức vào đĩa và dấu hoa thị trong thanh tiêu đề biến mất.

Lưu thành

Nhấn Ctrl+Shift+S hoặc chọn Tệp → Lưu thành… để lưu tài liệu hiện tại vào vị trí mới hoặc với tên mới.

Cây tệp

Ngăn bên trái hiển thị danh sách phẳng các tệp .md và hình ảnh trong thư mục gốc hiện tại. Các thư mục con không chứa tệp liên quan trực tiếp sẽ bị ẩn để giảm lộn xộn. Nhấp vào tệp để mở nó. Tiêu đề cây hiển thị [GIT] reponame bằng màu xanh lá khi tệp được quản lý bởi Git đang mở.

Mẹo

Bật/tắt cây tệp bằng Ctrl+B để tối đa hóa không gian trình soạn thảo. Tùy chọn của bạn được lưu giữa các phiên làm việc.

Tệp gần đây

MarkForge lưu danh sách 10 tệp được mở gần đây nhất. Truy cập qua Tệp → Tệp gần đây.

  • Nhấp vào bất kỳ mục nào để mở lại tệp ngay lập tức.
  • Nếu tệp không còn tồn tại trên đĩa, nó sẽ tự động bị xóa khỏi danh sách.
  • Chọn Xóa tệp gần đây ở đầu menu con để xóa toàn bộ lịch sử.
  • Danh sách được lưu giữa các phiên làm việc.

7. Tài liệu tham khảo Markdown

MarkForge hỗ trợ Markdown CommonMark tiêu chuẩn cộng với các phần mở rộng sau: extra (bảng, chú thích cuối trang, viết tắt), codehilite (khối mã có tô sáng cú pháp), toc (mục lục), sane_lists và gạch ngang.

Tiêu đề

Markdown
# Heading 1 ## Heading 2 ### Heading 3 #### Heading 4 ##### Heading 5 ###### Heading 6
Kết quả

Heading 1

Heading 2

Heading 3

Nhấn mạnh & Định dạng nội tuyến

Markdown
**bold text** *italic text* ~~strikethrough~~ `inline code` **_bold italic_**
Kết quả
bold text
italic text
strikethrough
inline code
bold italic

Danh sách

Danh sách không có thứ tự:

Markdown
- First item - Second item - Nested item - Third item
Kết quả
  • First item
  • Second item
    • Nested item
  • Third item

Danh sách có thứ tự:

8. Công thức toán học

MarkForge kết xuất ký hiệu toán LaTeX bằng MathJax. Yêu cầu kết nối internet khi sử dụng lần đầu (MathJax được tải từ CDN).

Toán học nội tuyến

Bao một biểu thức trong dấu đô la đơn để kết xuất nó nội tuyến trong một đoạn văn:

The famous equation $E = mc^2$ was published in 1905.

Kết quả: Phương trình nổi tiếng E = mc² được công bố năm 1905.

Toán học hiển thị (khối)

Bao biểu thức trong dấu đô la kép để kết xuất nó căn giữa trên dòng riêng:

$$\int_0^\infty e^{-x}\, dx = 1$$

Toán học khối một dòng

Bạn cũng có thể dùng ký hiệu $ đơn trên dòng riêng để mở và đóng một khối toán hiển thị:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
Mẹo

MathJax hỗ trợ toàn bộ chế độ toán LaTeX, bao gồm phân số, ma trận, chữ cái Hy Lạp, toán tử và macro tùy chỉnh. Tham khảo tài liệu MathJax để biết cú pháp đầy đủ.

9. Hộp thoại chèn

MarkForge cung cấp các hộp thoại có hướng dẫn để chèn các cấu trúc Markdown phổ biến. Tất cả hộp thoại đều chèn văn bản tại vị trí con trỏ hiện tại trong trình soạn thảo.

Chèn liên kết Ctrl+K

Mở qua Chèn → Liên kết… hoặc nhấn Ctrl+K.

Hộp thoại có ba trường:

  • Văn bản hiển thị — nhãn liên kết hiển thị (ví dụ Nhấp vào đây).
  • URL — địa chỉ đích (ví dụ https://example.com).
  • Tiêu đề (tùy chọn) — tooltip hiển thị khi người dùng di chuột qua liên kết.

Nhấp Chèn để đặt [Văn bản hiển thị](URL "Tiêu đề") trong trình soạn thảo.

Chèn hình ảnh Ctrl+Shift+K

Mở qua Chèn → Hình ảnh… hoặc nhấn Ctrl+Shift+K.

Hộp thoại có hai trường:

  • Văn bản thay thế — mô tả ngắn gọn cho trình đọc màn hình.
  • URL / Đường dẫn — URL từ xa hoặc đường dẫn tệp cục bộ. Dùng nút duyệt để chọn tệp hình ảnh cục bộ.

Nhấp Chèn để đặt ![Văn bản thay thế](URL) trong trình soạn thảo.

Chèn bảng Ctrl+Shift+T

Mở qua Chèn → Bảng… hoặc nhấn Ctrl+Shift+T.

Cấu hình cấu trúc bảng của bạn:

  • Hàng dữ liệu — số hàng không tính hàng tiêu đề.
  • Cột — số cột (tối đa được hỗ trợ).
  • Tên cột — nhập tiêu đề cho mỗi cột.
  • Căn chỉnh — chọn Trái, Giữa hoặc Phải cho từng cột độc lập.

Bản xem trước Markdown trực tiếp cập nhật ở cuối hộp thoại. Nhấp Chèn để đặt bảng trong trình soạn thảo.

Chèn PlantUML Ctrl+Shift+U

Mở qua Chèn → PlantUML… hoặc nhấn Ctrl+Shift+U.

Chọn từ 12 mẫu sơ đồ trong danh sách thả xuống:

  • Sơ đồ tuần tự
  • Sơ đồ lớp
  • Sơ đồ hoạt động
  • Sơ đồ trạng thái
  • Sơ đồ thành phần
  • Sơ đồ ca sử dụng
  • Sơ đồ Gantt
  • Bản đồ tư duy
  • WBS (Phân rã công việc)
  • Sơ đồ thời gian
  • Sơ đồ triển khai
  • JSON

Mẫu đã chọn điền vào trình soạn thảo mã PlantUML. Chỉnh sửa mã theo nhu cầu, sau đó nhấp Chèn. Sơ đồ được bao trong khối mã có hàng rào và được kết xuất qua dịch vụ trực tuyến plantuml.com trong phần xem trước. Yêu cầu kết nối internet.

Lưu ý

Bản xem trước sơ đồ tự động xuất hiện trong cửa sổ chính sau khi chèn. Quá trình kết xuất sử dụng máy chủ công cộng plantuml.com; không cần cài đặt PlantUML cục bộ.

Chèn Mermaid Ctrl+Shift+M

Mở qua Chèn → Mermaid… hoặc nhấn Ctrl+Shift+M.

Chọn từ 9 mẫu sơ đồ:

  • Lưu đồ
  • Sơ đồ tuần tự
  • Sơ đồ lớp
  • Sơ đồ trạng thái
  • Gantt
  • Biểu đồ tròn
  • Bản đồ tư duy
  • Sơ đồ ER
  • Đồ thị Git

Chỉnh sửa mã Mermaid trong vùng văn bản và nhấp Chèn. Sơ đồ được bao trong khối mã mermaid có hàng rào và được kết xuất trong phần xem trước qua kroki.io. Yêu cầu kết nối internet.

10. Tính năng PDF

Xuất dạng PDF Ctrl+Shift+E

Chọn Tệp → Xuất dạng PDF… hoặc nhấn Ctrl+Shift+E. MarkForge kết xuất tài liệu hiện tại thành tệp PDF bằng cùng quy trình như bản xem trước trực tiếp, nhưng với nền trắng và CSS tối ưu hóa cho in ấn. Hộp thoại lưu tệp xuất hiện để bạn có thể chọn vị trí đầu ra và tên tệp.

Mẹo

Chuyển xem trước sang giao diện Sáng trước khi xuất PDF để có đầu ra sẵn sàng in đẹp nhất.

Xuất sang HTML

Chọn Tệp → Xuất sang HTML…. MarkForge kết xuất tài liệu hiện tại thành tệp HTML độc lập với toàn bộ CSS nhúng nội tuyến và hình ảnh cục bộ được chuyển đổi thành URI dữ liệu Base64, giúp mở tệp trong bất kỳ trình duyệt nào mà không cần kết nối internet.

Xuất qua Pandoc (DOCX, EPUB, LaTeX)

Chọn Tệp → Xuất dạng DOCX (Pandoc)…, Xuất dạng EPUB (Pandoc)…, hoặc Xuất dạng LaTeX (Pandoc)…. Các tùy chọn này yêu cầu pandoc được cài đặt và có trên PATH. Nếu pandoc không được tìm thấy, hộp thoại cảnh báo sẽ hiển thị.

Nhập PDF Ctrl+Shift+I

Chọn Tệp → Nhập PDF… hoặc nhấn Ctrl+Shift+I. MarkForge chuyển đổi bất kỳ tệp PDF nào sang Markdown bằng thư viện pymupdf4llm. Quá trình chuyển đổi thông minh bảo toàn cấu trúc tài liệu:

  • Tiêu đề và đoạn văn
  • Bảng
  • Danh sách
  • Khối mã khi được phát hiện

Quá trình chuyển đổi chạy trong một luồng nền để giao diện vẫn phản hồi. Khi hoàn tất, MarkForge nhắc bạn chọn vị trí lưu cho tệp .md kết quả và sau đó tải nó trong trình soạn thảo.

Lưu ý

Nhập PDF yêu cầu gói pymupdf4llm. Nếu chưa được cài đặt, MarkForge sẽ nhắc bạn cài đặt: pip install pymupdf4llm

11. Tích hợp Git

MarkForge có thể mở tệp Markdown trực tiếp từ các kho lưu trữ Git được lưu trữ, cho phép bạn chỉnh sửa chúng và đẩy các thay đổi trở lại — mà không cần mở terminal. Các thao tác nâng cao như amend và squash cũng được hỗ trợ.

Nền tảng được hỗ trợ

Nền tảngVí dụ mẫu URL
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.mycompany.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.mycompany.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

Nền tảng được tự động phát hiện từ cấu trúc URL, không chỉ từ tên máy chủ — vì vậy các phiên bản tự lưu trữ hoạt động tự động.

Mở tệp từ Git

  1. Chọn Tệp → Mở từ Git… hoặc nhấn Ctrl+Shift+G.
  2. Dán URL của tệp bạn muốn mở (sao chép từ trình duyệt tệp GitHub hoặc Bitbucket).
  3. MarkForge xác nhận URL và hiển thị dòng xem trước: nền tảng · chủ sở hữu/repo · đường dẫn/tới/tệp.md.
  4. Tùy chọn ghi đè nhánh, thẻ hoặc mã hash commit trong trường Ref.
  5. Nhấp OK — kho lưu trữ được sao chép trong nền và tệp mở trong trình soạn thảo.

Trong khi tệp được quản lý bởi Git đang mở, tiêu đề cây tệp hiển thị [GIT] reponame bằng màu xanh lá.

Mẹo

Các kho lưu trữ công khai hoạt động mà không cần cấu hình thông tin đăng nhập. Đối với kho lưu trữ riêng tư, hãy cấu hình thông tin đăng nhập của bạn trong Xem → Cài đặt → Xác thực Git trước khi mở.

Lưu (Commit & Push)

Khi tệp được quản lý bởi Git đang mở, nhấn Ctrl+S sẽ mở hộp thoại Git Commit & Push thay vì lưu tệp thông thường. Hộp thoại chứa:

  • Thông điệp commit — bắt buộc. Nhập mô tả có ý nghĩa về các thay đổi của bạn.
  • Sửa đổi commit trước — có sẵn từ lần lưu thứ hai trở đi khi sử dụng xác thực SSH hoặc git-binary. Thay thế commit trước đó thay vì tạo mới; điền trước thông điệp commit trước đó; tự động sử dụng force-push (--force-with-lease). Chỉ có thể sửa đổi các commit được tạo bởi MarkForge trong phiên hiện tại.
  • Push lên:
    • Nhánh hiện tại — push trực tiếp lên nhánh bạn đã sao chép từ.
    • Nhánh mới — tạo nhánh mới với tên bạn chỉ định.
  • Tạo Pull Request — có sẵn khi "Nhánh mới" được chọn. Tự động mở pull request sau khi push. Hiển thị các trường cho tiêu đề PR và nhánh đích.

Squash Commit Ctrl+Shift+Q

Chỉ khả dụng với xác thực SSH và HTTPS (git binary).

Chọn Tệp → Git Squash… hoặc nhấn Ctrl+Shift+Q. Hộp thoại Squash Commit hiển thị bảng tất cả các commit trên nhánh hiện tại chưa có trong nhánh cơ sở đã chọn.

  • Trường nhánh cơ sở — nhánh tham chiếu (ví dụ main, master, develop). Nhấp Tải lại sau khi thay đổi.
  • Chọn tất cả commit mới — kiểm tra tất cả các commit được liệt kê cùng một lúc.
  • Thông điệp commit squash — thông điệp duy nhất sẽ thay thế tất cả các commit đã chọn.
  • Squash & Push — viết lại lịch sử nhánh và force-push lên remote.

Bạn phải chọn ít nhất 2 commit và chúng phải tạo thành một dải liên tục bắt đầu từ commit gần nhất (HEAD).

Chế độ xác thực

Cấu hình phương thức xác thực của bạn trong Xem → Cài đặt → Xác thực Git.

HTTPS (nhúng sẵn)

Hoạt động trên tất cả các nền tảng mà không cần thêm phần mềm. Sử dụng trực tiếp API REST của nền tảng. Không yêu cầu git binary cục bộ.

Hạn chế

Các tính năng amend và squash không có sẵn trong chế độ này vì không có lịch sử git cục bộ đầy đủ được duy trì.

Nền tảngLoại token
GitHubToken truy cập cá nhân (ghp_…) — yêu cầu quyền repo
GitHub EnterpriseToken truy cập cá nhân — giống GitHub
Bitbucket CloudMật khẩu ứng dụng (Tài khoản → Mật khẩu ứng dụng)
Bitbucket ServerToken truy cập cá nhân hoặc token truy cập HTTP

HTTPS (git binary)

Sử dụng tệp thực thi git của hệ thống cho tất cả các thao tác — sao chép cục bộ đầy đủ, commit và push. Yêu cầu Git for Windows (hoặc bất kỳ tệp nhị phân git nào trong PATH của bạn). Hỗ trợ amend và squash. Sử dụng cùng các trường tên người dùng và token như chế độ HTTPS (nhúng sẵn). Bạn cũng phải cung cấp Tên và Email cho danh tính commit.

SSH (tệp khóa)

Chọn SSH (tệp khóa) và cung cấp đường dẫn đến khóa SSH riêng tư và cụm mật khẩu tùy chọn. Chế độ này sử dụng paramiko cho kết nối SSH thuần Python (không yêu cầu client ssh hệ thống). Nếu paramiko chưa được cài đặt, MarkForge sẽ dự phòng sang client ssh hệ thống.

pip install paramiko

Xác thực SSH hỗ trợ amend và squash.

Lưu trữ thông tin xác thực

Token, cụm mật khẩu SSH và mật khẩu proxy được lưu trong kho khóa của hệ điều hành (GNOME Keyring / KWallet trên Linux, Keychain trên macOS, Windows Credential Manager trên Windows) — không lưu trong các tệp cấu hình dạng văn bản thuần túy.

HTTP Proxy

Nếu mạng của bạn yêu cầu HTTP proxy để đến các máy chủ Git, hãy cấu hình trong Xem → Cài đặt → HTTP Proxy:

  • URL Proxy — ví dụ http://proxy.company.com:8080
  • Tên người dùng proxy — để trống nếu không yêu cầu xác thực.
  • Mật khẩu proxy — để trống nếu không yêu cầu xác thực.

Để trống tất cả các trường proxy để sử dụng cài đặt proxy mặc định của hệ thống.

Đóng

Khi bạn đóng MarkForge trong khi tệp được quản lý bởi Git đang mở, MarkForge sẽ hỏi có xóa thư mục sao chép tạm thời không. Xóa thư mục sẽ giải phóng dung lượng đĩa; giữ lại sẽ cho phép bạn kiểm tra các tệp đã sao chép sau đó.
