دليل مستخدم MarkForge v1.6.1

١. المقدمة

MarkForge محرر Markdown حديث وغني بالميزات، مبني بلغة Python وإطار PyQt6. يجمع بين تجربة كتابة خالية من المشتتات ومعاينة جانبية في الوقت الفعلي، مع تكامل كامل مع سير عمل Git، ودعم للرسوم البيانية، وعرض الصيغ الرياضية، واستيراد/تصدير PDF — كل ذلك في تطبيق سطح مكتب نظيف ذي لوحتين.

سواء كنت تكتب توثيقاً تقنياً، أو ملفات README للمشاريع، أو تنشر محتوى مباشرة إلى مستودع GitHub، يتولى MarkForge دورة الحياة الكاملة للتحرير والنشر دون الحاجة إلى فتح نافذة طرفية.

معاينة حيةعرض HTML بأسلوب GitHub جنباً إلى جنب أثناء الكتابة
تمييز الصيغةنظام ألوان يدرك صيغة Markdown، بوضعَي الداكن والفاتح
تكامل Gitفتح وتحرير وإجراء commit وpush إلى GitHub أو Bitbucket
الرسوم البيانيةPlantUML (١٢ قالباً) وMermaid (٩ قوالب)
الصيغ الرياضيةترميز LaTeX يُعرض باستخدام MathJax
استيراد PDFتحويل أي PDF إلى Markdown تلقائياً
تصدير PDFPDF محسَّن للطباعة من أي مستند
متعدد اللغاتواجهة مستخدم بالإنجليزية والألمانية (Deutsch)

٢. التثبيت

Linux (Debian / Ubuntu)

نزِّل حزمة .deb من صفحة Releases على GitHub، ثم ثبِّتها بالأمر:

sudo apt install ./markforge_*.deb

تقوم الحزمة تلقائياً بتثبيت جميع التبعيات المطلوبة على مستوى النظام (python3-pyqt6، python3-pyqt6.qtwebengine، python3-markdown، python3-pygments). لا حاجة إلى بيئة افتراضية.

Windows

نزِّل MarkForge-Setup.exe من صفحة Releases وشغِّل المثبِّت. يرشدك معالج الإعداد خلال عملية التثبيت وينشئ اختصارات قائمة البدء تلقائياً.

ملاحظة

يتضمن مثبِّت Windows بيئة تشغيل Python وجميع التبعيات. لا حاجة إلى تثبيت Python بشكل منفصل.

من المصدر

المتطلبات: Python 3.8 أو أحدث، CMake 3.16 أو أحدث.

باستخدام CMake (موصى به):

# 1. Clone the repository
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Configure the build (creates a local virtual environment)
cmake -B build

# 3. Install Python dependencies into the local venv
cmake --build build --target install_deps

# 4. Run the application
cmake --build build --target run

تشغيل Python المباشر (بدون CMake):

pip install -r requirements.txt
python src/main.py
نصيحة

لعمليات Git عبر SSH، ثبِّت أيضاً paramiko: pip install paramiko

٣. نظرة عامة على الواجهة

التخطيط

تنقسم نافذة MarkForge إلى ثلاث لوحات قابلة للتغيير الحجم مفصولة بفواصل قابلة للسحب:

اللوحةالموضعالوصف
المخطط التفصيلي + شجرة الملفاتاليساريحتوي اللوح الأيسر على قسمين متراكبين: مخطط المستند (شجرة العناوين) في الأعلى، وشجرة الملفات (متصفح الملفات) في الأسفل. إظهار/إخفاء المخطط عبر عرض ← إظهار المخطط التفصيلي، وشجرة الملفات بـ Ctrl+B.
المحررالوسطمحرر نصوص عادي مع تمييز صيغة Markdown وأرقام الأسطر.
المعاينةاليمينعرض HTML حي للمستند الحالي. التبديل بـ Ctrl+Shift+P.

اسحب أي مقبض فاصل لتغيير حجم اللوحات. يُحفظ حجم اللوحات بين الجلسات.

مخطط المستند

يعرض لوح مخطط المستند جميع العناوين (H1–H6) كشجرة للتنقل في الشريط الجانبي الأيسر.

  • انقر على أي عنوان لنقل مؤشر المحرر إلى ذلك السطر.
  • يتحدث المخطط تلقائياً أثناء الكتابة.
  • إظهار/إخفاء عبر عرض ← إظهار المخطط التفصيلي. يُحفظ التفضيل بين الجلسات.

شريط الأدوات

يوفر شريط الأدوات أسفل شريط القوائم وصولاً بنقرة واحدة إلى أكثر الإجراءات شيوعاً: جديد، فتح، حفظ، إدراج رابط، إدراج صورة، إدراج جدول، إدراج PlantUML، إدراج Mermaid، تبديل المعاينة، والإعدادات. جميع إجراءات شريط الأدوات تكرار لعناصر القائمة واختصارات لوحة المفاتيح.

شريط الحالة

يعرض شريط الحالة في أسفل النافذة دائماً قطعتَي معلومات:

  • عدد الكلمات — إجمالي عدد الكلمات في المستند (يتحدث حياً أثناء الكتابة).
  • موضع المؤشر — رقم السطر والعمود الحاليَّين، مثل السطر 12، العمود 5.

شريط القوائم

يحتوي شريط القوائم على خمس قوائم رئيسية:

القائمةالمحتويات
ملفجديد، فتح، فتح مجلد، الملفات الأخيرة، استيراد PDF، فتح من Git، Git Squash، حفظ، حفظ باسم، تصدير كـ PDF، إنهاء
تحريرتراجع، إعادة، قص، نسخ، لصق
إدراجرابط، صورة، PlantUML، Mermaid، جدول
عرضإظهار شجرة الملفات، إظهار المخطط التفصيلي، إظهار المعاينة، التفاف الكلمات، تدقيق إملائي، لغة التدقيق الإملائي، الإعدادات
مساعدةمرجع Markdown، حول

عنوان النافذة

يعرض عنوان النافذة اسم الملف المفتوح حالياً. عند إجراء تغييرات غير محفوظة، يُضاف علامة نجمة (*) إلى اسم الملف، مثل README.md *. عندما لا يكون هناك ملف مفتوح، يعرض العنوان بلا عنوان.

٤. المحرر

تستضيف اللوحة الوسطى محرر نص عادي متكامل مُهيَّأ لتأليف Markdown.

تمييز الصيغة

يطبق MarkForge تمييز الصيغة الذي يدرك Markdown على المستند أثناء الكتابة. تُلوَّن العناوين وعلامات التأكيد والروابط ونطاقات الكود وكتل الكود المسيَّجة وغيرها من عناصر Markdown بألوان مميزة. يتكيف التمييز تلقائياً عند التبديل بين ثيمَي المحرر الفاتح والداكن.

أرقام الأسطر

يعرض هامش على الجانب الأيسر من المحرر أرقام الأسطر. يُبرَز السطر الحالي بلون خلفية خفيف لمساعدتك على تتبع موضعك في المستندات الطويلة.

التفاف الكلمات

التفاف الكلمات ممكَّن افتراضياً. تُلتف الأسطر الطويلة لتناسب عرض المحرر المرئي. يمكنك تبديل التفاف الكلمات في أي وقت عبر عرض ← التفاف الكلمات. يُحفظ التفضيل بين الجلسات.

ثيمات المحرر

يتوفر ثيمان مدمجان للمحرر:

  • داكن (الافتراضي) — خلفية داكنة مع ألوان صيغة عالية التباين.
  • فاتح — خلفية بيضاء مناسبة للبيئات المضاءة.

بدِّل الثيمات في عرض ← الإعدادات ← ثيم المحرر. يسري التغيير فوراً دون إعادة تشغيل.

نص العنصر النائب

عندما لا يكون هناك ملف مفتوح، يكون المحرر معطلاً ويعرض الرسالة: "افتح أو أنشئ ملف Markdown لبدء التحرير." يُنشَّط المحرر بمجرد إنشاء ملف جديد أو فتح ملف موجود.

عمليات التحرير

جميع عمليات تحرير النص المعيارية متاحة:

  • Ctrl+Z — تراجع
  • Ctrl+Y — إعادة
  • Ctrl+X / Ctrl+C / Ctrl+V — قص / نسخ / لصق

بحث واستبدال

اضغط Ctrl+F لفتح شريط البحث، أو Ctrl+H لفتحه مع صف الاستبدال. يظهر الشريط فوق المحرر.

  • اكتب في حقل البحث — تُسلَّط الضوء على جميع التطابقات فوراً في المحرر.
  • انتقل بين التطابقات باستخدام زرَّي ↑ / ↓، أو اضغط Enter (التالي) / Shift+Enter (السابق) في حقل البحث.
  • يُظهر عداد التطابقات الموضع الحالي (مثلاً ٣ / ١٢).
  • Aa — تفعيل مطابقة حالة الأحرف.
  • [W] — مطابقة الكلمات الكاملة فقط.
  • استخدم استبدال لاستبدال التطابق الحالي، أو استبدال الكل لاستبدال جميع التكرارات دفعةً واحدة.
  • اضغط Escape أو انقر على ✕ لإغلاق الشريط وإزالة جميع التمييزات.

إذا كان هناك نص محدد في المحرر عند فتح الشريط، يُملأ حقل البحث تلقائياً بهذا النص (أسطر مفردة فقط).

التدقيق الإملائي

يمكن لـ MarkForge وضع خط أحمر متموج تحت الكلمات المكتوبة بشكل خاطئ أثناء الكتابة. التدقيق الإملائي معطل افتراضياً.

  • تشغيل/إيقاف عبر عرض ← تدقيق إملائي (مفتاح تبديل).
  • اختر لغة القاموس عبر عرض ← لغة التدقيق الإملائي. اللغات المتاحة: English, Deutsch, Español, Français, Italiano, Nederlands, Português, Русский, العربية, فارسی.
  • الكلمات داخل أجزاء الكود المضمنة (`…`) وعناوين URL لا تُوضع تحتها خطوط أبداً.
  • الاختصارات بالأحرف الكبيرة (HTML, API, URL…) يتم تخطيها.
  • يُحفظ تفضيل التدقيق الإملائي واللغة المختارة بين الجلسات.
  • يتطلب مكتبة pyspellchecker. في حال عدم التثبيت، يُتجاهل المفتاح بصمت.

٥. المعاينة الحية

تعرض اللوحة اليمنى نسخة HTML مُقدَّمة بالكامل من مستند Markdown الخاص بك، تتحدث تلقائياً أثناء الكتابة.

العرض

تستخدم المعاينة ورقة أنماط مستوحاة من GitHub لإخراج نظيف وسهل القراءة. تدعم جميع امتدادات Markdown بما فيها الجداول وقوائم المهام والحواشي وتمييز الكود والرياضيات والرسوم البيانية. تعالج خطوط أنابيب العرض النص عبر مكتبة Python markdown مع الامتدادات extra وcodehilite وtoc وsane_lists.

الحفاظ على موضع التمرير

عند تحرير المستند، تتحدث المعاينة بعد تأخير تخميد قدره 300 مللي ثانية. قبل تحديث HTML، يقرأ MarkForge موضع التمرير الحالي من عرض الويب ويستعيده بعد العرض، بحيث لا تقفز المعاينة إلى الأعلى أثناء عملك في منتصف المستند.

ثيمات المعاينة

يتوفر ثيمان للمعاينة:

  • داكن (الافتراضي) — خلفية داكنة تتطابق مع ثيم التطبيق العام.
  • فاتح — خلفية بيضاء لتجربة قراءة تشبه الطباعة.

بدِّل عبر عرض ← الإعدادات ← ثيم المعاينة. يسري التغيير فوراً.

تبديل المعاينة

اضغط Ctrl+Shift+P أو استخدم عرض ← إظهار المعاينة لإخفاء لوحة المعاينة أو إظهارها. إخفاء المعاينة يمنح المحرر مساحة شاشة أكبر.

ملاحظة

تتطلب الصيغ الرياضية (MathJax) ورسوم Mermaid البيانية اتصالاً بالإنترنت عند الاستخدام الأول، إذ تُحمَّل المكتبات من CDN. دائماً ما تُجلَب رسوم PlantUML البيانية من plantuml.com وتتطلب الوصول إلى الإنترنت.

٦. عمليات الملفات

ملف جديد

اضغط Ctrl+N أو اختر ملف ← جديد. إذا كان المستند الحالي يحتوي على تغييرات غير محفوظة، سيطلب منك MarkForge الحفظ قبل المتابعة. يُفتح مستند جديد فارغ ويُنشَّط المحرر.

فتح ملف

اضغط Ctrl+O أو اختر ملف ← فتح… لفتح مربع حوار استعراض الملفات. ملفات Markdown (.md، .markdown) والملفات النصية (.txt) وجميع الملفات الأخرى قابلة للتحديد. ستتحدث شجرة الملفات لتعرض المجلد الذي يحتوي على الملف المفتوح.

فتح مجلد

اختر ملف ← فتح مجلد… (Ctrl+Shift+O) لتحديد مجلد. ستعرض شجرة الملفات جميع ملفات .md والصور في ذلك المجلد. انقر على أي ملف في الشجرة لفتحه في المحرر.

حفظ

اضغط Ctrl+S أو اختر ملف ← حفظ. إذا كان الملف مفتوحاً من مستودع Git، سيظهر مربع حوار Git commit وpush بدلاً من ذلك (انظر الحفظ عبر Git). بالنسبة للملفات المحلية، يُكتب الملف فوراً على القرص وتختفي علامة النجمة في شريط العنوان.

حفظ باسم

اضغط Ctrl+Shift+S أو اختر ملف ← حفظ باسم… لحفظ المستند الحالي في موقع جديد أو باسم جديد.

شجرة الملفات

تعرض اللوحة اليسرى قائمة مسطحة بملفات .md والصور في المجلد الجذر الحالي. المجلدات الفرعية التي لا تحتوي على ملفات ذات صلة مباشرة مخفية لتقليل الفوضى. انقر على ملف لفتحه. يعرض رأس الشجرة [GIT] اسم_المستودع باللون الأخضر عند فتح ملف تديره Git.

نصيحة

بدِّل شجرة الملفات بـ Ctrl+B لتعظيم مساحة المحرر. يُحفظ تفضيلك بين الجلسات.

الملفات الأخيرة

يحتفظ MarkForge بقائمة آخر 10 ملفات تم فتحها. يمكن الوصول إليها عبر ملف ← الملفات الأخيرة.

  • انقر على أي إدخال لإعادة فتح الملف فوراً.
  • إذا لم يعد الملف موجوداً على القرص، يُزال تلقائياً من القائمة.
  • اختر مسح الملفات الأخيرة في أعلى القائمة الفرعية لمسح السجل بأكمله.
  • يُحفظ السجل بين الجلسات.

٧. مرجع Markdown

يدعم MarkForge Markdown القياسي CommonMark إضافة إلى الامتدادات التالية: extra (الجداول والحواشي والاختصارات)، codehilite (كتل الكود المميَّزة بالصيغة)، toc (جدول المحتويات)، sane_lists، والشطب.

Markdown

٨. الصيغ الرياضية

يُقدِّم MarkForge ترميز الرياضيات بصيغة LaTeX باستخدام MathJax. يلزم اتصال بالإنترنت عند الاستخدام الأول (يُحمَّل MathJax من CDN).

الرياضيات المضمَّنة

أحِط التعبير بعلامتَي دولار مفردتَين لعرضه مضمَّناً داخل فقرة:

The famous equation $E = mc^2$ was published in 1905.

النتيجة: المعادلة الشهيرة E = mc² نُشرت عام 1905.

الرياضيات في كتلة مستقلة

أحِط التعبير بعلامتَي دولار مزدوجتَين لعرضه في منتصف سطره الخاص:

$$\int_0^\infty e^{-x}\, dx = 1$$

كتلة الرياضيات في سطر واحد

يمكنك أيضاً استخدام علامة $ منفردة في سطر خاص بها لفتح وإغلاق كتلة الرياضيات المعروضة:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
نصيحة

يدعم MathJax وضع الرياضيات الكامل في LaTeX، بما فيه الكسور والمصفوفات والحروف اليونانية والمعاملات والماكرو المخصصة. ارجع إلى توثيق MathJax للاطلاع على الصيغة الكاملة.

٩. نوافذ الإدراج

يوفر MarkForge نوافذ حوار موجَّهة لإدراج عناصر Markdown الشائعة. تُدرج جميع النوافذ النص في موضع المؤشر الحالي في المحرر.

إدراج رابط Ctrl+K

افتح عبر إدراج ← رابط… أو اضغط Ctrl+K.

يحتوي مربع الحوار على ثلاثة حقول:

  • نص العرض — تسمية الرابط المرئية (مثل انقر هنا).
  • URL — عنوان الوجهة (مثل https://example.com).
  • العنوان (اختياري) — تلميح يظهر عند تمرير المؤشر فوق الرابط.

انقر إدراج لوضع [نص العرض](URL "العنوان") في المحرر.

إدراج صورة Ctrl+Shift+K

افتح عبر إدراج ← صورة… أو اضغط Ctrl+Shift+K.

يحتوي مربع الحوار على حقلَين:

  • النص البديل — وصف قصير لقارئات الشاشة.
  • URL / المسار — عنوان URL بعيد أو مسار ملف محلي. استخدم زر الاستعراض لاختيار ملف صورة محلي.

انقر إدراج لوضع ![النص البديل](URL) في المحرر.

إدراج جدول Ctrl+Shift+T

افتح عبر إدراج ← جدول… أو اضغط Ctrl+Shift+T.

اضبط هيكل جدولك:

  • صفوف البيانات — عدد الصفوف باستثناء صف الرأس.
  • الأعمدة — عدد الأعمدة (حتى الحد الأقصى المدعوم).
  • أسماء الأعمدة — أدخل عنواناً لكل عمود.
  • المحاذاة — اختر يسار أو وسط أو يمين لكل عمود بشكل مستقل.

تتحدث معاينة Markdown الحية في أسفل مربع الحوار. انقر إدراج لوضع الجدول في المحرر.

إدراج PlantUML Ctrl+Shift+U

افتح عبر إدراج ← PlantUML… أو اضغط Ctrl+Shift+U.

اختر من ١٢ قالب رسم بياني في القائمة المنسدلة:

  • مخطط التسلسل (Sequence diagram)
  • مخطط الفئات (Class diagram)
  • مخطط الأنشطة (Activity diagram)
  • مخطط الحالة (State diagram)
  • مخطط المكونات (Component diagram)
  • مخطط حالات الاستخدام (Use case diagram)
  • مخطط غانت (Gantt diagram)
  • الخريطة الذهنية (Mind map)
  • هيكل تفصيل العمل (WBS)
  • مخطط التوقيت (Timing diagram)
  • مخطط النشر (Deployment diagram)
  • JSON

يملأ القالب المحدد محرر كود PlantUML. عدِّل الكود حسب الحاجة، ثم انقر إدراج. يُغلَّف الرسم البياني في كتلة كود مسيَّجة ويُعرض عبر خدمة plantuml.com الإلكترونية في المعاينة. يلزم اتصال بالإنترنت.

ملاحظة

تظهر معاينة الرسم البياني تلقائياً في النافذة الرئيسية بعد الإدراج. يستخدم العرض خادم plantuml.com العام؛ لا حاجة إلى تثبيت PlantUML محلياً.

إدراج Mermaid Ctrl+Shift+M

افتح عبر إدراج ← Mermaid… أو اضغط Ctrl+Shift+M.

اختر من ٩ قوالب رسوم بيانية:

  • مخطط انسيابي (Flowchart)
  • مخطط التسلسل (Sequence diagram)
  • مخطط الفئات (Class diagram)
  • مخطط الحالة (State diagram)
  • غانت (Gantt)
  • مخطط دائري (Pie chart)
  • خريطة ذهنية (Mindmap)
  • مخطط ER (ER diagram)
  • رسم بياني Git (Git graph)

عدِّل كود Mermaid في منطقة النص وانقر إدراج. يُغلَّف الرسم البياني في كتلة كود mermaid مسيَّجة ويُعرض في المعاينة عبر kroki.io. يلزم اتصال بالإنترنت.

١٠. ميزات PDF

تصدير كـ PDF Ctrl+Shift+E

اختر ملف ← تصدير كـ PDF… أو اضغط Ctrl+Shift+E. يُقدِّم MarkForge المستند الحالي إلى ملف PDF باستخدام نفس خط الأنابيب كالمعاينة الحية، ولكن بخلفية بيضاء وCSS محسَّن للطباعة. يظهر مربع حوار حفظ الملف لتتمكن من اختيار موقع الإخراج واسم الملف.

نصيحة

بدِّل المعاينة إلى الثيم الفاتح قبل التصدير إلى PDF للحصول على أنظف مخرجات جاهزة للطباعة.

استيراد PDF Ctrl+Shift+I

اختر ملف ← استيراد PDF… أو اضغط Ctrl+Shift+I. يُحوِّل MarkForge أي ملف PDF إلى Markdown باستخدام مكتبة pymupdf4llm. يحافظ التحويل بذكاء على هيكل المستند:

  • العناوين والفقرات
  • الجداول
  • القوائم
  • كتل الكود حيثما يتم اكتشافها

يعمل التحويل في خيط خلفي لتظل واجهة المستخدم متجاوبة. عند الانتهاء، يطلب منك MarkForge تحديد موقع الحفظ لملف .md الناتج ثم يحمِّله في المحرر.

ملاحظة

يتطلب استيراد PDF حزمة pymupdf4llm. إذا لم تكن مثبتة، سيطلب منك MarkForge تثبيتها: pip install pymupdf4llm

١١. تكامل Git

يمكن لـ MarkForge فتح ملفات Markdown مباشرة من مستودعات Git المستضافة، وتحريرها، ودفع التغييرات مرة أخرى — دون فتح طرفية على الإطلاق. العمليات المتقدمة مثل amend وsquash مدعومة أيضاً.

المنصات المدعومة

المنصةمثال على نمط الرابط
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.mycompany.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.mycompany.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

يُكتشف النظام الأساسي تلقائياً من هيكل الرابط، وليس من اسم المضيف فحسب — لذا تعمل الحالات المستضافة ذاتياً تلقائياً.

فتح ملف من Git

  1. اختر ملف ← فتح من Git… أو اضغط Ctrl+Shift+G.
  2. الصق رابط الملف الذي تريد فتحه (انسخه من متصفح ملفات GitHub أو Bitbucket).
  3. يتحقق MarkForge من الرابط ويعرض سطر معاينة: المنصة · المالك/المستودع · المسار/إلى/الملف.md.
  4. اختياريا، تجاوز الفرع أو الوسم أو تجزئة الـ commit في حقل Ref.
  5. انقر موافق — يُستنسخ المستودع في الخلفية ويُفتح الملف في المحرر.

بينما يكون ملف تديره Git مفتوحاً، يعرض رأس شجرة الملفات [GIT] اسم_المستودع باللون الأخضر.

نصيحة

تعمل المستودعات العامة دون الحاجة إلى إعداد أي بيانات اعتماد. للمستودعات الخاصة، اضبط بيانات اعتمادك في عرض ← الإعدادات ← مصادقة Git قبل الفتح.

الحفظ (Commit وPush)

عند فتح ملف تديره Git، يفتح الضغط على Ctrl+S مربع حوار Git Commit وPush بدلاً من الحفظ العادي للملف. يحتوي مربع الحوار على:

  • رسالة الـ commit — مطلوبة. أدخل وصفاً ذا مغزى لتغييراتك.
  • تعديل الـ commit السابق (amend) — متاح من الحفظ الثاني فصاعداً عند استخدام SSH أو مصادقة git-binary. يستبدل الـ commit السابق بدلاً من إنشاء commit جديد؛ يملأ رسالة الـ commit السابقة مسبقاً؛ يستخدم force-push تلقائياً (--force-with-lease). يمكن تعديل commits MarkForge المُنشأة في الجلسة الحالية فقط.
  • الدفع إلى:
    • الفرع الحالي — يدفع مباشرة إلى الفرع الذي استنسخته منه.
    • فرع جديد — ينشئ فرعاً جديداً باسم تحدده.
  • إنشاء Pull Request — متاح عند تحديد "فرع جديد". يفتح تلقائياً pull request بعد الدفع. يعرض حقولاً لعنوان PR والفرع الهدف.

دمج الـ Commits (Squash) Ctrl+Shift+Q

متاح مع مصادقة SSH وHTTPS (git binary) فقط.

اختر ملف ← Git Squash… أو اضغط Ctrl+Shift+Q. يعرض مربع حوار دمج الـ Commits جدولاً بجميع commits على الفرع الحالي التي لم تُضم بعد إلى الفرع الأساسي المحدد.

  • حقل الفرع الأساسي — الفرع المرجعي (مثل main، master، develop). انقر تحديث بعد تغييره.
  • تحديد جميع الـ commits الجديدة — يحدد جميع الـ commits المدرجة دفعة واحدة.
  • رسالة commit الدمج — الرسالة الواحدة التي ستحل محل جميع الـ commits المحددة.
  • دمج ودفع (Squash & Push) — يعيد كتابة تاريخ الفرع ويدفع بالقوة إلى البعيد.

يجب تحديد ٢ commits على الأقل، ويجب أن تشكل نطاقاً متصلاً يبدأ من أحدث commit (HEAD).

أوضاع المصادقة

اضبط طريقة المصادقة في عرض ← الإعدادات ← مصادقة Git.

HTTPS (مدمج)

يعمل على جميع المنصات دون الحاجة إلى أي برنامج إضافي مثبَّت. يستخدم REST API للمنصة مباشرة. لا يلزم وجود ملف git ثنائي محلي.

قيد

ميزتا amend وsquash غير متاحتَين في هذا الوضع لأنه لا يتم الاحتفاظ بسجل git محلي كامل.

المنصةنوع الرمز (Token)
GitHubرمز وصول شخصي (ghp_…) — يتطلب نطاق repo
GitHub Enterpriseرمز وصول شخصي — كـ GitHub
Bitbucket Cloudكلمة مرور التطبيق (الحساب ← كلمات مرور التطبيق)
Bitbucket Serverرمز وصول شخصي أو رمز وصول HTTP

HTTPS (git binary)

يستخدم الملف التنفيذي git للنظام لجميع العمليات — استنساخ محلي كامل وcommit وpush. يتطلب Git for Windows (أو أي ملف ثنائي git على PATH الخاص بك). يدعم amend وsquash. استخدم نفس حقلَي اسم المستخدم والرمز كوضع HTTPS (المدمج). يجب أيضاً تقديم الاسم والبريد الإلكتروني لهوية الـ commit.

SSH (ملف مفتاح)

حدد SSH (ملف مفتاح) وأدخل المسار إلى مفتاح SSH الخاص وعبارة مرور اختيارية. يستخدم هذا الوضع paramiko لاتصال SSH خالص بـ Python (لا يلزم عميل ssh للنظام). إذا لم يكن paramiko مثبتاً، يلجأ MarkForge إلى عميل ssh للنظام.

pip install paramiko

تدعم مصادقة SSH كلاً من amend وsquash.

تخزين بيانات الاعتماد

يتم تخزين الرموز وعبارات مرور SSH وكلمات مرور الوكيل في سلسلة مفاتيح نظام التشغيل (GNOME Keyring / KWallet على Linux، وKeychain على macOS، وWindows Credential Manager على Windows) — وليس في ملفات تهيئة نصية عادية.

وكيل HTTP (Proxy)

إذا كانت شبكتك تتطلب وكيل HTTP للوصول إلى خوادم Git، اضبطه في عرض ← الإعدادات ← وكيل HTTP:

  • رابط الوكيل — مثل http://proxy.company.com:8080
  • اسم مستخدم الوكيل — اتركه فارغاً إذا لم تكن هناك مصادقة مطلوبة.
  • كلمة مرور الوكيل — اتركها فارغة إذا لم تكن هناك مصادقة مطلوبة.

اترك جميع حقول الوكيل فارغة لاستخدام إعدادات الوكيل الافتراضية للنظام.

الإغلاق

عند إغلاق MarkForge بينما يكون ملف تديره Git مفتوحاً، سيسأل MarkForge عما إذا كنت تريد حذف دليل الاستنساخ المؤقت. حذف الدليل يوفر مساحة على القرص؛ الإبقاء عليه يسمح لك بفحص الملفات المستنسخة لاحقاً.
