Manuale utente MarkForge v1.8.0

1. Introduzione

MarkForge è un editor Markdown moderno e ricco di funzionalità, costruito con Python e PyQt6. Combina un'esperienza di scrittura senza distrazioni con un'anteprima in tempo reale fianco a fianco, piena integrazione del flusso di lavoro Git, supporto per diagrammi, rendering matematico e importazione/esportazione PDF — tutto in un'applicazione desktop a doppio pannello pulita.

Che tu stia scrivendo documentazione tecnica, README di progetto o pubblicando contenuti direttamente in un repository GitHub, MarkForge gestisce l'intero ciclo di vita di modifica e pubblicazione senza richiedere un terminale.

2. Installazione

Linux (Debian / Ubuntu)

Scarica il pacchetto .deb dalla pagina Releases su GitHub, quindi installalo con:

sudo apt install ./markforge_*.deb

Il pacchetto installa automaticamente tutte le dipendenze di sistema necessarie (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments). Non è richiesto alcun ambiente virtuale.

Windows

Scarica MarkForge-Setup.exe dalla pagina Releases ed esegui il programma di installazione. L'assistente di installazione ti guida attraverso il processo e crea automaticamente le scorciatoie nel menu Start.

Nota

Il programma di installazione di Windows include il runtime Python e tutte le dipendenze. Non è necessaria un'installazione separata di Python.

Dal codice sorgente

Requisiti: Python 3.8 o successivo, CMake 3.16 o successivo.

Con CMake (consigliato):

# 1. Clone the repository
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Configure the build (creates a local virtual environment)
cmake -B build

# 3. Install Python dependencies into the local venv
cmake --build build --target install_deps

# 4. Run the application
cmake --build build --target run

Avvio diretto con Python (senza CMake):

pip install -r requirements.txt
python src/main.py
Suggerimento

Per le operazioni Git basate su SSH, installa anche paramiko: pip install paramiko

3. Panoramica dell'interfaccia

Layout

La finestra di MarkForge è divisa in tre pannelli ridimensionabili separati da divisori trascinabili:

PannelloPosizioneDescrizione
Struttura + Albero fileSinistraIl pannello sinistro ha due sezioni sovrapposte: la Struttura del documento (albero delle intestazioni) in alto, e l'Albero dei file (browser dei file) in basso. Attiva/disattiva la struttura tramite Visualizza → Mostra struttura, l'albero dei file con Ctrl+B.
EditorCentroEditor di testo normale con evidenziazione sintassi Markdown e numeri di riga.
AnteprimaDestraRendering HTML in tempo reale del documento corrente. Attiva/disattiva con Ctrl+Shift+P.

Trascina qualsiasi divisore per ridimensionare i pannelli. Le dimensioni dei pannelli vengono ricordate tra le sessioni.

Struttura del documento

Il pannello Struttura del documento si trova nella parte superiore della barra laterale sinistra ed elenca tutte le intestazioni (H1–H6) del documento corrente come albero navigabile.

  • Fai clic su qualsiasi intestazione per spostare il cursore dell'editor su quella riga.
  • La struttura si aggiorna automaticamente mentre scrivi.
  • Attiva/disattiva la visibilità tramite Visualizza → Mostra struttura. La preferenza viene salvata tra le sessioni.

Barra degli strumenti

La barra degli strumenti sotto la barra dei menu fornisce accesso con un clic alle azioni più comuni: Nuovo, Apri, Salva, Inserisci link, Inserisci immagine, Inserisci tabella, Inserisci PlantUML, Inserisci Mermaid, Attiva/disattiva anteprima e Impostazioni. Tutte le azioni della barra degli strumenti duplicano le voci di menu e le scorciatoie da tastiera.

Barra di stato

La barra di stato nella parte inferiore della finestra mostra sempre:

  • Conteggio parole & tempo di lettura — conteggio parole e tempo di lettura stimato (es. 342 parole · 2 min di lettura), aggiornato in tempo reale.
  • Avanzamento obiettivo parole — se impostato un obiettivo tramite Visualizza → Imposta obiettivo parole, appare una barra di avanzamento compatta.
  • Posizione cursore — riga e colonna correnti, es. Riga 12, Col 5.

Barra dei menu

La barra dei menu contiene cinque menu di primo livello:

MenuContenuto
FileNuovo, Apri, Apri cartella, File recenti, Importa PDF, Apri da Git, Git Squash, Salva, Salva con nome, Esporta come HTML, Esporta come DOCX (Pandoc), Esporta come EPUB (Pandoc), Esporta come LaTeX (Pandoc), Esporta come PDF, Esci
ModificaAnnulla, Ripeti, Taglia, Copia, Incolla

Titolo della finestra

Il titolo della finestra mostra il nome del file attualmente aperto. Quando apporti modifiche non salvate, un asterisco (*) viene aggiunto al nome del file, ad esempio README.md *. Quando nessun file è aperto, il titolo mostra Untitled.

4. L'editor

Il pannello centrale ospita un editor di testo normale completo, ottimizzato per la creazione di Markdown.

Evidenziazione sintassi

MarkForge applica l'evidenziazione sintassi Markdown al documento mentre scrivi. Le intestazioni, i marcatori di enfasi, i link, i frammenti di codice, i blocchi di codice delimitati e altre costruzioni Markdown vengono colorati in modo distinto. L'evidenziazione si adatta automaticamente quando si passa tra i temi editor Chiaro e Scuro.

Numeri di riga

Un margine sul lato sinistro dell'editor mostra i numeri di riga. La riga corrente è evidenziata con un colore di sfondo sottile per aiutarti a tenere traccia della tua posizione in documenti lunghi.

Marcatori diff inline

Quando un file gestito da Git è aperto, il margine dell'editor mostra barre colorate accanto alle righe che differiscono dall'ultima versione committata:

  • Verde — righe aggiunte
  • Arancione — righe modificate
  • Rosso — blocchi eliminati (mostrati come un segno sottile sopra il punto di eliminazione)

I marcatori si aggiornano in tempo reale durante la digitazione e si reimpostano dopo ogni commit & push riuscito.

A capo automatico

Il ritorno a capo automatico è abilitato per impostazione predefinita. Le righe lunghe vengono adattate alla larghezza visibile dell'editor. Attiva/disattiva il ritorno a capo automatico in qualsiasi momento tramite Visualizza → A capo automatico. La preferenza viene salvata tra le sessioni.

Temi dell'editor

Sono disponibili due temi dell'editor integrati:

  • Scuro (predefinito) — sfondo scuro con colori di sintassi ad alto contrasto.
  • Chiaro — sfondo bianco adatto agli ambienti luminosi.

Cambia i temi in Visualizza → Impostazioni → Tema dell'editor. La modifica ha effetto immediatamente senza riavviare.

Testo segnaposto

Quando nessun file è aperto, l'editor è disabilitato e mostra il messaggio: "Open or create a Markdown file to start editing." L'editor si attiva non appena crei un nuovo file o ne apri uno esistente.

Operazioni di modifica

Tutte le operazioni standard di modifica del testo sono disponibili:

  • Ctrl+Z — Annulla
  • Ctrl+Y — Ripeti
  • Ctrl+X / Ctrl+C / Ctrl+V — Taglia / Copia / Incolla

Trova & Sostituisci

Premi Ctrl+F per aprire la barra di ricerca, o Ctrl+H per aprirla con la riga di sostituzione visibile. La barra appare sopra l'editor.

  • Digita nel campo di ricerca — tutte le corrispondenze vengono immediatamente evidenziate nell'editor.
  • Naviga tra le corrispondenze con i pulsanti ↑ / ↓, o premi Invio (successivo) / Shift+Invio (precedente) nel campo di ricerca.
  • Il contatore di corrispondenze mostra la posizione corrente (ad esempio 3 / 12).
  • Aa — attiva/disattiva la ricerca con distinzione maiuscole/minuscole.
  • [W] — attiva/disattiva la corrispondenza di parole intere.
  • Usa Sostituisci per sostituire la corrispondenza corrente, o Sostituisci tutto per sostituire tutte le occorrenze in una volta.
  • Premi Esc o fai clic su ✕ per chiudere la barra e cancellare tutte le evidenziazioni.

Se c'è del testo selezionato nell'editor quando apri la barra, il campo di ricerca viene precompilato con la selezione (solo selezioni di una riga).

Controllo ortografico

MarkForge può sottolineare le parole errate con una linea ondulata rossa mentre scrivi. Il controllo ortografico è disabilitato per impostazione predefinita.

  • Abilitalo tramite Visualizza → Controllo ortografico (interruttore).
  • Seleziona la lingua del dizionario tramite Visualizza → Lingua controllo ortografico. Lingue disponibili: English, Deutsch, Español, Français, Italiano, Nederlands, Português, العربية, فارسی (tramite pyspellchecker) e Tiếng Việt*, Svenska*, Українська*, ಕನ್ನಡ*, हिंदी* (tramite enchant/Hunspell).
  • Le parole all'interno di frammenti di codice in linea (`…`) e gli URL non vengono mai sottolineati.
  • Le abbreviazioni IN MAIUSCOLO (HTML, API, URL…) vengono ignorate.
  • La preferenza del controllo ortografico e la lingua selezionata vengono salvate tra le sessioni.
  • Richiede la libreria pyspellchecker. Se non è installata, l'interruttore viene ignorato silenziosamente.
  • Le lingue contrassegnate con * richiedono la libreria pyenchant e i dizionari Hunspell di sistema (ad esempio hunspell-vi su Debian/Ubuntu).

5. Anteprima in tempo reale

Il pannello destro mostra una versione HTML completamente renderizzata del documento Markdown, aggiornata automaticamente mentre scrivi.

Rendering

L'anteprima utilizza un foglio di stile ispirato a GitHub per un output pulito e leggibile. Supporta tutte le estensioni Markdown, incluse tabelle, elenchi di attività, note a piè di pagina, evidenziazione del codice, matematica e diagrammi. Il pipeline di rendering elabora il tuo testo tramite la libreria Python markdown con le estensioni extra, codehilite, toc e sane_lists.

Preservazione della posizione di scorrimento

Quando modifichi il documento, l'anteprima si aggiorna dopo un breve ritardo di debounce di 300 ms. Prima di aggiornare l'HTML, MarkForge legge la posizione di scorrimento corrente dalla vista web e la ripristina dopo il rendering, in modo che l'anteprima non salti all'inizio mentre stai lavorando nel mezzo di un documento.

Temi dell'anteprima

Sono disponibili due temi di anteprima:

  • Scuro (predefinito) — sfondo scuro che corrisponde al tema generale dell'applicazione.
  • Chiaro — sfondo bianco per un'esperienza di lettura simile alla stampa.

Cambia tramite Visualizza → Impostazioni → Tema dell'anteprima. La modifica ha effetto immediatamente.

Attivare/disattivare l'anteprima

Premi Ctrl+Shift+P o usa Visualizza → Mostra anteprima per nascondere o mostrare il pannello di anteprima. Nascondere l'anteprima dà all'editor più spazio sullo schermo.

Nota

Le formule matematiche (MathJax) e i diagrammi Mermaid richiedono una connessione Internet al primo utilizzo, poiché le librerie vengono caricate da una CDN. I diagrammi PlantUML vengono sempre recuperati da plantuml.com e richiedono l'accesso a Internet.

6. Operazioni sui file

Nuovo file

Premi Ctrl+N o scegli File → Nuovo. Se il documento corrente ha modifiche non salvate, MarkForge ti chiederà di salvare prima di procedere. Viene aperto un nuovo documento vuoto e l'editor viene attivato.

Apri file

Premi Ctrl+O o scegli File → Apri… per aprire una finestra di dialogo del browser file. Sono selezionabili file Markdown (.md, .markdown), file di testo (.txt) e tutti gli altri file. L'albero dei file verrà aggiornato per mostrare la cartella contenente il file aperto.

Apri cartella

Scegli File → Apri cartella… (Ctrl+Shift+O) per selezionare una directory. L'albero dei file mostrerà quindi tutti i file .md e le immagini in quella directory. Fai clic su qualsiasi file nell'albero per aprirlo nell'editor.

Salva

Premi Ctrl+S o scegli File → Salva. Se il file è stato aperto da un repository Git, apparirà invece una finestra di dialogo Git commit & push (vedi Salvataggio tramite Git). Per i file locali, il file viene scritto immediatamente su disco e l'asterisco nella barra del titolo scompare.

Salva con nome

Premi Ctrl+Shift+S o scegli File → Salva con nome… per salvare il documento corrente in una nuova posizione o con un nuovo nome.

Albero dei file

Il pannello sinistro mostra un elenco piatto di file .md e immagini nella directory radice corrente. Le sottodirectory che non contengono file direttamente rilevanti sono nascoste per ridurre il disordine. Fai clic su un file per aprirlo. L'intestazione dell'albero mostra [GIT] reponame in verde quando è aperto un file gestito da Git.

Suggerimento

Attiva/disattiva l'albero dei file con Ctrl+B per massimizzare lo spazio dell'editor. La tua preferenza viene salvata tra le sessioni.

File recenti

MarkForge mantiene un elenco degli ultimi 10 file che hai aperto. Accedi a essi tramite File → File recenti.

  • Fai clic su qualsiasi voce per riaprire immediatamente il file.
  • Se un file non esiste più sul disco, viene rimosso automaticamente dall'elenco.
  • Scegli Cancella file recenti nella parte superiore del sottomenu per cancellare l'intera cronologia.
  • L'elenco viene salvato tra le sessioni.

Trascina & Rilascia

Puoi trascinare file dal tuo gestore file sull'editor. MarkForge mostra una finestra di dialogo per scegliere come gestire ogni tipo di file:

  • File immagine (.png, .jpg, .gif, ecc.) — scegli di inserire con il percorso originale o di copiare nella cartella assets/ e inserire un collegamento relativo.
  • File Markdown (.md, .markdown) — Apri il file dalla posizione originale o Copia nella cartella (copia nella stessa directory del documento attuale e apre la copia).
  • File PDF — Importa … (flusso di importazione standard con finestra di salvataggio) o Importa nella cartella (importa automaticamente nella directory del documento attuale).
  • Qualsiasi altro file — inserisce un collegamento Markdown [nome file](percorso/al/file).
Nota

Per le operazioni che copiano o importano nella cartella del documento attuale, un file deve essere aperto e salvato affinché MarkForge conosca la directory di destinazione.

Modalità focus F11

Premi F11 o scegli Visualizza → Modalità focus per entrare nella modalità di scrittura senza distrazioni. MarkForge nasconde l'albero dei file, il contorno del documento, l'anteprima in tempo reale e la barra di stato, lasciando l'editor occupare l'intera finestra.

Premi di nuovo F11 per uscire: tutti i pannelli e la barra di stato vengono ripristinati esattamente come erano prima.

Suggerimento

La modalità focus funziona indipendentemente dai pulsanti di attivazione/disattivazione dei pannelli individuali (Ctrl+B, Ctrl+Shift+P).

7. Riferimento Markdown

MarkForge supporta il Markdown CommonMark standard più le seguenti estensioni: extra (tabelle, note a piè di pagina, abbreviazioni), codehilite (blocchi di codice con evidenziazione sintassi), toc (indice), sane_lists e testo barrato.

Intestazioni

Markdown
# Heading 1 ## Heading 2 ### Heading 3 #### Heading 4 ##### Heading 5 ###### Heading 6
Risultato

Heading 1

Heading 2

Heading 3

Enfasi & Formattazione in linea

Markdown
**bold text** *italic text* ~~strikethrough~~ `inline code` **_bold italic_**
Risultato
bold text
italic text
strikethrough
inline code
bold italic

Elenchi

Elenco non ordinato:

Markdown
- First item - Second item - Nested item - Third item
Risultato
  • First item
  • Second item
    • Nested item
  • Third item

Elenco ordinato:

Markdown
1. First item 2. Second item 3. Third item
Risultato
  1. First item
  2. Second item
  3. Third item

Elenco di attività (caselle di controllo):

Markdown
- [x] Done task - [ ] Pending task - [ ] Another task
Risultato
  • ✅ Done task
  • ☐ Pending task
  • ☐ Another task

Link & Immagini

Markdown
[Link text](https://example.com) [Tooltip link](https://example.com "My tooltip") ![Alt text](image.png) ![Alt text](https://example.com/img.png)
Risultato
Link text
Tooltip link

[image: Alt text]

Tabelle (GFM)

Markdown
| Left | Center | Right | |:-----|:------:|------:| | A | B | C | | D | E | F |
Risultato
LeftCenterRight
ABC
DEF

L'allineamento delle colonne è controllato dalla posizione dei due punti nella riga separatrice: :--- sinistra, :---: centro, ---: destra.

Blocchi di codice

Markdown
```python def hello(name): print(f"Hello, {name}!") ```
Risultato
def hello(name):
    print(f"Hello, {name}!")

Specifica qualsiasi identificatore di linguaggio dopo il triplo backtick di apertura per l'evidenziazione sintassi: python, javascript, bash, sql, json, ecc. (alimentato da Pygments).

Altri elementi

Citazioni

Markdown
> This is a blockquote. > It can span multiple lines.
Risultato
This is a blockquote. It can span multiple lines.

Note a piè di pagina

Markdown
Some text[^1] with a footnote. [^1]: The footnote content.
Risultato
Some text[1] with a footnote.

Abbreviazioni

Markdown
The HTML spec is long. *[HTML]: HyperText Markup Language
Risultato
The HTML spec is long.

Indice

Inserisci [TOC] ovunque nel documento per inserire un indice auto-generato basato sulle intestazioni del documento.

Linea orizzontale

Markdown
--- or ***
Risultato

8. Formule matematiche

MarkForge renderizza la notazione matematica LaTeX usando MathJax. È richiesta una connessione Internet al primo utilizzo (MathJax viene caricato da una CDN).

Matematica in linea

Racchiudi un'espressione in simboli del dollaro singoli per renderizzarla in linea all'interno di un paragrafo:

The famous equation $E = mc^2$ was published in 1905.

Risultato: La famosa equazione E = mc² fu pubblicata nel 1905.

Matematica in blocco

Racchiudi l'espressione in simboli del dollaro doppi per renderizzarla centrata sulla propria riga:

$$\int_0^\infty e^{-x}\, dx = 1$$

Matematica in blocco su una singola riga

Puoi anche usare un $ solitario sulla propria riga per aprire e chiudere un blocco di matematica in visualizzazione:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
Suggerimento

MathJax supporta la modalità matematica LaTeX completa, incluse frazioni, matrici, lettere greche, operatori e macro personalizzate. Consulta la documentazione MathJax per la sintassi completa.

9. Finestre di inserimento

MarkForge fornisce finestre di dialogo guidate per l'inserimento di costruzioni Markdown comuni. Tutte le finestre di dialogo inseriscono testo nella posizione corrente del cursore nell'editor.

Inserisci link Ctrl+K

Apri tramite Inserisci → Link… o premi Ctrl+K.

La finestra di dialogo ha tre campi:

  • Testo di visualizzazione — l'etichetta del link visibile (ad esempio Fai clic qui).
  • URL — l'indirizzo di destinazione (ad esempio https://example.com).
  • Titolo (facoltativo) — un tooltip mostrato quando l'utente passa il mouse sul link.

Fai clic su Inserisci per posizionare [Testo di visualizzazione](URL "Titolo") nell'editor.

Inserisci immagine Ctrl+Shift+K

Apri tramite Inserisci → Immagine… o premi Ctrl+Shift+K.

La finestra di dialogo ha due campi:

  • Testo alternativo — una breve descrizione per i lettori di schermo.
  • URL / Percorso — un URL remoto o un percorso di file locale. Usa il pulsante Sfoglia per selezionare un file immagine locale.

Fai clic su Inserisci per posizionare ![Testo alternativo](URL) nell'editor.

Inserisci tabella Ctrl+Shift+T

Apri tramite Inserisci → Tabella… o premi Ctrl+Shift+T.

Configura la struttura della tabella:

  • Righe di dati — numero di righe esclusa la riga di intestazione.
  • Colonne — numero di colonne (fino al massimo supportato).
  • Nomi delle colonne — inserisci un titolo per ogni colonna.
  • Allineamento — scegli Sinistra, Centro o Destra per ogni colonna in modo indipendente.

Un'anteprima Markdown in tempo reale si aggiorna nella parte inferiore della finestra di dialogo. Fai clic su Inserisci per posizionare la tabella nell'editor.

Inserisci PlantUML Ctrl+Shift+U

Apri tramite Inserisci → PlantUML… o premi Ctrl+Shift+U.

Scegli tra 12 modelli di diagramma nel menu a discesa:

  • Diagramma di sequenza
  • Diagramma delle classi
  • Diagramma di attività
  • Diagramma di stato
  • Diagramma dei componenti
  • Diagramma dei casi d'uso
  • Diagramma di Gantt
  • Mappa mentale
  • WBS (Scomposizione del lavoro)
  • Diagramma di temporizzazione
  • Diagramma di distribuzione
  • JSON

Il modello selezionato riempie l'editor di codice PlantUML. Modifica il codice secondo necessità, quindi fai clic su Inserisci. Il diagramma è avvolto in un blocco di codice delimitato e renderizzato tramite il servizio online plantuml.com nell'anteprima. È richiesta una connessione Internet.

Nota

L'anteprima del diagramma appare automaticamente nella finestra principale dopo l'inserimento. Il rendering utilizza il server pubblico plantuml.com; non è necessaria alcuna installazione locale di PlantUML.

Inserisci Mermaid Ctrl+Shift+M

Apri tramite Inserisci → Mermaid… o premi Ctrl+Shift+M.

Scegli tra 9 modelli di diagramma:

  • Diagramma di flusso
  • Diagramma di sequenza
  • Diagramma delle classi
  • Diagramma di stato
  • Gantt
  • Grafico a torta
  • Mappa mentale
  • Diagramma ER
  • Grafico Git

Modifica il codice Mermaid nell'area di testo e fai clic su Inserisci. Il diagramma è avvolto in un blocco di codice mermaid delimitato e renderizzato nell'anteprima tramite kroki.io. È richiesta una connessione Internet.

10. Funzionalità PDF

Esporta come PDF Ctrl+Shift+E

Scegli File → Esporta come PDF… o premi Ctrl+Shift+E. MarkForge renderizza il documento corrente in un file PDF usando lo stesso processo dell'anteprima in tempo reale, ma con uno sfondo bianco e CSS ottimizzato per la stampa. Appare una finestra di dialogo per il salvataggio del file in modo che tu possa scegliere la posizione di output e il nome del file.

Suggerimento

Passa l'anteprima al tema Chiaro prima di esportare in PDF per ottenere l'output più pulito pronto per la stampa.

Esporta come HTML

Scegli File → Esporta come HTML…. MarkForge renderizza il documento corrente in un file HTML autonomo con tutto il CSS integrato e le immagini locali convertite in URI dati Base64, in modo che il file possa essere aperto in qualsiasi browser senza connessione a internet.

Esporta tramite Pandoc (DOCX, EPUB, LaTeX)

Scegli File → Esporta come DOCX (Pandoc)…, Esporta come EPUB (Pandoc)… o Esporta come LaTeX (Pandoc)…. Queste opzioni richiedono che pandoc sia installato e disponibile nel PATH. Se pandoc non viene trovato, viene mostrato un dialogo di avviso.

Importa PDF Ctrl+Shift+I

Scegli File → Importa PDF… o premi Ctrl+Shift+I. MarkForge converte qualsiasi file PDF in Markdown usando la libreria pymupdf4llm. La conversione preserva intelligentemente la struttura del documento:

  • Intestazioni e paragrafi
  • Tabelle
  • Elenchi
  • Blocchi di codice dove rilevati

La conversione viene eseguita in un thread in background in modo che l'interfaccia utente rimanga reattiva. Al termine, MarkForge ti chiede di scegliere una posizione di salvataggio per il file .md risultante e poi lo carica nell'editor.

Nota

L'importazione PDF richiede il pacchetto pymupdf4llm. Se non è installato, MarkForge ti chiederà di installarlo: pip install pymupdf4llm

11. Integrazione Git

MarkForge può aprire file Markdown direttamente da repository Git ospitati, permetterti di modificarli e inviare le modifiche — senza mai aprire un terminale. Sono supportate anche operazioni avanzate come amend e squash.

Piattaforme supportate

PiattaformaEsempio di pattern URL
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.mycompany.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.mycompany.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

La piattaforma viene rilevata automaticamente dalla struttura dell'URL, non solo dal nome host — quindi le istanze auto-ospitate funzionano automaticamente.

Apertura di un file da Git

  1. Scegli File → Apri da Git… o premi Ctrl+Shift+G.
  2. Incolla l'URL del file che vuoi aprire (copialo dal browser di file di GitHub o Bitbucket).
  3. MarkForge convalida l'URL e mostra una riga di anteprima: piattaforma · proprietario/repo · percorso/al/file.md.
  4. Facoltativamente, sostituisci il branch, il tag o l'hash del commit nel campo Ref.
  5. Fai clic su OK — il repository viene clonato in background e il file si apre nell'editor.

Mentre è aperto un file gestito da Git, l'intestazione dell'albero dei file mostra [GIT] reponame in verde.

Suggerimento

I repository pubblici funzionano senza credenziali configurate. Per i repository privati, configura le tue credenziali in Visualizza → Impostazioni → Autenticazione Git prima di aprire.

Salvataggio (Commit & Push)

Quando è aperto un file gestito da Git, premere Ctrl+S apre la finestra di dialogo Git Commit & Push invece del normale salvataggio del file. La finestra di dialogo contiene:

  • Messaggio di commit — obbligatorio. Inserisci una descrizione significativa delle tue modifiche.
  • Modifica commit precedente — disponibile dal secondo salvataggio in poi quando si utilizza l'autenticazione SSH o git-binary. Sostituisce il commit precedente invece di crearne uno nuovo; precompila il messaggio di commit precedente; utilizza automaticamente force-push (--force-with-lease). Solo i commit effettuati da MarkForge nella sessione corrente possono essere modificati.
  • Push su:
    • Branch corrente — invia direttamente al branch da cui hai clonato.
    • Nuovo branch — crea un nuovo branch con un nome che specifichi.
  • Crea Pull Request — disponibile quando è selezionato "Nuovo branch". Apre automaticamente una pull request dopo il push. Mostra campi per il titolo della PR e il branch di destinazione.

Squash dei commit Ctrl+Shift+Q

Disponibile solo con autenticazione SSH e HTTPS (git binary).

Scegli File → Git Squash… o premi Ctrl+Shift+Q. La finestra di dialogo Squash dei commit mostra una tabella di tutti i commit sul branch corrente che non sono ancora nel branch base scelto.

  • Campo branch base — il branch di riferimento (ad esempio main, master, develop). Fai clic su Ricarica dopo averlo modificato.
  • Seleziona tutti i nuovi commit — seleziona tutti i commit elencati in una volta.
  • Messaggio di commit squash — il messaggio unico che sostituirà tutti i commit selezionati.
  • Squash & Push — riscrive la cronologia del branch e fa force-push al remoto.

È necessario selezionare almeno 2 commit, e devono formare un intervallo contiguo a partire dal commit più recente (HEAD).

Cambio ramo

Quando un file gestito da Git è aperto, la barra di stato mostra il nome del ramo attuale in verde. Clicca su di esso per aprire il dialogo Cambio ramo.

  • Scarica più recente — riscaricare le ultime modifiche per il ramo attuale.
  • Filtro — digita per cercare nell'elenco dei rami remoti.
  • Cambia — seleziona un ramo diverso e clicca su Cambia (o doppio clic) per attivarlo. Il file si ricarica automaticamente con il contenuto del nuovo ramo.

Tutti e tre i metodi di autenticazione sono supportati: HTTPS (API), HTTPS (git binary) e SSH.

Modalità di autenticazione

Configura il tuo metodo di autenticazione in Visualizza → Impostazioni → Autenticazione Git.

HTTPS (integrato)

Funziona su tutte le piattaforme senza alcun software aggiuntivo installato. Utilizza direttamente l'API REST della piattaforma. Non è richiesto alcun binario git locale.

Limitazione

Le funzioni amend e squash non sono disponibili in questa modalità perché non viene mantenuta alcuna cronologia git locale completa.

PiattaformaTipo di token
GitHubToken di accesso personale (ghp_…) — richiede l'ambito repo
GitHub EnterpriseToken di accesso personale — come GitHub
Bitbucket CloudPassword dell'app (Account → Password app)
Bitbucket ServerToken di accesso personale o token di accesso HTTP

HTTPS (git binary)

Utilizza l'eseguibile git del sistema per tutte le operazioni — clone locale completo, commit e push. Richiede Git for Windows (o qualsiasi binario git nel PATH). Supporta amend e squash. Usa gli stessi campi di nome utente e token della modalità HTTPS (integrato). Devi anche fornire il tuo Nome e E-mail per l'identità del commit.

SSH (file di chiave)

Seleziona SSH (file di chiave) e fornisci il percorso della tua chiave SSH privata e una passphrase opzionale. Questa modalità utilizza paramiko per una connessione SSH in Python puro (non è richiesto un client ssh di sistema). Se paramiko non è installato, MarkForge ricade sul client ssh di sistema.

pip install paramiko

L'autenticazione SSH supporta amend e squash.

Archiviazione delle credenziali

I token, le passphrase SSH e le password proxy sono archiviati nel portachiavi del sistema operativo (GNOME Keyring / KWallet su Linux, Keychain su macOS, Windows Credential Manager su Windows) — non in file di configurazione in testo normale.

Proxy HTTP

Se la tua rete richiede un proxy HTTP per raggiungere i server Git, configuralo in Visualizza → Impostazioni → Proxy HTTP:

  • URL del proxy — ad esempio http://proxy.company.com:8080
  • Nome utente del proxy — lascialo vuoto se non è richiesta autenticazione.
  • Password del proxy — lasciala vuota se non è richiesta autenticazione.

Lascia tutti i campi del proxy vuoti per utilizzare le impostazioni proxy predefinite del sistema.

Chiusura

Quando chiudi MarkForge mentre è aperto un file gestito da Git, MarkForge ti chiederà se eliminare la directory di clone temporanea. L'eliminazione della directory libera spazio su disco; mantenerla ti permette di ispezionare i file clonati in seguito.
