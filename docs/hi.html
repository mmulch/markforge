MarkForge उपयोगकर्ता मैनुअल v1.6.1

1. परिचय

MarkForge एक आधुनिक, सुविधा-संपन्न Markdown संपादक है जो Python और PyQt6 से बना है। यह एक साफ़, दो-पैनल वाले डेस्कटॉप अनुप्रयोग में बिना व्यवधान के लेखन अनुभव, रीयल-टाइम साथ-साथ पूर्वावलोकन, पूर्ण Git वर्कफ़्लो एकीकरण, डायग्राम समर्थन, गणित रेंडरिंग, और PDF आयात/निर्यात — सब कुछ एक साथ प्रदान करता है।

चाहे आप तकनीकी दस्तावेज़ीकरण लिख रहे हों, प्रोजेक्ट README बना रहे हों, या सीधे GitHub रिपॉज़िटरी में सामग्री प्रकाशित कर रहे हों — MarkForge बिना टर्मिनल खोले पूरे संपादन-और-प्रकाशन चक्र को संभालता है।

2. स्थापना

Linux (Debian / Ubuntu)

GitHub के Releases पृष्ठ से .deb पैकेज डाउनलोड करें, फिर इसे इस कमांड से स्थापित करें:

sudo apt install ./markforge_*.deb

पैकेज सभी आवश्यक सिस्टम-स्तरीय निर्भरताएँ (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments) स्वचालित रूप से स्थापित करता है। किसी virtual environment की आवश्यकता नहीं है।

Windows

Releases पृष्ठ से MarkForge-Setup.exe डाउनलोड करें और इंस्टॉलर चलाएँ। सेटअप विज़ार्ड आपको स्थापना के माध्यम से मार्गदर्शन करता है और स्वचालित रूप से स्टार्ट मेनू शॉर्टकट बनाता है।

नोट

Windows इंस्टॉलर में Python रनटाइम और सभी निर्भरताएँ शामिल हैं। अलग Python स्थापना की आवश्यकता नहीं है।

स्रोत से

आवश्यकताएँ: Python 3.8 या बाद का संस्करण, CMake 3.16 या बाद का संस्करण।

CMake का उपयोग करके (अनुशंसित):

# 1. Clone the repository
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Configure the build (creates a local virtual environment)
cmake -B build

# 3. Install Python dependencies into the local venv
cmake --build build --target install_deps

# 4. Run the application
cmake --build build --target run

सीधे Python से चलाना (CMake के बिना):

pip install -r requirements.txt
python src/main.py
सुझाव

SSH-आधारित Git संचालन के लिए, paramiko भी स्थापित करें: pip install paramiko

3. इंटरफ़ेस का अवलोकन

लेआउट

MarkForge विंडो तीन आकार-परिवर्तनीय पैनलों में विभाजित है जो खींचने योग्य स्प्लिटर से अलग किए गए हैं:

पैनलस्थितिविवरण
File Treeबाएँवर्तमान फ़ोल्डर में .md और चित्र फ़ाइलें ब्राउज़ करें। Ctrl+B से टॉगल करें।
संपादकबीच मेंMarkdown सिंटैक्स हाइलाइटिंग और लाइन नंबर के साथ सादा-पाठ संपादक।
पूर्वावलोकनदाएँवर्तमान दस्तावेज़ का लाइव HTML रेंडरिंग। Ctrl+Shift+P से टॉगल करें।

पैनल का आकार बदलने के लिए किसी भी स्प्लिटर हैंडल को खींचें। पैनल आकार सत्रों के बीच याद रखे जाते हैं।

टूलबार

मेनू बार के नीचे टूलबार सबसे सामान्य क्रियाओं तक एक-क्लिक पहुँच प्रदान करता है: नया, खोलें, सहेजें, लिंक सम्मिलित करें, चित्र सम्मिलित करें, तालिका सम्मिलित करें, PlantUML सम्मिलित करें, Mermaid सम्मिलित करें, पूर्वावलोकन टॉगल करें, और सेटिंग्स। सभी टूलबार क्रियाएँ मेनू आइटम और कीबोर्ड शॉर्टकट की नकल हैं।

स्टेटस बार

विंडो के निचले भाग में स्टेटस बार हमेशा दो जानकारियाँ दिखाता है:

  • शब्द गणना — दस्तावेज़ में कुल शब्दों की संख्या (टाइप करते समय लाइव अपडेट होती है)।
  • कर्सर स्थिति — वर्तमान पंक्ति और कॉलम संख्या, जैसे Line 12, Col 5।

मेनू बार

मेनू बार में पाँच शीर्ष-स्तरीय मेनू हैं:

मेनूसामग्री
File (फ़ाइल)New, Open, Open Folder, Import PDF, Open from Git, Git Squash, Save, Save As, Export as PDF, Quit
Edit (संपादन)Undo, Redo, Cut, Copy, Paste
Insert (सम्मिलित करें)Link, Image, PlantUML, Mermaid, Table
View (दृश्य)Show file tree, Show preview, Word wrap, Settings
Help (सहायता)Markdown reference, About

विंडो शीर्षक

विंडो शीर्षक वर्तमान में खुली फ़ाइल का नाम दिखाता है। जब आप सहेजे न गए परिवर्तन करते हैं, तो फ़ाइलनाम के बाद एक तारांकन (*) जुड़ जाता है, जैसे README.md *। जब कोई फ़ाइल नहीं खुली होती, तो शीर्षक Untitled दिखाता है।

4. संपादक

बीच का पैनल Markdown लेखन के लिए तैयार एक पूर्ण-सुविधा वाला सादा-पाठ संपादक होस्ट करता है।

सिंटैक्स हाइलाइटिंग

MarkForge टाइप करते समय दस्तावेज़ पर Markdown-अनुकूल सिंटैक्स हाइलाइटिंग लागू करता है। शीर्षक, जोर मार्कर, लिंक, कोड स्पैन, बाड़-घिरे कोड ब्लॉक, और अन्य Markdown संरचनाएँ अलग-अलग रंगों में दिखाई जाती हैं। जब आप Light और Dark संपादक थीम के बीच स्विच करते हैं तो हाइलाइटिंग स्वचालित रूप से अनुकूलित होती है।

लाइन नंबर

संपादक के बाईं ओर एक गटर लाइन नंबर प्रदर्शित करता है। वर्तमान पंक्ति एक सूक्ष्म पृष्ठभूमि रंग से हाइलाइट की जाती है जो आपको लंबे दस्तावेज़ों में अपनी स्थिति ट्रैक करने में मदद करती है।

Word Wrap

Word wrap डिफ़ॉल्ट रूप से सक्षम है। लंबी पंक्तियाँ दृश्यमान संपादक चौड़ाई में फिट होने के लिए लपेटती हैं। किसी भी समय View → Word wrap के माध्यम से Word wrap टॉगल करें। प्राथमिकता सत्रों के बीच सहेजी जाती है।

संपादक थीम

दो अंतर्निहित संपादक थीम उपलब्ध हैं:

  • Dark (डिफ़ॉल्ट) — उच्च-कंट्रास्ट सिंटैक्स रंगों के साथ गहरा पृष्ठभूमि।
  • Light — उज्ज्वल वातावरण के लिए उपयुक्त सफेद पृष्ठभूमि।

View → Settings → Editor theme में थीम बदलें। परिवर्तन पुनरारंभ के बिना तुरंत प्रभावी होता है।

प्लेसहोल्डर टेक्स्ट

जब कोई फ़ाइल नहीं खुली होती, तो संपादक अक्षम रहता है और यह संदेश दिखाता है: "Open or create a Markdown file to start editing." जैसे ही आप एक नई फ़ाइल बनाते हैं या कोई मौजूदा फ़ाइल खोलते हैं, संपादक सक्रिय हो जाता है।

संपादन संचालन

सभी मानक टेक्स्ट संपादन संचालन उपलब्ध हैं:

  • Ctrl+Z — Undo (पूर्ववत करें)
  • Ctrl+Y — Redo (फिर से करें)
  • Ctrl+X / Ctrl+C / Ctrl+V — Cut / Copy / Paste

खोजें & बदलें

खोज बार खोलने के लिए Ctrl+F दबाएँ, या बदलें पंक्ति के साथ खोलने के लिए Ctrl+H दबाएँ। बार संपादक के ऊपर प्रकट होती है।

  • खोज फ़ील्ड में टाइप करें — सभी मिलान संपादक में तुरंत हाइलाइट हो जाते हैं।
  • ↑ / ↓ बटन से मिलानों के बीच नेविगेट करें, या खोज फ़ील्ड में Enter (अगला) / Shift+Enter (पिछला) दबाएँ।
  • मिलान काउंटर वर्तमान स्थिति दिखाता है (जैसे 3 / 12)।
  • Aa — केस-संवेदनशील खोज चालू/बंद।
  • [W] — पूरे शब्द का मिलान चालू/बंद।
  • वर्तमान मिलान बदलने के लिए बदलें उपयोग करें, या एक साथ सभी मिलान बदलने के लिए सभी बदलें उपयोग करें।
  • बार बंद करने और सभी हाइलाइट हटाने के लिए Escape दबाएँ या ✕ पर क्लिक करें।

यदि बार खोलते समय संपादक में टेक्स्ट चुना हुआ है, तो खोज फ़ील्ड स्वतः उस टेक्स्ट से भर जाती है (केवल एकल-पंक्ति चयन के लिए)।

5. लाइव पूर्वावलोकन

दाएँ पैनल में आपके Markdown दस्तावेज़ का पूरी तरह रेंडर किया गया HTML संस्करण प्रदर्शित होता है, जो टाइप करते समय स्वचालित रूप से अपडेट होता है।

रेंडरिंग

पूर्वावलोकन साफ़, पठनीय आउटपुट के लिए GitHub-प्रेरित स्टाइलशीट का उपयोग करता है। यह सभी Markdown एक्सटेंशन का समर्थन करता है जिनमें तालिकाएँ, टास्क सूचियाँ, फ़ुटनोट, कोड हाइलाइटिंग, गणित, और डायग्राम शामिल हैं। रेंडरिंग पाइपलाइन आपके टेक्स्ट को Python markdown लाइब्रेरी के माध्यम से extra, codehilite, toc, और sane_lists एक्सटेंशन के साथ प्रोसेस करती है।

स्क्रॉल स्थिति संरक्षण

जब आप दस्तावेज़ संपादित करते हैं तो पूर्वावलोकन 300 ms की छोटी देरी के बाद अपडेट होता है। HTML रीफ्रेश करने से पहले, MarkForge वेब व्यू से वर्तमान स्क्रॉल स्थिति पढ़ता है और रेंडरिंग के बाद इसे पुनर्स्थापित करता है, ताकि जब आप दस्तावेज़ के बीच में काम कर रहे हों तो पूर्वावलोकन शीर्ष पर न जाए।

पूर्वावलोकन थीम

दो पूर्वावलोकन थीम उपलब्ध हैं:

  • Dark (डिफ़ॉल्ट) — समग्र अनुप्रयोग थीम से मेल खाती गहरी पृष्ठभूमि।
  • Light — प्रिंट जैसे पढ़ने के अनुभव के लिए सफेद पृष्ठभूमि।

View → Settings → Preview theme के माध्यम से बदलें। परिवर्तन तुरंत प्रभावी होता है।

पूर्वावलोकन टॉगल करना

पूर्वावलोकन पैनल छिपाने या दिखाने के लिए Ctrl+Shift+P दबाएँ या View → Show preview का उपयोग करें। पूर्वावलोकन छिपाने से संपादक को अधिक स्क्रीन स्थान मिलता है।

नोट

गणित सूत्र (MathJax) और Mermaid डायग्राम को पहले उपयोग पर इंटरनेट कनेक्शन की आवश्यकता होती है, क्योंकि लाइब्रेरी CDN से लोड की जाती हैं। PlantUML डायग्राम हमेशा plantuml.com से प्राप्त की जाती हैं और इंटरनेट एक्सेस की आवश्यकता होती है।

6. फ़ाइल संचालन

नई फ़ाइल

Ctrl+N दबाएँ या File → New चुनें। यदि वर्तमान दस्तावेज़ में सहेजे न गए परिवर्तन हैं, तो MarkForge आगे बढ़ने से पहले आपसे सहेजने का अनुरोध करेगा। एक नया खाली दस्तावेज़ खोला जाता है और संपादक सक्रिय होता है।

फ़ाइल खोलें

फ़ाइल ब्राउज़र डायलॉग खोलने के लिए Ctrl+O दबाएँ या File → Open… चुनें। Markdown फ़ाइलें (.md, .markdown), टेक्स्ट फ़ाइलें (.txt), और अन्य सभी फ़ाइलें चुनी जा सकती हैं। फ़ाइल ट्री खुली फ़ाइल वाले फ़ोल्डर को दिखाने के लिए अपडेट होगा।

फ़ोल्डर खोलें

एक डायरेक्टरी चुनने के लिए File → Open Folder… (Ctrl+Shift+O) चुनें। फ़ाइल ट्री तब उस डायरेक्टरी में सभी .md और चित्र फ़ाइलें दिखाता है। संपादक में खोलने के लिए ट्री में किसी भी फ़ाइल पर क्लिक करें।

सहेजें

Ctrl+S दबाएँ या File → Save चुनें। यदि फ़ाइल Git रिपॉज़िटरी से खोली गई थी, तो इसके बजाय Git commit & push डायलॉग दिखाई देगा (देखें Git के माध्यम से सहेजना)। स्थानीय फ़ाइलों के लिए, फ़ाइल तुरंत डिस्क पर लिखी जाती है और शीर्षक बार में तारांकन गायब हो जाता है।

इस रूप में सहेजें

वर्तमान दस्तावेज़ को नए स्थान या नए नाम से सहेजने के लिए Ctrl+Shift+S दबाएँ या File → Save As… चुनें।

File Tree

बाएँ पैनल में वर्तमान रूट डायरेक्टरी में .md फ़ाइलों और चित्रों की एक सपाट सूची प्रदर्शित होती है। सब-डायरेक्टरी जिनमें कोई सीधी संबंधित फ़ाइलें नहीं हैं, अव्यवस्था कम करने के लिए छिपाई जाती हैं। फ़ाइल खोलने के लिए उस पर क्लिक करें। जब Git-प्रबंधित फ़ाइल खुली होती है, तो ट्री हेडर हरे रंग में [GIT] reponame दिखाता है।

सुझाव

संपादक स्थान अधिकतम करने के लिए Ctrl+B से फ़ाइल ट्री टॉगल करें। आपकी प्राथमिकता सत्रों के बीच सहेजी जाती है।

8. गणित सूत्र

MarkForge MathJax का उपयोग करके LaTeX गणित नोटेशन रेंडर करता है। पहले उपयोग पर इंटरनेट कनेक्शन की आवश्यकता होती है (MathJax एक CDN से लोड किया जाता है)।

इनलाइन गणित

एक अभिव्यक्ति को एकल डॉलर चिह्नों में लपेटें ताकि इसे एक पैराग्राफ के भीतर इनलाइन रेंडर किया जा सके:

The famous equation $E = mc^2$ was published in 1905.

परिणाम: The famous equation E = mc² was published in 1905.

डिस्प्ले (ब्लॉक) गणित

अभिव्यक्ति को दोहरे डॉलर चिह्नों में लपेटें ताकि इसे अपनी पंक्ति पर केंद्रित रेंडर किया जा सके:

$$\int_0^\infty e^{-x}\, dx = 1$$

एकल-पंक्ति ब्लॉक गणित

आप एक डिस्प्ले-गणित ब्लॉक खोलने और बंद करने के लिए अपनी पंक्ति पर अकेले $ का भी उपयोग कर सकते हैं:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
सुझाव

MathJax पूर्ण LaTeX गणित मोड का समर्थन करता है, जिसमें भिन्न, मैट्रिसेस, ग्रीक अक्षर, ऑपरेटर, और कस्टम मैक्रो शामिल हैं। पूर्ण सिंटैक्स के लिए MathJax दस्तावेज़ीकरण देखें।

9. सम्मिलित करें डायलॉग

MarkForge सामान्य Markdown संरचनाएँ सम्मिलित करने के लिए गाइडेड डायलॉग प्रदान करता है। सभी डायलॉग संपादक में वर्तमान कर्सर स्थिति पर टेक्स्ट सम्मिलित करते हैं।

लिंक सम्मिलित करें Ctrl+K

Insert → Link… के माध्यम से खोलें या Ctrl+K दबाएँ।

डायलॉग में तीन फ़ील्ड हैं:

  • Display text (प्रदर्शन टेक्स्ट) — दृश्यमान लिंक लेबल (जैसे यहाँ क्लिक करें)।
  • URL — गंतव्य पता (जैसे https://example.com)।
  • Title (शीर्षक) (वैकल्पिक) — एक टूलटिप जो उपयोगकर्ता के लिंक पर होवर करने पर दिखाई देती है।

संपादक में [Display text](URL "Title") रखने के लिए Insert पर क्लिक करें।

चित्र सम्मिलित करें Ctrl+Shift+K

Insert → Image… के माध्यम से खोलें या Ctrl+Shift+K दबाएँ।

डायलॉग में दो फ़ील्ड हैं:

  • Alt Text (वैकल्पिक टेक्स्ट) — स्क्रीन रीडर के लिए एक संक्षिप्त विवरण।
  • URL / Path (पथ) — एक दूरस्थ URL या स्थानीय फ़ाइल पथ। स्थानीय चित्र फ़ाइल चुनने के लिए ब्राउज़ बटन का उपयोग करें।

संपादक में ![Alt text](URL) रखने के लिए Insert पर क्लिक करें।

तालिका सम्मिलित करें Ctrl+Shift+T

Insert → Table… के माध्यम से खोलें या Ctrl+Shift+T दबाएँ।

अपनी तालिका संरचना कॉन्फ़िगर करें:

  • Data rows (डेटा पंक्तियाँ) — हेडर पंक्ति को छोड़कर पंक्तियों की संख्या।
  • Columns (कॉलम) — कॉलमों की संख्या (अधिकतम समर्थित तक)।
  • Column names (कॉलम नाम) — प्रत्येक कॉलम के लिए एक शीर्षक दर्ज करें।
  • Alignment (संरेखण) — प्रत्येक कॉलम के लिए स्वतंत्र रूप से Left, Center, या Right चुनें।

डायलॉग के निचले भाग में एक लाइव Markdown पूर्वावलोकन अपडेट होता है। तालिका संपादक में रखने के लिए Insert पर क्लिक करें।

PlantUML सम्मिलित करें Ctrl+Shift+U

Insert → PlantUML… के माध्यम से खोलें या Ctrl+Shift+U दबाएँ।

ड्रॉपडाउन से 12 डायग्राम टेम्पलेट में से चुनें:

  • Sequence diagram (अनुक्रम डायग्राम)
  • Class diagram (क्लास डायग्राम)
  • Activity diagram (गतिविधि डायग्राम)
  • State diagram (स्थिति डायग्राम)
  • Component diagram (घटक डायग्राम)
  • Use case diagram (उपयोग केस डायग्राम)
  • Gantt diagram (गैंट डायग्राम)
  • Mind map (माइंड मैप)
  • WBS (Work breakdown — कार्य विभाजन)
  • Timing diagram (समय डायग्राम)
  • Deployment diagram (परिनियोजन डायग्राम)
  • JSON

चुना गया टेम्पलेट PlantUML कोड संपादक को भरता है। आवश्यकतानुसार कोड संपादित करें, फिर Insert पर क्लिक करें। डायग्राम एक बाड़-घिरे कोड ब्लॉक में लपेटा जाता है और पूर्वावलोकन में plantuml.com ऑनलाइन सेवा के माध्यम से रेंडर किया जाता है। इंटरनेट कनेक्शन आवश्यक है।

नोट

सम्मिलित करने के बाद मुख्य विंडो में डायग्राम पूर्वावलोकन स्वचालित रूप से दिखाई देता है। रेंडरिंग सार्वजनिक plantuml.com सर्वर का उपयोग करती है; कोई स्थानीय PlantUML स्थापना की आवश्यकता नहीं है।

Mermaid सम्मिलित करें Ctrl+Shift+M

Insert → Mermaid… के माध्यम से खोलें या Ctrl+Shift+M दबाएँ।

9 डायग्राम टेम्पलेट में से चुनें:

  • Flowchart (फ़्लोचार्ट)
  • Sequence diagram (अनुक्रम डायग्राम)
  • Class diagram (क्लास डायग्राम)
  • State diagram (स्थिति डायग्राम)
  • Gantt (गैंट)
  • Pie chart (पाई चार्ट)
  • Mindmap (माइंडमैप)
  • ER diagram (ER डायग्राम)
  • Git graph (Git ग्राफ़)

टेक्स्ट एरिया में Mermaid कोड संपादित करें और Insert पर क्लिक करें। डायग्राम एक बाड़-घिरे mermaid कोड ब्लॉक में लपेटा जाता है और kroki.io के माध्यम से पूर्वावलोकन में रेंडर किया जाता है। इंटरनेट कनेक्शन आवश्यक है।

10. PDF सुविधाएँ

PDF के रूप में निर्यात करें Ctrl+Shift+E

File → Export as PDF… चुनें या Ctrl+Shift+E दबाएँ। MarkForge लाइव पूर्वावलोकन के समान पाइपलाइन का उपयोग करके वर्तमान दस्तावेज़ को PDF फ़ाइल में रेंडर करता है, लेकिन सफेद पृष्ठभूमि और प्रिंट-अनुकूलित CSS के साथ। आउटपुट स्थान और फ़ाइलनाम चुनने के लिए एक फ़ाइल सहेजें डायलॉग दिखाई देता है।

सुझाव

सबसे साफ़ प्रिंट-तैयार आउटपुट के लिए PDF में निर्यात करने से पहले पूर्वावलोकन को Light थीम पर स्विच करें।

PDF आयात करें Ctrl+Shift+I

File → Import PDF… चुनें या Ctrl+Shift+I दबाएँ। MarkForge pymupdf4llm लाइब्रेरी का उपयोग करके किसी भी PDF फ़ाइल को Markdown में परिवर्तित करता है। रूपांतरण बुद्धिमानी से दस्तावेज़ संरचना को सुरक्षित रखता है:

  • शीर्षक और पैराग्राफ
  • तालिकाएँ
  • सूचियाँ
  • जहाँ पता चला, कोड ब्लॉक

रूपांतरण एक बैकग्राउंड थ्रेड में चलता है ताकि UI उत्तरदायी रहे। पूरा होने पर, MarkForge परिणामी .md फ़ाइल के लिए एक सहेजने का स्थान पूछता है और फिर इसे संपादक में लोड करता है।

नोट

PDF आयात के लिए pymupdf4llm पैकेज की आवश्यकता है। यदि यह स्थापित नहीं है, तो MarkForge आपको इसे स्थापित करने का संकेत देगा: pip install pymupdf4llm

11. Git एकीकरण

MarkForge होस्टेड Git रिपॉज़िटरी से सीधे Markdown फ़ाइलें खोल सकता है, उन्हें संपादित करने दे सकता है, और बिना कभी टर्मिनल खोले परिवर्तन वापस push कर सकता है। amend और squash जैसे उन्नत संचालन भी समर्थित हैं।

समर्थित प्लेटफ़ॉर्म

प्लेटफ़ॉर्मउदाहरण URL पैटर्न
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.mycompany.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.mycompany.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

प्लेटफ़ॉर्म URL संरचना से स्वतः-पहचाना जाता है, न केवल होस्टनाम से — इसलिए स्व-होस्टेड इंस्टेंस स्वचालित रूप से काम करते हैं।

Git से फ़ाइल खोलना

  1. File → Open from Git… चुनें या Ctrl+Shift+G दबाएँ।
  2. जो फ़ाइल आप खोलना चाहते हैं उसका URL पेस्ट करें (इसे GitHub या Bitbucket फ़ाइल ब्राउज़र से कॉपी करें)।
  3. MarkForge URL को मान्य करता है और एक पूर्वावलोकन पंक्ति दिखाता है: platform · owner/repo · path/to/file.md।
  4. वैकल्पिक रूप से Ref फ़ील्ड में branch, tag, या commit hash ओवरराइड करें।
  5. OK पर क्लिक करें — रिपॉज़िटरी बैकग्राउंड में clone हो जाती है और फ़ाइल संपादक में खुलती है।

जब Git-प्रबंधित फ़ाइल खुली होती है, तो फ़ाइल ट्री हेडर हरे रंग में [GIT] reponame दिखाता है।

सुझाव

सार्वजनिक रिपॉज़िटरी बिना किसी क्रेडेंशियल के काम करती हैं। निजी रिपॉज़िटरी के लिए, खोलने से पहले View → Settings → Git Authentication में अपने क्रेडेंशियल कॉन्फ़िगर करें।

सहेजना (Commit और Push)

जब Git-प्रबंधित फ़ाइल खुली होती है, तो Ctrl+S दबाने से सामान्य फ़ाइल सहेजने की बजाय Git Commit & Push डायलॉग खुलता है। डायलॉग में शामिल हैं:

  • Commit message (commit संदेश) — आवश्यक। अपने परिवर्तनों का एक अर्थपूर्ण विवरण दर्ज करें।
  • Amend previous commit — SSH या git-binary प्रमाणीकरण का उपयोग करते समय दूसरे सहेजने से उपलब्ध। नया commit बनाने के बजाय पिछले commit को बदलता है; पिछले commit संदेश को पहले से भरता है; स्वचालित रूप से force-push (--force-with-lease) का उपयोग करता है। केवल वर्तमान सत्र में MarkForge द्वारा किए गए commit को amend किया जा सकता है।
  • Push to (इसमें push करें):
    • Current branch — जिस branch से आपने clone किया उस पर सीधे push करता है।
    • New branch — आपके द्वारा निर्दिष्ट नाम से एक नई branch बनाता है।
  • Create Pull Request — "New branch" चुने जाने पर उपलब्ध। Push करने के बाद स्वचालित रूप से एक pull request खोलता है। PR शीर्षक और लक्ष्य branch के लिए फ़ील्ड दिखाता है।

Squash Commits Ctrl+Shift+Q

केवल SSH और HTTPS (git binary) प्रमाणीकरण के साथ उपलब्ध।

File → Git Squash… चुनें या Ctrl+Shift+Q दबाएँ। Squash Commits डायलॉग वर्तमान branch पर सभी commits की एक तालिका दिखाता है जो चुने गए आधार branch में अभी तक नहीं हैं।

  • Base branch field (आधार branch फ़ील्ड) — संदर्भ branch (जैसे main, master, develop)। इसे बदलने के बाद Reload पर क्लिक करें।
  • Select all new commits — एक साथ सभी सूचीबद्ध commits चेक करता है।
  • Squash commit message — एकल संदेश जो सभी चुने गए commits को बदल देगा।
  • Squash & Push — branch इतिहास को फिर से लिखता है और remote पर force-push करता है।

आपको कम से कम 2 commits चुनने होंगे, और वे सबसे हालिया commit (HEAD) से शुरू होकर एक सतत श्रृंखला बनाने चाहिए।

प्रमाणीकरण मोड

View → Settings → Git Authentication में अपना प्रमाणीकरण तरीका कॉन्फ़िगर करें।

HTTPS (embedded)

बिना किसी अतिरिक्त सॉफ़्टवेयर के सभी प्लेटफ़ॉर्म पर काम करता है। प्लेटफ़ॉर्म REST API का सीधे उपयोग करता है। कोई स्थानीय git binary की आवश्यकता नहीं है।

सीमा

amend और squash सुविधाएँ इस मोड में उपलब्ध नहीं हैं क्योंकि कोई पूर्ण स्थानीय git इतिहास नहीं रखा जाता।

प्लेटफ़ॉर्मटोकन प्रकार
GitHubPersonal access token (ghp_…) — repo scope आवश्यक
GitHub EnterprisePersonal access token — GitHub जैसा ही
Bitbucket CloudApp password (Account → App passwords)
Bitbucket ServerPersonal access token या HTTP access token

HTTPS (git binary)

सभी संचालन के लिए सिस्टम git executable का उपयोग करता है — पूर्ण स्थानीय clone, commit, और push। Git for Windows (या आपके PATH पर कोई git binary) की आवश्यकता है। amend और squash का समर्थन करता है। HTTPS (embedded) मोड की तरह ही username और token फ़ील्ड का उपयोग करें। commit पहचान के लिए आपको अपना Name और Email भी देना होगा।

SSH (key file)

SSH (key file) चुनें और अपनी निजी SSH key और एक वैकल्पिक passphrase का पथ दें। यह मोड एक शुद्ध-Python SSH कनेक्शन के लिए paramiko का उपयोग करता है (कोई सिस्टम ssh client की आवश्यकता नहीं)। यदि paramiko स्थापित नहीं है, तो MarkForge सिस्टम ssh client पर वापस आ जाता है।

pip install paramiko

SSH प्रमाणीकरण amend और squash का समर्थन करता है।

क्रेडेंशियल संग्रहण

टोकन, SSH पासफ़्रेज़ और प्रॉक्सी पासवर्ड OS कीरिंग में संग्रहीत होते हैं (Linux पर GNOME Keyring / KWallet, macOS पर Keychain, Windows पर Windows Credential Manager) — सादे-पाठ कॉन्फ़िगरेशन फ़ाइलों में नहीं।

HTTP Proxy

यदि आपके नेटवर्क को Git सर्वर तक पहुँचने के लिए HTTP proxy की आवश्यकता है, तो इसे View → Settings → HTTP Proxy में कॉन्फ़िगर करें:

  • Proxy URL — जैसे http://proxy.company.com:8080
  • Proxy username — यदि कोई प्रमाणीकरण आवश्यक नहीं है तो खाली छोड़ें।
  • Proxy password — यदि कोई प्रमाणीकरण आवश्यक नहीं है तो खाली छोड़ें।

सिस्टम के डिफ़ॉल्ट proxy सेटिंग्स का उपयोग करने के लिए सभी proxy फ़ील्ड खाली छोड़ें।

बंद करना

जब आप Git-प्रबंधित फ़ाइल खुली होने पर MarkForge बंद करते हैं, तो MarkForge पूछेगा कि क्या अस्थायी clone डायरेक्टरी हटाई जाए। डायरेक्टरी हटाने से डिस्क स्थान खाली होता है; इसे रखने से आप बाद में clone की गई फ़ाइलों का निरीक्षण कर सकते हैं।
