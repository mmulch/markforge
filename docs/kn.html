1. ಪರಿಚಯ

MarkForge ಎಂಬುದು Python ಮತ್ತು PyQt6 ಬಳಸಿ ನಿರ್ಮಿಸಲಾದ ಆಧುನಿಕ, ಸಮೃದ್ಧ ವೈಶಿಷ್ಟ್ಯಗಳುಳ್ಳ Markdown ಸಂಪಾದಕ. ಇದು ಏಕಾಗ್ರತೆಯ ಬರವಣಿಗೆ ಅನುಭವವನ್ನು ನೈಜ-ಸಮಯದ ಪಕ್ಕದ ಮುನ್ನೋಟದೊಂದಿಗೆ, ಸಂಪೂರ್ಣ Git ವರ್ಕ್‌ಫ್ಲೋ ಸಮನ್ವಯ, ರೇಖಾಚಿತ್ರ ಬೆಂಬಲ, ಗಣಿತ ರೆಂಡರಿಂಗ್, ಮತ್ತು PDF ಆಮದು/ರಫ್ತು — ಎಲ್ಲವನ್ನೂ ಒಂದೇ ಸ್ವಚ್ಛ, ಎರಡು-ಫಲಕ ಡೆಸ್ಕ್‌ಟಾಪ್ ಅಪ್ಲಿಕೇಶನ್‌ನಲ್ಲಿ ಸಂಯೋಜಿಸುತ್ತದೆ.

ತಾಂತ್ರಿಕ ದಾಖಲೆ ಬರೆಯುವಾಗ, ಯೋಜನೆಯ README ಗಳು, ಅಥವಾ GitHub ರೆಪೊಸಿಟರಿಗೆ ನೇರವಾಗಿ ವಿಷಯ ಪ್ರಕಟಿಸುವಾಗ — MarkForge ಟರ್ಮಿನಲ್ ಅಗತ್ಯವಿಲ್ಲದೆ ಸಂಪೂರ್ಣ ಸಂಪಾದನೆ-ಮತ್ತು-ಪ್ರಕಟಣೆ ಚಕ್ರವನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ.

2. ಸ್ಥಾಪನೆ

Linux (Debian / Ubuntu)

GitHub ನ Releases ಪುಟದಿಂದ .deb ಪ್ಯಾಕೇಜ್ ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ, ನಂತರ ಈ ಕೆಳಗಿನಂತೆ ಸ್ಥಾಪಿಸಿ:

sudo apt install ./markforge_*.deb

ಈ ಪ್ಯಾಕೇಜ್ ಎಲ್ಲಾ ಅಗತ್ಯ ಸಿಸ್ಟಮ್-ಮಟ್ಟದ ಅವಲಂಬನೆಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸ್ಥಾಪಿಸುತ್ತದೆ (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments). ಯಾವುದೇ virtual environment ಅಗತ್ಯವಿಲ್ಲ.

Windows

Releases ಪುಟದಿಂದ MarkForge-Setup.exe ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ ಮತ್ತು ಇನ್‌ಸ್ಟಾಲರ್ ಚಲಾಯಿಸಿ. ಸೆಟಪ್ ವಿಝಾರ್ಡ್ ಸ್ಥಾಪನಾ ಪ್ರಕ್ರಿಯೆಯ ಮೂಲಕ ನಿಮ್ಮನ್ನು ಮಾರ್ಗದರ್ಶಿಸುತ್ತದೆ ಮತ್ತು ಸ್ಟಾರ್ಟ್ ಮೆನು ಶಾರ್ಟ್‌ಕಟ್‌ಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸುತ್ತದೆ.

ಗಮನಿಸಿ

Windows ಇನ್‌ಸ್ಟಾಲರ್ Python runtime ಮತ್ತು ಎಲ್ಲಾ ಅವಲಂಬನೆಗಳನ್ನು ಒಳಗೊಂಡಿದೆ. ಪ್ರತ್ಯೇಕ Python ಸ್ಥಾಪನೆ ಅಗತ್ಯವಿಲ್ಲ.

ಮೂಲದಿಂದ

ಅವಶ್ಯಕತೆಗಳು: Python 3.8 ಅಥವಾ ನಂತರದ ಆವೃತ್ತಿ, CMake 3.16 ಅಥವಾ ನಂತರದ ಆವೃತ್ತಿ.

CMake ಬಳಸಿ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ):

# 1. Clone the repository
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Configure the build (creates a local virtual environment)
cmake -B build

# 3. Install Python dependencies into the local venv
cmake --build build --target install_deps

# 4. Run the application
cmake --build build --target run

ನೇರ Python ಉಡಾವಣೆ (CMake ಇಲ್ಲದೆ):

pip install -r requirements.txt
python src/main.py
ಸಲಹೆ

SSH-ಆಧಾರಿತ Git ಕಾರ್ಯಾಚರಣೆಗಳಿಗಾಗಿ, paramiko ಸಹ ಸ್ಥಾಪಿಸಿ: pip install paramiko

3. ಇಂಟರ್‌ಫೇಸ್ ಅವಲೋಕನ

ವಿನ್ಯಾಸ

MarkForge ವಿಂಡೋ ಎಳೆಯಬಹುದಾದ ಸ್ಪ್ಲಿಟರ್‌ಗಳಿಂದ ಬೇರ್ಪಟ್ಟ ಮೂರು ಗಾತ್ರ ಬದಲಾಯಿಸಬಹುದಾದ ಫಲಕಗಳಾಗಿ ವಿಂಗಡಿಸಲಾಗಿದೆ:

ಫಲಕಸ್ಥಾನವಿವರಣೆ
ಫೈಲ್ ಟ್ರೀಎಡಪ್ರಸ್ತುತ ಫೋಲ್ಡರ್‌ನಲ್ಲಿ .md ಮತ್ತು ಚಿತ್ರ ಫೈಲ್‌ಗಳನ್ನು ಬ್ರೌಸ್ ಮಾಡಿ. Ctrl+B ಯಿಂದ ಟಾಗಲ್ ಮಾಡಿ.
ಸಂಪಾದಕಮಧ್ಯMarkdown syntax highlighting ಮತ್ತು ಸಾಲು ಸಂಖ್ಯೆಗಳೊಂದಿಗೆ ಸಾದಾ-ಪಠ್ಯ ಸಂಪಾದಕ.
ಮುನ್ನೋಟಬಲಪ್ರಸ್ತುತ ದಾಖಲೆಯ ನೇರ HTML ರೆಂಡರಿಂಗ್. Ctrl+Shift+P ಯಿಂದ ಟಾಗಲ್ ಮಾಡಿ.

ಫಲಕಗಳ ಗಾತ್ರ ಬದಲಾಯಿಸಲು ಯಾವುದೇ ಸ್ಪ್ಲಿಟರ್ ಹ್ಯಾಂಡಲ್ ಎಳೆಯಿರಿ. ಫಲಕ ಗಾತ್ರಗಳನ್ನು ಸೆಶನ್‌ಗಳ ನಡುವೆ ನೆನಪಿನಲ್ಲಿಡಲಾಗುತ್ತದೆ.

ಟೂಲ್‌ಬಾರ್

ಮೆನು ಪಟ್ಟಿಯ ಕೆಳಗಿರುವ ಟೂಲ್‌ಬಾರ್ ಅತ್ಯಂತ ಸಾಮಾನ್ಯ ಕ್ರಿಯೆಗಳಿಗೆ ಒಂದು-ಕ್ಲಿಕ್ ಪ್ರವೇಶ ನೀಡುತ್ತದೆ: ಹೊಸದು, ತೆರೆ, ಉಳಿಸಿ, ಲಿಂಕ್ ಸೇರಿಸಿ, ಚಿತ್ರ ಸೇರಿಸಿ, ಕೋಷ್ಟಕ ಸೇರಿಸಿ, PlantUML ಸೇರಿಸಿ, Mermaid ಸೇರಿಸಿ, ಮುನ್ನೋಟ ಟಾಗಲ್, ಮತ್ತು ಸೆಟ್ಟಿಂಗ್‌ಗಳು. ಎಲ್ಲಾ ಟೂಲ್‌ಬಾರ್ ಕ್ರಿಯೆಗಳು ಮೆನು ಐಟಂ ಮತ್ತು ಕೀಬೋರ್ಡ್ ಶಾರ್ಟ್‌ಕಟ್‌ಗಳನ್ನು ನಕಲಿಸುತ್ತವೆ.

ಸ್ಥಿತಿ ಪಟ್ಟಿ

ವಿಂಡೋದ ಕೆಳಭಾಗದಲ್ಲಿರುವ ಸ್ಥಿತಿ ಪಟ್ಟಿ ಯಾವಾಗಲೂ ಎರಡು ಮಾಹಿತಿ ತೋರಿಸುತ್ತದೆ:

  • ಪದ ಎಣಿಕೆ — ದಾಖಲೆಯಲ್ಲಿನ ಒಟ್ಟು ಪದಗಳ ಸಂಖ್ಯೆ (ನೀವು ಟೈಪ್ ಮಾಡುತ್ತಿದ್ದಂತೆ ನೇರವಾಗಿ ನವೀಕರಣ ಆಗುತ್ತದೆ).
  • ಕರ್ಸರ್ ಸ್ಥಾನ — ಪ್ರಸ್ತುತ ಸಾಲು ಮತ್ತು ಕಾಲಮ್ ಸಂಖ್ಯೆ, ಉದಾ. ಸಾಲು 12, ಕಾಲಮ್ 5.

ವಿಂಡೋ ಶೀರ್ಷಿಕೆ

ವಿಂಡೋ ಶೀರ್ಷಿಕೆ ಪ್ರಸ್ತುತ ತೆರೆದ ಫೈಲ್‌ನ ಹೆಸರನ್ನು ತೋರಿಸುತ್ತದೆ. ನೀವು ಉಳಿಸದ ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡಿದಾಗ, ಫೈಲ್‌ನಾಮ್‌ಗೆ ನಕ್ಷತ್ರ (*) ಸೇರಿಸಲಾಗುತ್ತದೆ, ಉದಾ. README.md *. ಯಾವುದೇ ಫೈಲ್ ತೆರೆಯದ್ದಾಗ ಶೀರ್ಷಿಕೆ Untitled ತೋರಿಸುತ್ತದೆ.

4. ಸಂಪಾದಕ

ಮಧ್ಯದ ಫಲಕ Markdown ಬರವಣಿಗೆಗಾಗಿ ಹೊಂದಿಸಲಾದ ಸಂಪೂರ್ಣ ವೈಶಿಷ್ಟ್ಯಗಳ ಸಾದಾ-ಪಠ್ಯ ಸಂಪಾದಕವನ್ನು ಹೊಂದಿದೆ.

Syntax Highlighting

MarkForge ನೀವು ಟೈಪ್ ಮಾಡುತ್ತಿದ್ದಂತೆ ದಾಖಲೆಗೆ Markdown-ಅರಿವಿನ syntax highlighting ಅನ್ವಯಿಸುತ್ತದೆ. ಶೀರ್ಷಿಕೆಗಳು, ಒತ್ತು ಗುರುತುಗಳು, ಲಿಂಕ್‌ಗಳು, ಕೋಡ್ ಸ್ಪ್ಯಾನ್‌ಗಳು, ಫೆನ್ಸ್ಡ್ ಕೋಡ್ ಬ್ಲಾಕ್‌ಗಳು, ಮತ್ತು ಇತರ Markdown ರಚನೆಗಳನ್ನು ವಿಭಿನ್ನ ಬಣ್ಣಗಳಲ್ಲಿ ತೋರಿಸಲಾಗುತ್ತದೆ. ನೀವು Light ಮತ್ತು Dark ಸಂಪಾದಕ ಥೀಮ್‌ಗಳ ನಡುವೆ ಬದಲಾಯಿಸಿದಾಗ highlighting ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಹೊಂದಿಕೊಳ್ಳುತ್ತದೆ.

ಸಾಲು ಸಂಖ್ಯೆಗಳು

ಸಂಪಾದಕದ ಎಡ ಭಾಗದಲ್ಲಿರುವ gutter ಸಾಲು ಸಂಖ್ಯೆಗಳನ್ನು ತೋರಿಸುತ್ತದೆ. ದೀರ್ಘ ದಾಖಲೆಗಳಲ್ಲಿ ನಿಮ್ಮ ಸ್ಥಾನ ಟ್ರ್ಯಾಕ್ ಮಾಡಲು ಸಹಾಯ ಮಾಡಲು ಪ್ರಸ್ತುತ ಸಾಲನ್ನು ಸೂಕ್ಷ್ಮ ಹಿನ್ನೆಲೆ ಬಣ್ಣದಿಂದ ಹೈಲೈಟ್ ಮಾಡಲಾಗಿದೆ.

ಪದ ಮಡಿಕೆ

ಪದ ಮಡಿಕೆ ಡಿಫಾಲ್ಟ್ ಆಗಿ ಸಕ್ರಿಯಗೊಂಡಿರುತ್ತದೆ. ದೀರ್ಘ ಸಾಲುಗಳು ಗೋಚರ ಸಂಪಾದಕ ಅಗಲಕ್ಕೆ ಮಡಿಕೆ ಆಗುತ್ತವೆ. ನೋಟ → ಪದ ಮಡಿಕೆ ಮೂಲಕ ಯಾವುದೇ ಸಮಯದಲ್ಲಿ ಪದ ಮಡಿಕೆ ಟಾಗಲ್ ಮಾಡಿ. ಈ ಆದ್ಯತೆ ಸೆಶನ್‌ಗಳ ನಡುವೆ ಉಳಿಸಲಾಗುತ್ತದೆ.

ಸಂಪಾದಕ ಥೀಮ್‌ಗಳು

ಎರಡು ಅಂತರ್ನಿರ್ಮಿತ ಸಂಪಾದಕ ಥೀಮ್‌ಗಳು ಲಭ್ಯವಿವೆ:

  • Dark (ಡಿಫಾಲ್ಟ್) — ಹೆಚ್ಚಿನ-ಕಾಂಟ್ರಾಸ್ಟ್ syntax ಬಣ್ಣಗಳೊಂದಿಗೆ ಡಾರ್ಕ್ ಹಿನ್ನೆಲೆ.
  • Light — ಪ್ರಕಾಶಮಾನ ಪರಿಸರಕ್ಕೆ ಸೂಕ್ತವಾದ ಬಿಳಿ ಹಿನ್ನೆಲೆ.

ನೋಟ → ಸೆಟ್ಟಿಂಗ್‌ಗಳು → ಸಂಪಾದಕ ಥೀಮ್ ನಲ್ಲಿ ಥೀಮ್ ಬದಲಾಯಿಸಿ. ಬದಲಾವಣೆ ಮರಾರ್ಟ್ ಇಲ್ಲದೆ ತಕ್ಷಣ ಜಾರಿಗೆ ಬರುತ್ತದೆ.

ಪ್ಲೇಸ್‌ಹೋಲ್ಡರ್ ಪಠ್ಯ

ಯಾವುದೇ ಫೈಲ್ ತೆರೆಯದ್ದಾಗ, ಸಂಪಾದಕ ನಿಷ್ಕ್ರಿಯಗೊಳ್ಳುತ್ತದೆ ಮತ್ತು ಈ ಸಂದೇಶ ತೋರಿಸುತ್ತದೆ: "ಸಂಪಾದನೆ ಪ್ರಾರಂಭಿಸಲು Markdown ಫೈಲ್ ತೆರೆಯಿರಿ ಅಥವಾ ರಚಿಸಿ." ನೀವು ಹೊಸ ಫೈಲ್ ರಚಿಸಿದ ತಕ್ಷಣ ಅಥವಾ ಅಸ್ತಿತ್ವದಲ್ಲಿರುವದನ್ನು ತೆರೆದ ತಕ್ಷಣ ಸಂಪಾದಕ ಸಕ್ರಿಯಗೊಳ್ಳುತ್ತದೆ.

ಸಂಪಾದನಾ ಕಾರ್ಯಾಚರಣೆಗಳು

ಎಲ್ಲಾ ಸ್ಟ್ಯಾಂಡರ್ಡ್ ಪಠ್ಯ ಸಂಪಾದನಾ ಕಾರ್ಯಾಚರಣೆಗಳು ಲಭ್ಯವಿವೆ:

  • Ctrl+Z — Undo
  • Ctrl+Y — Redo
  • Ctrl+X / Ctrl+C / Ctrl+V — ಕಟ್ / ಕಾಪಿ / ಪೇಸ್ಟ್

5. ನೇರ ಮುನ್ನೋಟ

ಬಲ ಫಲಕ ನಿಮ್ಮ Markdown ದಾಖಲೆಯ ಸಂಪೂರ್ಣ ರೆಂಡರ್ ಆದ HTML ಆವೃತ್ತಿಯನ್ನು ತೋರಿಸುತ್ತದೆ, ನೀವು ಟೈಪ್ ಮಾಡುತ್ತಿದ್ದಂತೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ನವೀಕರಿಸಲ್ಪಡುತ್ತದೆ.

ರೆಂಡರಿಂಗ್

ಮುನ್ನೋಟ ಸ್ವಚ್ಛ, ಓದಲು ಸುಲಭವಾದ ಔಟ್‌ಪುಟ್‌ಗಾಗಿ GitHub-ಪ್ರೇರಿತ stylesheet ಬಳಸುತ್ತದೆ. ಇದು ಕೋಷ್ಟಕಗಳು, ಟಾಸ್ಕ್ ಪಟ್ಟಿಗಳು, ಅಡಿಟಿಪ್ಪಣಿಗಳು, ಕೋಡ್ ಹೈಲೈಟಿಂಗ್, ಗಣಿತ ಮತ್ತು ರೇಖಾಚಿತ್ರಗಳ ಸೇರಿದಂತೆ ಎಲ್ಲಾ Markdown ವಿಸ್ತರಣೆಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ರೆಂಡರಿಂಗ್ ಪೈಪ್‌ಲೈನ್ extra, codehilite, toc, ಮತ್ತು sane_lists ವಿಸ್ತರಣೆಗಳೊಂದಿಗೆ Python markdown ಲೈಬ್ರರಿ ಮೂಲಕ ಪಠ್ಯ ಪ್ರಕ್ರಿಯೆ ಮಾಡುತ್ತದೆ.

ಸ್ಕ್ರೋಲ್ ಸ್ಥಾನ ಸಂರಕ್ಷಣೆ

ನೀವು ದಾಖಲೆ ಸಂಪಾದಿಸಿದಾಗ ಮುನ್ನೋಟ 300 ms ಡಿಬೌನ್ಸ್ ವಿಳಂಬದ ನಂತರ ನವೀಕರಣಗೊಳ್ಳುತ್ತದೆ. HTML ರಿಫ್ರೆಶ್ ಮಾಡುವ ಮೊದಲು, MarkForge ವೆಬ್ ವ್ಯೂನಿಂದ ಪ್ರಸ್ತುತ ಸ್ಕ್ರೋಲ್ ಸ್ಥಾನ ಓದುತ್ತದೆ ಮತ್ತು ರೆಂಡರಿಂಗ್ ನಂತರ ಅದನ್ನು ಮರುಸ್ಥಾಪಿಸುತ್ತದೆ, ಇದರಿಂದ ನೀವು ದಾಖಲೆಯ ಮಧ್ಯದಲ್ಲಿ ಕೆಲಸ ಮಾಡುತ್ತಿರುವಾಗ ಮುನ್ನೋಟ ಮೇಲ್ಭಾಗಕ್ಕೆ ಜಿಗಿಯುವುದಿಲ್ಲ.

ಮುನ್ನೋಟ ಥೀಮ್‌ಗಳು

ಎರಡು ಮುನ್ನೋಟ ಥೀಮ್‌ಗಳು ಲಭ್ಯವಿವೆ:

  • Dark (ಡಿಫಾಲ್ಟ್) — ಒಟ್ಟಾರೆ ಅಪ್ಲಿಕೇಶನ್ ಥೀಮ್‌ಗೆ ಹೊಂದುವ ಡಾರ್ಕ್ ಹಿನ್ನೆಲೆ.
  • Light — ಮುದ್ರಣದಂತಹ ಓದುವ ಅನುಭವಕ್ಕಾಗಿ ಬಿಳಿ ಹಿನ್ನೆಲೆ.

ನೋಟ → ಸೆಟ್ಟಿಂಗ್‌ಗಳು → ಮುನ್ನೋಟ ಥೀಮ್ ಮೂಲಕ ಬದಲಾಯಿಸಿ. ಬದಲಾವಣೆ ತಕ್ಷಣ ಜಾರಿಗೆ ಬರುತ್ತದೆ.

ಮುನ್ನೋಟ ಟಾಗಲ್ ಮಾಡುವುದು

ಮುನ್ನೋಟ ಫಲಕ ಮರೆಮಾಡಲು ಅಥವಾ ತೋರಿಸಲು Ctrl+Shift+P ಒತ್ತಿರಿ ಅಥವಾ ನೋಟ → ಮುನ್ನೋಟ ತೋರಿಸಿ ಬಳಸಿ. ಮುನ್ನೋಟ ಮರೆಮಾಡಿದರೆ ಸಂಪಾದಕಕ್ಕೆ ಹೆಚ್ಚಿನ ಸ್ಕ್ರೀನ್ ಜಾಗ ದೊರೆಯುತ್ತದೆ.

ಗಮನಿಸಿ

ಗಣಿತ ಸೂತ್ರಗಳು (MathJax) ಮತ್ತು Mermaid ರೇಖಾಚಿತ್ರಗಳಿಗೆ ಮೊದಲ ಬಳಕೆಯಲ್ಲಿ ಇಂಟರ್ನೆಟ್ ಸಂಪರ್ಕ ಅಗತ್ಯ, ಏಕೆಂದರೆ ಲೈಬ್ರರಿಗಳನ್ನು CDN ನಿಂದ ಲೋಡ್ ಮಾಡಲಾಗುತ್ತದೆ. PlantUML ರೇಖಾಚಿತ್ರಗಳನ್ನು ಯಾವಾಗಲೂ plantuml.com ನಿಂದ ತರಲಾಗುತ್ತದೆ ಮತ್ತು ಇಂಟರ್ನೆಟ್ ಪ್ರವೇಶ ಅಗತ್ಯ.

6. ಫೈಲ್ ಕಾರ್ಯಾಚರಣೆಗಳು

ಹೊಸ ಫೈಲ್

Ctrl+N ಒತ್ತಿರಿ ಅಥವಾ ಫೈಲ್ → ಹೊಸದು ಆರಿಸಿ. ಪ್ರಸ್ತುತ ದಾಖಲೆಯಲ್ಲಿ ಉಳಿಸದ ಬದಲಾವಣೆಗಳಿದ್ದರೆ, MarkForge ಮುಂದುವರಿಯುವ ಮೊದಲು ಉಳಿಸಲು ಕೇಳುತ್ತದೆ. ಹೊಸ ಖಾಲಿ ದಾಖಲೆ ತೆರೆಯಲ್ಪಡುತ್ತದೆ ಮತ್ತು ಸಂಪಾದಕ ಸಕ್ರಿಯಗೊಳ್ಳುತ್ತದೆ.

ಫೈಲ್ ತೆರೆ

ಫೈಲ್ ಬ್ರೌಸರ್ ಸಂವಾದ ತೆರೆಯಲು Ctrl+O ಒತ್ತಿರಿ ಅಥವಾ ಫೈಲ್ → ತೆರೆ… ಆರಿಸಿ. Markdown ಫೈಲ್‌ಗಳು (.md, .markdown), ಪಠ್ಯ ಫೈಲ್‌ಗಳು (.txt), ಮತ್ತು ಎಲ್ಲಾ ಇತರ ಫೈಲ್‌ಗಳನ್ನು ಆಯ್ಕೆ ಮಾಡಬಹುದು. ತೆರೆದ ಫೈಲ್ ಇರುವ ಫೋಲ್ಡರ್ ತೋರಿಸಲು ಫೈಲ್ ಟ್ರೀ ನವೀಕರಣಗೊಳ್ಳುತ್ತದೆ.

ಫೋಲ್ಡರ್ ತೆರೆ

ಡೈರೆಕ್ಟರಿ ಆಯ್ಕೆ ಮಾಡಲು ಫೈಲ್ → ಫೋಲ್ಡರ್ ತೆರೆ… (Ctrl+Shift+O) ಆರಿಸಿ. ಆ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿನ ಎಲ್ಲಾ .md ಮತ್ತು ಚಿತ್ರ ಫೈಲ್‌ಗಳನ್ನು ಫೈಲ್ ಟ್ರೀ ತೋರಿಸುತ್ತದೆ. ಸಂಪಾದಕದಲ್ಲಿ ತೆರೆಯಲು ಟ್ರೀನಲ್ಲಿ ಯಾವುದೇ ಫೈಲ್ ಕ್ಲಿಕ್ ಮಾಡಿ.

ಉಳಿಸಿ

Ctrl+S ಒತ್ತಿರಿ ಅಥವಾ ಫೈಲ್ → ಉಳಿಸಿ ಆರಿಸಿ. ಫೈಲ್ Git ರೆಪೊಸಿಟರಿಯಿಂದ ತೆರೆಯಲ್ಪಟ್ಟಿದ್ದರೆ, ಬದಲಿಗೆ Git commit & push ಸಂವಾದ ಕಾಣಿಸಿಕೊಳ್ಳುತ್ತದೆ (ನೋಡಿ Git ಮೂಲಕ ಉಳಿಸುವುದು). ಸ್ಥಳೀಯ ಫೈಲ್‌ಗಳಿಗೆ, ಫೈಲ್ ತಕ್ಷಣ ಡಿಸ್ಕ್‌ಗೆ ಬರೆಯಲ್ಪಡುತ್ತದೆ ಮತ್ತು ಶೀರ್ಷಿಕೆ ಪಟ್ಟಿಯ ನಕ್ಷತ್ರ ಮಾಯವಾಗುತ್ತದೆ.

ಹೆಸರಿನಡಿ ಉಳಿಸಿ

ಪ್ರಸ್ತುತ ದಾಖಲೆಯನ್ನು ಹೊಸ ಸ್ಥಳದಲ್ಲಿ ಅಥವಾ ಹೊಸ ಹೆಸರಿನಿಂದ ಉಳಿಸಲು Ctrl+Shift+S ಒತ್ತಿರಿ ಅಥವಾ ಫೈಲ್ → ಹೆಸರಿನಡಿ ಉಳಿಸಿ… ಆರಿಸಿ.

ಫೈಲ್ ಟ್ರೀ

ಎಡ ಫಲಕ ಪ್ರಸ್ತುತ ರೂಟ್ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿನ .md ಫೈಲ್‌ಗಳು ಮತ್ತು ಚಿತ್ರಗಳ ಸಮತಟ್ಟಾದ ಪಟ್ಟಿ ತೋರಿಸುತ್ತದೆ. ನೇರಳೆ ಆಗಾಗ ಸಂಬಂಧಿತ ಫೈಲ್‌ಗಳಿಲ್ಲದ ಉಪ-ಡೈರೆಕ್ಟರಿಗಳನ್ನು ಅಗ್ಗಿಗಳನ್ನು ಕಡಿಮೆ ಮಾಡಲು ಮರೆಮಾಡಲಾಗಿದೆ. ಫೈಲ್ ತೆರೆಯಲು ಕ್ಲಿಕ್ ಮಾಡಿ. Git-ನಿರ್ವಹಿತ ಫೈಲ್ ತೆರೆದಾಗ ಟ್ರೀ ಹೆಡರ್ ಹಸಿರು ಬಣ್ಣದಲ್ಲಿ [GIT] reponame ತೋರಿಸುತ್ತದೆ.

ಸಲಹೆ

ಸಂಪಾದಕ ಜಾಗ ಗರಿಷ್ಠಗೊಳಿಸಲು Ctrl+B ಯಿಂದ ಫೈಲ್ ಟ್ರೀ ಟಾಗಲ್ ಮಾಡಿ. ನಿಮ್ಮ ಆದ್ಯತೆ ಸೆಶನ್‌ಗಳ ನಡುವೆ ಉಳಿಸಲ್ಪಡುತ್ತದೆ.

8. ಗಣಿತ ಸೂತ್ರಗಳು

MarkForge MathJax ಬಳಸಿ LaTeX ಗಣಿತ ನೋಟೇಶನ್ ರೆಂಡರ್ ಮಾಡುತ್ತದೆ. ಮೊದಲ ಬಳಕೆಯಲ್ಲಿ ಇಂಟರ್ನೆಟ್ ಸಂಪರ್ಕ ಅಗತ್ಯ (MathJax ಅನ್ನು CDN ನಿಂದ ಲೋಡ್ ಮಾಡಲಾಗುತ್ತದೆ).

ಇನ್‌ಲೈನ್ ಗಣಿತ

ಒಂದು ಪ್ಯಾರಾಗ್ರಾಫ್ ಒಳಗೆ ಇನ್‌ಲೈನ್ ಆಗಿ ರೆಂಡರ್ ಮಾಡಲು ಅಭಿವ್ಯಕ್ತಿಯನ್ನು ಒಂದೇ ಡಾಲರ್ ಗುರುತುಗಳಲ್ಲಿ ಸುತ್ತಿ:

The famous equation $E = mc^2$ was published in 1905.

ಫಲಿತಾಂಶ: The famous equation E = mc² was published in 1905.

ಡಿಸ್‌ಪ್ಲೇ (ಬ್ಲಾಕ್) ಗಣಿತ

ತನ್ನದೇ ಸಾಲಿನಲ್ಲಿ ಕೇಂದ್ರೀಕೃತವಾಗಿ ರೆಂಡರ್ ಮಾಡಲು ಅಭಿವ್ಯಕ್ತಿಯನ್ನು ಡಬಲ್ ಡಾಲರ್ ಗುರುತುಗಳಲ್ಲಿ ಸುತ್ತಿ:

$$\int_0^\infty e^{-x}\, dx = 1$$

ಒಂದು-ಸಾಲು ಬ್ಲಾಕ್ ಗಣಿತ

ಡಿಸ್‌ಪ್ಲೇ-ಮ್ಯಾಥ್ ಬ್ಲಾಕ್ ತೆರೆಯಲು ಮತ್ತು ಮುಚ್ಚಲು ತನ್ನದೇ ಸಾಲಿನಲ್ಲಿ ಒಂದೇ $ ಸಹ ಬಳಸಬಹುದು:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
ಸಲಹೆ

MathJax ಭಿನ್ನರಾಶಿಗಳು, ಮ್ಯಾಟ್ರಿಕ್ಸ್‌ಗಳು, ಗ್ರೀಕ್ ಅಕ್ಷರಗಳು, ಆಪರೇಟರ್‌ಗಳು, ಮತ್ತು ಕಸ್ಟಮ್ ಮ್ಯಾಕ್ರೋಗಳ ಸೇರಿದಂತೆ ಸಂಪೂರ್ಣ LaTeX ಮ್ಯಾಥ್ ಮೋಡ್ ಅನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ಸಂಪೂರ್ಣ syntax ಗಾಗಿ MathJax ದಾಖಲೆ ನೋಡಿ.

9. ಸೇರಿಸುವ ಸಂವಾದಗಳು

MarkForge ಸಾಮಾನ್ಯ Markdown ರಚನೆಗಳನ್ನು ಸೇರಿಸಲು ಮಾರ್ಗದರ್ಶಿ ಸಂವಾದಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ. ಎಲ್ಲಾ ಸಂವಾದಗಳು ಸಂಪಾದಕದಲ್ಲಿ ಪ್ರಸ್ತುತ ಕರ್ಸರ್ ಸ್ಥಾನದಲ್ಲಿ ಪಠ್ಯ ಸೇರಿಸುತ್ತವೆ.

ಲಿಂಕ್ ಸೇರಿಸಿ Ctrl+K

ಸೇರಿಸಿ → ಲಿಂಕ್… ಮೂಲಕ ತೆರೆಯಿರಿ ಅಥವಾ Ctrl+K ಒತ್ತಿರಿ.

ಸಂವಾದದಲ್ಲಿ ಮೂರು ಕ್ಷೇತ್ರಗಳಿವೆ:

  • ಪ್ರದರ್ಶನ ಪಠ್ಯ — ಗೋಚರ ಲಿಂಕ್ ಲೇಬಲ್ (ಉದಾ. ಇಲ್ಲಿ ಕ್ಲಿಕ್ ಮಾಡಿ).
  • URL — ಗಮ್ಯ ವಿಳಾಸ (ಉದಾ. https://example.com).
  • ಶೀರ್ಷಿಕೆ (ಐಚ್ಛಿಕ) — ಬಳಕೆದಾರ ಲಿಂಕ್ ಮೇಲೆ ಸುಳಿದಾಡಿದಾಗ ತೋರಿಸುವ tooltip.

ಸಂಪಾದಕದಲ್ಲಿ [ಪ್ರದರ್ಶನ ಪಠ್ಯ](URL "ಶೀರ್ಷಿಕೆ") ಇರಿಸಲು ಸೇರಿಸಿ ಕ್ಲಿಕ್ ಮಾಡಿ.

ಚಿತ್ರ ಸೇರಿಸಿ Ctrl+Shift+K

ಸೇರಿಸಿ → ಚಿತ್ರ… ಮೂಲಕ ತೆರೆಯಿರಿ ಅಥವಾ Ctrl+Shift+K ಒತ್ತಿರಿ.

ಸಂವಾದದಲ್ಲಿ ಎರಡು ಕ್ಷೇತ್ರಗಳಿವೆ:

  • Alt ಪಠ್ಯ — ಸ್ಕ್ರೀನ್ ರೀಡರ್‌ಗಳಿಗಾಗಿ ಸಣ್ಣ ವಿವರಣೆ.
  • URL / ಮಾರ್ಗ — ರಿಮೋಟ್ URL ಅಥವಾ ಸ್ಥಳೀಯ ಫೈಲ್ ಮಾರ್ಗ. ಸ್ಥಳೀಯ ಚಿತ್ರ ಫೈಲ್ ಆರಿಸಲು ಬ್ರೌಸ್ ಬಟನ್ ಬಳಸಿ.

ಸಂಪಾದಕದಲ್ಲಿ ![Alt ಪಠ್ಯ](URL) ಇರಿಸಲು ಸೇರಿಸಿ ಕ್ಲಿಕ್ ಮಾಡಿ.

ಕೋಷ್ಟಕ ಸೇರಿಸಿ Ctrl+Shift+T

ಸೇರಿಸಿ → ಕೋಷ್ಟಕ… ಮೂಲಕ ತೆರೆಯಿರಿ ಅಥವಾ Ctrl+Shift+T ಒತ್ತಿರಿ.

ನಿಮ್ಮ ಕೋಷ್ಟಕ ರಚನೆ ಸಂರಚಿಸಿ:

  • ಡೇಟಾ ಸಾಲುಗಳು — ಹೆಡರ್ ಸಾಲು ಹೊರತುಪಡಿಸಿ ಸಾಲುಗಳ ಸಂಖ್ಯೆ.
  • ಕಾಲಮ್‌ಗಳು — ಕಾಲಮ್‌ಗಳ ಸಂಖ್ಯೆ (ಗರಿಷ್ಠ ಬೆಂಬಲಿತ ಮಿತಿಯವರೆಗೆ).
  • ಕಾಲಮ್ ಹೆಸರುಗಳು — ಪ್ರತಿ ಕಾಲಮ್‌ಗೆ ಶೀರ್ಷಿಕೆ ನಮೂದಿಸಿ.
  • ಜೋಡಣೆ — ಪ್ರತಿ ಕಾಲಮ್‌ಗೆ ಸ್ವತಂತ್ರವಾಗಿ ಎಡ, ಮಧ್ಯ, ಅಥವಾ ಬಲ ಆರಿಸಿ.

ಸಂವಾದದ ಕೆಳಭಾಗದಲ್ಲಿ ನೇರ Markdown ಮುನ್ನೋಟ ನವೀಕರಿಸಲ್ಪಡುತ್ತದೆ. ಸಂಪಾದಕದಲ್ಲಿ ಕೋಷ್ಟಕ ಇರಿಸಲು ಸೇರಿಸಿ ಕ್ಲಿಕ್ ಮಾಡಿ.

10. PDF ವೈಶಿಷ್ಟ್ಯಗಳು

PDF ಆಗಿ ರಫ್ತು Ctrl+Shift+E

ಫೈಲ್ → PDF ಆಗಿ ರಫ್ತು… ಆರಿಸಿ ಅಥವಾ Ctrl+Shift+E ಒತ್ತಿರಿ. MarkForge ನೇರ ಮುನ್ನೋಟದಂತೆಯೇ ಪೈಪ್‌ಲೈನ್ ಬಳಸಿ ಪ್ರಸ್ತುತ ದಾಖಲೆಯನ್ನು PDF ಫೈಲ್‌ಗೆ ರೆಂಡರ್ ಮಾಡುತ್ತದೆ, ಆದರೆ ಬಿಳಿ ಹಿನ್ನೆಲೆ ಮತ್ತು ಮುದ್ರಣ-ಅನುಕೂಲ CSS ಯೊಂದಿಗೆ. ಔಟ್‌ಪುಟ್ ಸ್ಥಳ ಮತ್ತು ಫೈಲ್‌ನಾಮ್ ಆರಿಸಲು ಫೈಲ್ ಉಳಿಸುವ ಸಂವಾದ ಕಾಣಿಸಿಕೊಳ್ಳುತ್ತದೆ.

PDF ಆಮದು Ctrl+Shift+I

  • ಶೀರ್ಷಿಕೆಗಳು ಮತ್ತು ಪ್ಯಾರಾಗ್ರಾಫ್‌ಗಳು
  • ಕೋಷ್ಟಕಗಳು
  • ಪಟ್ಟಿಗಳು
  • ಪತ್ತೆಯಾದ ಕೋಡ್ ಬ್ಲಾಕ್‌ಗಳು

UI ಸ್ಪಂದಿಸುವಂತಿರಲು ಪರಿವರ್ತನೆ ಹಿನ್ನೆಲೆ thread ನಲ್ಲಿ ನಡೆಯುತ್ತದೆ. ಪೂರ್ಣಗೊಂಡ ನಂತರ, MarkForge ಫಲಿತಾಂಶದ .md ಫೈಲ್ ಉಳಿಸಲು ಸ್ಥಳ ಕೇಳುತ್ತದೆ ಮತ್ತು ನಂತರ ಅದನ್ನು ಸಂಪಾದಕದಲ್ಲಿ ತೆರೆಯುತ್ತದೆ.

ಗಮನಿಸಿ

11. Git ಸಮನ್ವಯ

MarkForge ಹೋಸ್ಟ್ ಮಾಡಿದ Git ರೆಪೊಸಿಟರಿಗಳಿಂದ ನೇರವಾಗಿ Markdown ಫೈಲ್‌ಗಳನ್ನು ತೆರೆಯಬಹುದು, ಸಂಪಾದಿಸಬಹುದು, ಮತ್ತು ಬದಲಾವಣೆಗಳನ್ನು ಹಿಂದೆ push ಮಾಡಬಹುದು — ಟರ್ಮಿನಲ್ ತೆರೆಯದೆ. amend ಮತ್ತು squash ನಂತಹ ಸುಧಾರಿತ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನೂ ಬೆಂಬಲಿಸಲಾಗಿದೆ.

ಬೆಂಬಲಿತ ವೇದಿಕೆಗಳು

ವೇದಿಕೆಉದಾಹರಣೆ URL ಮಾದರಿ
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.mycompany.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.mycompany.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

ವೇದಿಕೆಯನ್ನು URL ರಚನೆಯಿಂದ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪತ್ತೆ ಮಾಡಲಾಗುತ್ತದೆ, ಕೇವಲ hostname ಅಲ್ಲ — ಆದ್ದರಿಂದ ಸ್ವಯಂ-ಹೋಸ್ಟ್ ಮಾಡಿದ ಇನ್‌ಸ್ಟಾನ್ಸ್‌ಗಳು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ.

Git ನಿಂದ ಫೈಲ್ ತೆರೆಯುವುದು

  1. ಫೈಲ್ → Git ನಿಂದ ತೆರೆ… ಆರಿಸಿ ಅಥವಾ Ctrl+Shift+G ಒತ್ತಿರಿ.
  2. ತೆರೆಯಲು ಬಯಸುವ ಫೈಲ್‌ನ URL ಅಂಟಿಸಿ (GitHub ಅಥವಾ Bitbucket ಫೈಲ್ ಬ್ರೌಸರ್‌ನಿಂದ ನಕಲಿಸಿ).
  3. MarkForge URL ಮಾನ್ಯ ಮಾಡಿ ಮುನ್ನೋಟ ಸಾಲು ತೋರಿಸುತ್ತದೆ: platform · owner/repo · path/to/file.md.
  4. Ref ಕ್ಷೇತ್ರದಲ್ಲಿ branch, tag, ಅಥವಾ commit hash ಐಚ್ಛಿಕವಾಗಿ ಬದಲಾಯಿಸಿ.
  5. OK ಕ್ಲಿಕ್ ಮಾಡಿ — ರೆಪೊಸಿಟರಿ ಹಿನ್ನೆಲೆಯಲ್ಲಿ clone ಆಗಿ ಫೈಲ್ ಸಂಪಾದಕದಲ್ಲಿ ತೆರೆಯುತ್ತದೆ.

Git-ನಿರ್ವಹಿತ ಫೈಲ್ ತೆರೆದಿರುವಾಗ, ಫೈಲ್ ಟ್ರೀ ಹೆಡರ್ ಹಸಿರು ಬಣ್ಣದಲ್ಲಿ [GIT] reponame ತೋರಿಸುತ್ತದೆ.

ಸಲಹೆ

ಸಾರ್ವಜನಿಕ ರೆಪೊಸಿಟರಿಗಳಿಗೆ ಯಾವುದೇ credentials ಸಂರಚನೆ ಇಲ್ಲದೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ. ಖಾಸಗಿ ರೆಪೊಸಿಟರಿಗಳಿಗೆ, ತೆರೆಯುವ ಮೊದಲು ನೋಟ → ಸೆಟ್ಟಿಂಗ್‌ಗಳು → Git ದೃಢೀಕರಣ ನಲ್ಲಿ ನಿಮ್ಮ credentials ಸಂರಚಿಸಿ.

ಉಳಿಸುವುದು (Commit & Push)

Git-ನಿರ್ವಹಿತ ಫೈಲ್ ತೆರೆದಿರುವಾಗ, Ctrl+S ಒತ್ತಿದಾಗ ಸಾಮಾನ್ಯ ಫೈಲ್ ಉಳಿಸುವ ಬದಲಿಗೆ Git Commit & Push ಸಂವಾದ ತೆರೆಯುತ್ತದೆ. ಸಂವಾದದಲ್ಲಿ ಇವೆ:

  • Commit ಸಂದೇಶ — ಅಗತ್ಯ. ನಿಮ್ಮ ಬದಲಾವಣೆಗಳ ಅರ್ಥಪೂರ್ಣ ವಿವರಣೆ ನಮೂದಿಸಿ.
  • ಹಿಂದಿನ commit ಅನ್ನು amend ಮಾಡಿ — SSH ಅಥವಾ git-binary ದೃಢೀಕರಣ ಬಳಸಿ ಎರಡನೇ ಉಳಿಕೆಯಿಂದ ಮುಂದೆ ಲಭ್ಯ. ಹೊಸ commit ರಚಿಸುವ ಬದಲಿಗೆ ಹಿಂದಿನ commit ಅನ್ನು ಬದಲಾಯಿಸುತ್ತದೆ; ಹಿಂದಿನ commit ಸಂದೇಶ ಮೊದಲೇ ತುಂಬಿಸಲಾಗುತ್ತದೆ; ಸ್ವಯಂಚಾಲಿತವಾಗಿ force-push (--force-with-lease) ಬಳಸುತ್ತದೆ. ಪ್ರಸ್ತುತ ಸೆಶನ್‌ನಲ್ಲಿ MarkForge ಮಾಡಿದ commits ಮಾತ್ರ amend ಮಾಡಬಹುದು.
  • Push ಮಾಡಲು:
    • ಪ್ರಸ್ತುತ branch — clone ಮಾಡಿದ branch ಗೆ ನೇರವಾಗಿ push ಮಾಡುತ್ತದೆ.
    • ಹೊಸ branch — ನೀವು ನಿರ್ದಿಷ್ಟಪಡಿಸಿದ ಹೆಸರಿನಿಂದ ಹೊಸ branch ರಚಿಸುತ್ತದೆ.
  • Pull Request ರಚಿಸಿ — "ಹೊಸ branch" ಆಯ್ಕೆ ಮಾಡಿದಾಗ ಲಭ್ಯ. Push ಮಾಡಿದ ನಂತರ ಸ್ವಯಂಚಾಲಿತವಾಗಿ pull request ತೆರೆಯುತ್ತದೆ. PR ಶೀರ್ಷಿಕೆ ಮತ್ತು ಗುರಿ branch ಗಾಗಿ ಕ್ಷೇತ್ರಗಳನ್ನು ತೋರಿಸುತ್ತದೆ.

Squash Commits Ctrl+Shift+Q

SSH ಮತ್ತು HTTPS (git binary) ದೃಢೀಕರಣದಲ್ಲಿ ಮಾತ್ರ ಲಭ್ಯ.

ಫೈಲ್ → Git Squash… ಆರಿಸಿ ಅಥವಾ Ctrl+Shift+Q ಒತ್ತಿರಿ. Squash Commits ಸಂವಾದ ಪ್ರಸ್ತುತ branch ನಲ್ಲಿ ಆಯ್ಕೆ ಮಾಡಿದ ಬೇಸ್ branch ನಲ್ಲಿ ಇನ್ನೂ ಇಲ್ಲದ ಎಲ್ಲಾ commits ಕೋಷ್ಟಕ ತೋರಿಸುತ್ತದೆ.

  • ಬೇಸ್ branch ಕ್ಷೇತ್ರ — ಉಲ್ಲೇಖ branch (ಉದಾ. main, master, develop). ಬದಲಾಯಿಸಿದ ನಂತರ ಮರುಲೋಡ್ ಮಾಡಿ ಕ್ಲಿಕ್ ಮಾಡಿ.
  • ಎಲ್ಲಾ ಹೊಸ commits ಆಯ್ಕೆ ಮಾಡಿ — ಪಟ್ಟಿ ಮಾಡಿದ ಎಲ್ಲಾ commits ಒಂದೇ ಬಾರಿ ಟಿಕ್ ಮಾಡುತ್ತದೆ.
  • Squash commit ಸಂದೇಶ — ಆಯ್ಕೆ ಮಾಡಿದ ಎಲ್ಲಾ commits ಅನ್ನು ಬದಲಾಯಿಸುವ ಒಂದೇ ಸಂದೇಶ.
  • Squash & Push — branch ಇತಿಹಾಸ ಮರುಬರಹ ಮಾಡಿ remote ಗೆ force-push ಮಾಡುತ್ತದೆ.

ಕನಿಷ್ಠ 2 commits ಆಯ್ಕೆ ಮಾಡಬೇಕು, ಮತ್ತು ಅವು ಅತ್ಯಂತ ಇತ್ತೀಚಿನ commit (HEAD) ನಿಂದ ಪ್ರಾರಂಭವಾಗುವ ನಿರಂತರ ಶ್ರೇಣಿ ರೂಪಿಸಬೇಕು.

ದೃಢೀಕರಣ ವಿಧಾನಗಳು

ನೋಟ → ಸೆಟ್ಟಿಂಗ್‌ಗಳು → Git ದೃಢೀಕರಣ ನಲ್ಲಿ ನಿಮ್ಮ ದೃಢೀಕರಣ ವಿಧಾನ ಸಂರಚಿಸಿ.

HTTPS (ಅಂತರ್ನಿರ್ಮಿತ)

ಯಾವುದೇ ಹೆಚ್ಚುವರಿ ಸಾಫ್ಟ್‌ವೇರ್ ಸ್ಥಾಪನೆಯಿಲ್ಲದೆ ಎಲ್ಲಾ ಪ್ಲಾಟ್‌ಫಾರ್ಮ್‌ಗಳಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ. ವೇದಿಕೆ REST API ನೇರವಾಗಿ ಬಳಸುತ್ತದೆ. ಸ್ಥಳೀಯ git binary ಅಗತ್ಯವಿಲ್ಲ.

ಮಿತಿ

ಈ ವಿಧಾನದಲ್ಲಿ amend ಮತ್ತು squash ವೈಶಿಷ್ಟ್ಯಗಳು ಲಭ್ಯವಿಲ್ಲ ಏಕೆಂದರೆ ಸಂಪೂರ್ಣ ಸ್ಥಳೀಯ git ಇತಿಹಾಸ ನಿರ್ವಹಿಸಲಾಗುವುದಿಲ್ಲ.

HTTPS (git binary)

ಎಲ್ಲಾ ಕಾರ್ಯಾಚರಣೆಗಳಿಗೆ ಸಿಸ್ಟಮ್ git executable ಬಳಸುತ್ತದೆ — ಸಂಪೂರ್ಣ ಸ್ಥಳೀಯ clone, commit, ಮತ್ತು push. Git for Windows (ಅಥವಾ PATH ನಲ್ಲಿ ಯಾವುದೇ git binary) ಅಗತ್ಯ. amend ಮತ್ತು squash ಬೆಂಬಲಿಸುತ್ತದೆ. HTTPS (ಅಂತರ್ನಿರ್ಮಿತ) ವಿಧಾನದಂತೆಯೇ ಬಳಕೆದಾರಹೆಸರು ಮತ್ತು token ಕ್ಷೇತ್ರಗಳು ಬಳಸಿ. Commit ಗುರುತಿಗಾಗಿ ನಿಮ್ಮ ಹೆಸರು ಮತ್ತು Email ಸಹ ನಮೂದಿಸಬೇಕು.

SSH (key file)

SSH (key file) ಆರಿಸಿ ಮತ್ತು ನಿಮ್ಮ ಖಾಸಗಿ SSH key ಮಾರ್ಗ ಮತ್ತು ಐಚ್ಛಿಕ passphrase ನೀಡಿ. ಈ ವಿಧಾನ ಶುದ್ಧ-Python SSH ಸಂಪರ್ಕಕ್ಕಾಗಿ paramiko ಬಳಸುತ್ತದೆ (ಸಿಸ್ಟಮ್ ssh client ಅಗತ್ಯವಿಲ್ಲ). paramiko ಸ್ಥಾಪಿಸದಿದ್ದರೆ, MarkForge ಸಿಸ್ಟಮ್ ssh client ಗೆ ಹಿಂದಿರುಗುತ್ತದೆ.

pip install paramiko

SSH ದೃಢೀಕರಣ amend ಮತ್ತು squash ಬೆಂಬಲಿಸುತ್ತದೆ.

ಪ್ರಮಾಣಪತ್ರ ಸಂಗ್ರಹಣೆ

Tokens, SSH passphrases ಮತ್ತು proxy passwords OS keyring ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗುತ್ತದೆ (Linux ನಲ್ಲಿ GNOME Keyring / KWallet, macOS ನಲ್ಲಿ Keychain, Windows ನಲ್ಲಿ Windows Credential Manager) — ಸಾದಾ-ಪಠ್ಯ ಸಂರಚನಾ ಫೈಲ್‌ಗಳಲ್ಲಿ ಅಲ್ಲ.

HTTP Proxy

ನಿಮ್ಮ ನೆಟ್‌ವರ್ಕ್ Git ಸರ್ವರ್‌ಗಳನ್ನು ತಲುಪಲು HTTP proxy ಅಗತ್ಯಪಡಿಸಿದರೆ, ನೋಟ → ಸೆಟ್ಟಿಂಗ್‌ಗಳು → HTTP Proxy ನಲ್ಲಿ ಸಂರಚಿಸಿ:

  • Proxy URL — ಉದಾ. http://proxy.company.com:8080
  • Proxy ಬಳಕೆದಾರಹೆಸರು — ದೃಢೀಕರಣ ಅಗತ್ಯವಿಲ್ಲದಿದ್ದರೆ ಖಾಲಿ ಬಿಡಿ.
  • Proxy ಪಾಸ್‌ವರ್ಡ್ — ದೃಢೀಕರಣ ಅಗತ್ಯವಿಲ್ಲದಿದ್ದರೆ ಖಾಲಿ ಬಿಡಿ.

ಸಿಸ್ಟಮ್‌ನ ಡಿಫಾಲ್ಟ್ proxy ಸೆಟ್ಟಿಂಗ್‌ಗಳನ್ನು ಬಳಸಲು ಎಲ್ಲಾ proxy ಕ್ಷೇತ್ರಗಳನ್ನು ಖಾಲಿ ಬಿಡಿ.

ಮುಚ್ಚುವುದು

Git-ನಿರ್ವಹಿತ ಫೈಲ್ ತೆರೆದಿರುವಾಗ MarkForge ಮುಚ್ಚಿದಾಗ, MarkForge ತಾತ್ಕಾಲಿಕ clone ಡೈರೆಕ್ಟರಿ ಅಳಿಸಬೇಕೆ ಎಂದು ಕೇಳುತ್ತದೆ. ಡೈರೆಕ್ಟರಿ ಅಳಿಸಿದರೆ ಡಿಸ್ಕ್ ಸ್ಥಳ ಬಿಡುಗಡೆ ಆಗುತ್ತದೆ; ಇರಿಸಿಕೊಂಡರೆ clone ಮಾಡಿದ ಫೈಲ್‌ಗಳನ್ನು ನಂತರ ಪರಿಶೀಲಿಸಬಹುದು.
