Manual do Utilizador MarkForge v1.8.0

1. Introdução

O MarkForge é um editor Markdown moderno e repleto de funcionalidades, construído com Python e PyQt6. Combina uma experiência de escrita sem distrações com uma pré-visualização lado a lado em tempo real, integração completa com fluxo Git, suporte a diagramas, renderização matemática e importação/exportação PDF — tudo numa aplicação de ambiente de trabalho limpa com dois painéis.

Quer esteja a escrever documentação técnica, READMEs de projeto ou a publicar conteúdo diretamente num repositório GitHub, o MarkForge gere todo o ciclo de vida de edição e publicação sem necessidade de um terminal.

2. Instalação

Linux (Debian / Ubuntu)

Descarregue o pacote .deb da página Releases no GitHub e instale-o com:

sudo apt install ./markforge_*.deb

O pacote instala automaticamente todas as dependências de sistema necessárias (python3-pyqt6, python3-pyqt6.qtwebengine, python3-markdown, python3-pygments). Não é necessário ambiente virtual.

Windows

Descarregue MarkForge-Setup.exe da página Releases e execute o instalador. O assistente de instalação guia-o durante a instalação e cria automaticamente atalhos no menu Iniciar.

Nota

O instalador Windows inclui o runtime Python e todas as dependências. Não é necessária instalação separada do Python.

A Partir do Código-Fonte

Requisitos: Python 3.8 ou posterior, CMake 3.16 ou posterior.

Usando CMake (recomendado):

# 1. Clonar o repositório
git clone https://github.com/mmulch/markforge.git
cd markforge

# 2. Configurar a compilação (cria um ambiente virtual local)
cmake -B build

# 3. Instalar dependências Python no venv local
cmake --build build --target install_deps

# 4. Executar a aplicação
cmake --build build --target run

Lançamento direto com Python (sem CMake):

pip install -r requirements.txt
python src/main.py
Dica

Para operações Git baseadas em SSH, instale também paramiko: pip install paramiko

3. Visão Geral da Interface

Esquema

A janela MarkForge está dividida em três painéis redimensionáveis separados por divisórias arrastáveis:

Arraste qualquer alça divisória para redimensionar painéis. Os tamanhos dos painéis são guardados entre sessões.

Estrutura do Documento

O painel Estrutura do Documento fica no topo da barra lateral esquerda e lista todos os títulos (H1–H6) do documento atual como uma árvore navegável.

  • Clique em qualquer título para mover o cursor do editor para essa linha.
  • A estrutura atualiza-se automaticamente enquanto escreve.
  • Alternar visibilidade via Ver → Mostrar estrutura. A preferência é guardada entre sessões.

Barra de Ferramentas

A barra de ferramentas abaixo da barra de menus fornece acesso com um clique às ações mais comuns: Novo, Abrir, Guardar, Inserir Ligação, Inserir Imagem, Inserir Tabela, Inserir PlantUML, Inserir Mermaid, Alternar Pré-visualização e Definições. Todas as ações da barra de ferramentas duplicam itens de menu e atalhos de teclado.

Barra de Estado

A barra de status na parte inferior da janela sempre exibe:

  • Contagem de palavras & tempo de leitura — contagem de palavras e tempo de leitura estimado (ex. 342 palavras · 2 min de leitura), atualizado em tempo real.
  • Progresso da meta de palavras — se uma meta for definida via Exibir → Definir meta de palavras, uma barra de progresso compacta aparece.
  • Posição do cursor — linha e coluna atuais, ex. Linha 12, Col 5.

Título da Janela

O título da janela mostra o nome do ficheiro atualmente aberto. Quando faz alterações não guardadas, um asterisco (*) é adicionado ao nome do ficheiro, por exemplo README.md *. Quando nenhum ficheiro está aberto, o título mostra Sem título.

4. O Editor

O painel central aloja um editor de texto simples com todas as funcionalidades, otimizado para a criação de Markdown.

Realce de Sintaxe

O MarkForge aplica realce de sintaxe consciente de Markdown ao documento enquanto escreve. Títulos, marcadores de ênfase, ligações, spans de código, blocos de código delimitados e outros constructs Markdown são coloridos de forma distinta. O realce adapta-se automaticamente quando alterna entre os temas de editor Claro e Escuro.

Números de Linha

Uma margem no lado esquerdo do editor mostra números de linha. A linha atual é realçada com uma cor de fundo subtil para ajudá-lo a acompanhar a sua posição em documentos longos.

Moldagem de Palavras

A moldagem de palavras está ativada por padrão. Linhas longas moldam-se para caber na largura visível do editor. Alternar moldagem de palavras a qualquer momento via Ver → Moldagem de palavras. A preferência é guardada entre sessões.

Temas do Editor

Dois temas de editor integrados estão disponíveis:

  • Escuro (padrão) — fundo escuro com cores de sintaxe de alto contraste.
  • Claro — fundo branco adequado para ambientes brilhantes.

Alterar temas em Ver → Definições → Tema do editor. A alteração tem efeito imediato sem reiniciar.

Texto de Marcador de Posição

Quando nenhum ficheiro está aberto, o editor está desativado e mostra a mensagem: "Abra ou crie um ficheiro Markdown para começar a editar." O editor ativa-se assim que cria um novo ficheiro ou abre um existente.

Operações de Edição

Todas as operações padrão de edição de texto estão disponíveis:

  • Ctrl+Z — Desfazer
  • Ctrl+Y — Refazer
  • Ctrl+X / Ctrl+C / Ctrl+V — Cortar / Copiar / Colar

Localizar & Substituir

Prima Ctrl+F para abrir a barra de localização, ou Ctrl+H para a abrir com a linha de substituição visível. A barra aparece acima do editor.

  • Escreva no campo de pesquisa — todas as correspondências são realçadas instantaneamente no editor.
  • Navegue entre correspondências com os botões ↑ / ↓, ou prima Enter (seguinte) / Shift+Enter (anterior) no campo de pesquisa.
  • O contador de correspondências mostra a posição atual (por exemplo 3 / 12).
  • Aa — alternar pesquisa sensível a maiúsculas/minúsculas.
  • [W] — alternar correspondência de palavra inteira.
  • Use Substituir para substituir a correspondência atual, ou Substituir Tudo para substituir todas as ocorrências de uma vez.
  • Prima Escape ou clique em ✕ para fechar a barra e limpar todos os realces.

Se houver texto selecionado no editor quando abre a barra, o campo de pesquisa é pré-preenchido com a seleção (apenas seleções de linha única).

Verificação Ortográfica

O MarkForge pode sublinhar palavras mal escritas com uma linha ondulada vermelha enquanto escreve. A verificação ortográfica está desativada por padrão.

  • Ativar via Ver → Verificação ortográfica (alternar).
  • Selecionar o idioma do dicionário via Ver → Idioma da verificação ortográfica. Idiomas disponíveis: English, Deutsch, Español, Français, Italiano, Nederlands, Português, عربي, فارسی, Tiếng Việt, Svenska, Українська, ಕನ್ನಡ, हिंदी. Os idiomas EN/DE/ES/FR/IT/NL/PT/AR/FA utilizam pyspellchecker; VI/SV/UK/KN/HI utilizam enchant/Hunspell.
  • Palavras dentro de spans de código inline (`…`) e URLs nunca são sublinhadas.
  • Abreviações em MAIÚSCULAS (HTML, API, URL…) são ignoradas.
  • A preferência de verificação ortográfica e o idioma selecionado são guardados entre sessões.
  • Requer a biblioteca pyspellchecker. Se não estiver instalada, o alternador é silenciosamente ignorado.

5. Pré-visualização em Direto

O painel direito mostra uma versão HTML totalmente renderizada do seu documento Markdown, atualizada automaticamente enquanto escreve.

Renderização

A pré-visualização usa uma folha de estilos inspirada no GitHub para uma saída limpa e legível. Suporta todas as extensões Markdown incluindo tabelas, listas de tarefas, notas de rodapé, realce de código, matemática e diagramas. O pipeline de renderização processa o seu texto através da biblioteca Python markdown com as extensões extra, codehilite, toc e sane_lists.

Preservação da Posição de Deslocamento

Quando edita o documento, a pré-visualização é atualizada após um curto atraso de 300 ms. Antes de atualizar o HTML, o MarkForge lê a posição de deslocamento atual da vista web e restaura-a após a renderização, para que a pré-visualização não salte para o topo enquanto trabalha no meio de um documento.

Temas de Pré-visualização

Dois temas de pré-visualização estão disponíveis:

  • Escuro (padrão) — fundo escuro a combinar com o tema geral da aplicação.
  • Claro — fundo branco para uma experiência de leitura semelhante à impressão.

Alterar via Ver → Definições → Tema de pré-visualização. A alteração tem efeito imediato.

Alternar a Pré-visualização

Prima Ctrl+Shift+P ou use Ver → Mostrar pré-visualização para ocultar ou mostrar o painel de pré-visualização. Ocultar a pré-visualização dá ao editor mais espaço no ecrã.

Nota

Fórmulas matemáticas (MathJax) e diagramas Mermaid requerem ligação à Internet na primeira utilização, pois as bibliotecas são carregadas de uma CDN. Diagramas PlantUML são sempre obtidos de plantuml.com e requerem acesso à Internet.

6. Operações de Ficheiro

Novo Ficheiro

Prima Ctrl+N ou escolha Ficheiro → Novo. Se o documento atual tiver alterações não guardadas, o MarkForge pedirá que guarde antes de continuar. Um novo documento vazio é aberto e o editor é ativado.

Abrir Ficheiro

Prima Ctrl+O ou escolha Ficheiro → Abrir… para abrir um diálogo de explorador de ficheiros. Ficheiros Markdown (.md, .markdown), ficheiros de texto (.txt) e todos os outros ficheiros são selecionáveis. A árvore de ficheiros será atualizada para mostrar a pasta que contém o ficheiro aberto.

Abrir Pasta

Escolha Ficheiro → Abrir Pasta… (Ctrl+Shift+O) para selecionar um diretório. A árvore de ficheiros mostra então todos os ficheiros .md e imagens nesse diretório. Clique em qualquer ficheiro na árvore para o abrir no editor.

Guardar

Prima Ctrl+S ou escolha Ficheiro → Guardar. Se o ficheiro foi aberto a partir de um repositório Git, aparecerá um diálogo Git commit & push (consulte Guardar via Git). Para ficheiros locais, o ficheiro é escrito no disco imediatamente e o asterisco na barra de título desaparece.

Guardar Como

Prima Ctrl+Shift+S ou escolha Ficheiro → Guardar Como… para guardar o documento atual numa nova localização ou com um novo nome.

Árvore de Ficheiros

O painel esquerdo mostra uma lista plana de ficheiros .md e imagens no diretório raiz atual. Os subdiretórios sem ficheiros diretamente relevantes estão ocultos para reduzir a desordem. Clique num ficheiro para o abrir. O cabeçalho da árvore mostra [GIT] nomerepo a verde quando um ficheiro gerido pelo Git está aberto.

Dica

Alterne a árvore de ficheiros com Ctrl+B para maximizar o espaço do editor. A sua preferência é guardada entre sessões.

Ficheiros Recentes

O MarkForge mantém uma lista dos últimos 10 ficheiros que abriu. Aceda-lhes via Ficheiro → Ficheiros Recentes.

  • Clique em qualquer entrada para reabrir o ficheiro imediatamente.
  • Se um ficheiro já não existir no disco, é removido automaticamente da lista.
  • Escolha Limpar Ficheiros Recentes no topo do submenu para limpar todo o histórico.
  • A lista é guardada entre sessões.

Arrastar & Soltar

Você pode arrastar arquivos do seu gerenciador de arquivos para o editor. MarkForge exibe um diálogo para escolher como tratar cada tipo de arquivo:

  • Arquivos de imagem (.png, .jpg, .gif, etc.) — escolha inserir usando o caminho original ou copiar para a pasta assets/ e inserir um link relativo.
  • Arquivos Markdown (.md, .markdown) — Abrir o arquivo de seu local original ou Copiar para pasta (copia para o mesmo diretório do documento atual e abre a cópia).
  • Arquivos PDF — Importar … (fluxo de importação padrão com diálogo de salvar) ou Importar para pasta (importa automaticamente para o diretório do documento atual).
  • Qualquer outro arquivo — insere um link Markdown [nome do arquivo](caminho/para/arquivo).
Nota

Para operações que copiam ou importam para a pasta do documento atual, um arquivo deve estar aberto e salvo para que MarkForge conheça o diretório de destino.

O MarkForge suporta Markdown CommonMark padrão mais as seguintes extensões: extra (tabelas, notas de rodapé, abreviações), codehilite (blocos de código com realce de sintaxe), toc (índice), sane_lists e rasurado.

8. Fórmulas Matemáticas

O MarkForge renderiza notação matemática LaTeX usando MathJax. É necessária ligação à Internet na primeira utilização (o MathJax é carregado de uma CDN).

Matemática Inline

Envolva uma expressão em sinais de dólar simples para a renderizar inline dentro de um parágrafo:

A famosa equação $E = mc^2$ foi publicada em 1905.

Resultado: A famosa equação E = mc² foi publicada em 1905.

Matemática em Bloco (Centrada)

Envolva a expressão em sinais de dólar duplos para a renderizar centrada na sua própria linha:

$$\int_0^\infty e^{-x}\, dx = 1$$

Matemática em Bloco de Linha Única

Pode também usar um $ sozinho numa linha para abrir e fechar um bloco de matemática de exibição:

$
\sum_{n=1}^{\infty} \frac{1}{n^2} = \frac{\pi^2}{6}
$
Dica

O MathJax suporta o modo matemático LaTeX completo, incluindo frações, matrizes, letras gregas, operadores e macros personalizados. Consulte a documentação do MathJax para a sintaxe completa.

9. Diálogos de Inserção

O MarkForge fornece diálogos guiados para inserir constructs Markdown comuns. Todos os diálogos inserem texto na posição atual do cursor no editor.

Inserir Ligação Ctrl+K

Abrir via Inserir → Ligação… ou prima Ctrl+K.

O diálogo tem três campos:

  • Texto de exibição — a etiqueta de ligação visível (por exemplo Clique aqui).
  • URL — o endereço de destino (por exemplo https://example.com).
  • Título (opcional) — uma dica mostrada quando o utilizador passa o cursor sobre a ligação.

Clique em Inserir para colocar [Texto de exibição](URL "Título") no editor.

Inserir Imagem Ctrl+Shift+K

Abrir via Inserir → Imagem… ou prima Ctrl+Shift+K.

O diálogo tem dois campos:

  • Texto Alternativo — uma breve descrição para leitores de ecrã.
  • URL / Caminho — um URL remoto ou um caminho de ficheiro local. Use o botão de pesquisa para escolher um ficheiro de imagem local.

Clique em Inserir para colocar ![Texto alternativo](URL) no editor.

Inserir Tabela Ctrl+Shift+T

Abrir via Inserir → Tabela… ou prima Ctrl+Shift+T.

Configure a estrutura da sua tabela:

  • Linhas de dados — número de linhas excluindo a linha de cabeçalho.
  • Colunas — número de colunas (até ao máximo suportado).
  • Nomes das colunas — introduza um título para cada coluna.
  • Alinhamento — escolha Esquerda, Centro ou Direita para cada coluna independentemente.

Uma pré-visualização Markdown em direto é atualizada na parte inferior do diálogo. Clique em Inserir para colocar a tabela no editor.

Inserir PlantUML Ctrl+Shift+U

Abrir via Inserir → PlantUML… ou prima Ctrl+Shift+U.

Escolha entre 12 modelos de diagrama no menu suspenso:

  • Diagrama de sequência
  • Diagrama de classes
  • Diagrama de atividade
  • Diagrama de estado
  • Diagrama de componentes
  • Diagrama de caso de uso
  • Diagrama de Gantt
  • Mapa mental
  • WBS (Estrutura analítica do trabalho)
  • Diagrama de temporização
  • Diagrama de implantação
  • JSON

O modelo selecionado preenche o editor de código PlantUML. Edite o código conforme necessário e clique em Inserir. O diagrama é envolvido num bloco de código delimitado e renderizado via o serviço online plantuml.com na pré-visualização. É necessária ligação à Internet.

Nota

A pré-visualização do diagrama aparece automaticamente na janela principal após a inserção. A renderização usa o servidor público plantuml.com; não é necessária instalação local de PlantUML.

Inserir Mermaid Ctrl+Shift+M

Abrir via Inserir → Mermaid… ou prima Ctrl+Shift+M.

Escolha entre 9 modelos de diagrama:

  • Fluxograma
  • Diagrama de sequência
  • Diagrama de classes
  • Diagrama de estado
  • Gantt
  • Gráfico circular
  • Mapa mental
  • Diagrama ER
  • Gráfico Git

Edite o código Mermaid na área de texto e clique em Inserir. O diagrama é envolvido num bloco de código mermaid delimitado e renderizado na pré-visualização via kroki.io. É necessária ligação à Internet.

10. Funcionalidades PDF

Exportar como PDF Ctrl+Shift+E

Escolha Ficheiro → Exportar como PDF… ou prima Ctrl+Shift+E. O MarkForge renderiza o documento atual para um ficheiro PDF usando o mesmo pipeline que a pré-visualização em direto, mas com fundo branco e CSS otimizado para impressão. Aparece um diálogo de guardar para que possa escolher a localização de saída e o nome do ficheiro.

Dica

Mude a pré-visualização para o tema Claro antes de exportar para PDF para a saída mais limpa pronta para impressão.

Exportar como HTML

Escolha Ficheiro → Exportar como HTML…. O MarkForge renderiza o documento atual para um ficheiro HTML autossuficiente com todo o CSS integrado e imagens locais convertidas em URI de dados Base64, para que o ficheiro possa ser aberto em qualquer browser sem ligação à internet.

Exportar via Pandoc (DOCX, EPUB, LaTeX)

Escolha Ficheiro → Exportar como DOCX (Pandoc)…, Exportar como EPUB (Pandoc)… ou Exportar como LaTeX (Pandoc)…. Estas opções requerem que o pandoc esteja instalado e disponível no PATH. Se o pandoc não for encontrado, é mostrado um diálogo de aviso.

Importar PDF Ctrl+Shift+I

Escolha Ficheiro → Importar PDF… ou prima Ctrl+Shift+I. O MarkForge converte qualquer ficheiro PDF para Markdown usando a biblioteca pymupdf4llm. A conversão preserva inteligentemente a estrutura do documento:

  • Títulos e parágrafos
  • Tabelas
  • Listas
  • Blocos de código onde detetados

11. Integração Git

O MarkForge pode abrir ficheiros Markdown diretamente de repositórios Git alojados, permitir-lhe editá-los e enviar alterações de volta — sem nunca abrir um terminal. Operações avançadas como amend e squash também são suportadas.

Plataformas Suportadas

PlataformaExemplo de padrão de URL
GitHubhttps://github.com/owner/repo/blob/main/README.md
GitHub Enterprisehttps://github.minhaempresa.com/owner/repo/blob/main/docs/guide.md
Bitbucket Cloudhttps://bitbucket.org/owner/repo/src/main/README.md
Bitbucket Server / Data Centerhttps://bitbucket.minhaempresa.com/projects/PROJ/repos/myrepo/browse/README.md?at=main

A plataforma é detetada automaticamente a partir da estrutura da URL, não apenas do nome de host — por isso instâncias auto-alojadas funcionam automaticamente.

Abrir um Ficheiro do Git

  1. Escolha Ficheiro → Abrir a partir do Git… ou prima Ctrl+Shift+G.
  2. Cole o URL do ficheiro que quer abrir (copie-o do explorador de ficheiros GitHub ou Bitbucket).
  3. O MarkForge valida o URL e mostra uma linha de pré-visualização: plataforma · owner/repo · caminho/para/ficheiro.md.
  4. Opcionalmente substitua o ramo, tag ou hash de commit no campo Ref.
  5. Clique em OK — o repositório é clonado em segundo plano e o ficheiro abre no editor.

Enquanto um ficheiro gerido pelo Git está aberto, o cabeçalho da árvore de ficheiros mostra [GIT] nomerepo a verde.

Dica

Repositórios públicos funcionam sem credenciais configuradas. Para repositórios privados, configure as suas credenciais em Ver → Definições → Autenticação Git antes de abrir.

Guardar (Commit & Push)

Quando um ficheiro gerido pelo Git está aberto, premir Ctrl+S abre o diálogo Git Commit & Push em vez do guardar normal. O diálogo contém:

  • Mensagem de commit — obrigatória. Introduza uma descrição significativa das suas alterações.
  • Emendar commit anterior — disponível a partir do segundo guardar quando se usa autenticação SSH ou git-binary. Substitui o commit anterior em vez de criar um novo; pré-preenche a mensagem do commit anterior; usa automaticamente force-push (--force-with-lease). Apenas commits feitos pelo MarkForge na sessão atual podem ser emendados.
  • Enviar para:
    • Ramo atual — envia diretamente para o ramo a partir do qual clonou.
    • Novo ramo — cria um novo ramo com um nome que especifica.
  • Criar Pull Request — disponível quando "Novo ramo" está selecionado. Abre automaticamente um pull request após o envio. Mostra campos para título do PR e ramo alvo.

Squash de Commits Ctrl+Shift+Q

Disponível apenas com autenticação SSH e HTTPS (git binary).

Escolha Ficheiro → Git Squash… ou prima Ctrl+Shift+Q. O diálogo Squash mostra uma tabela de todos os commits no ramo atual que ainda não estão no ramo base escolhido.

  • Campo do ramo base — o ramo de referência (por exemplo main, master, develop). Clique em Recarregar após alterá-lo.
  • Selecionar todos os novos commits — marca todos os commits listados de uma vez.
  • Mensagem do commit squash — a mensagem única que substituirá todos os commits selecionados.
  • Squash & Push — reescreve o histórico do ramo e faz force-push para o repositório remoto.

Deve selecionar pelo menos 2 commits, e devem formar um intervalo contíguo a partir do commit mais recente (HEAD).

Modos de Autenticação

Configure o seu método de autenticação em Ver → Definições → Autenticação Git.

HTTPS (incorporado)

Funciona em todas as plataformas sem software adicional instalado. Usa a API REST da plataforma diretamente. Não é necessário git binary local.

Limitação

As funcionalidades amend e squash não estão disponíveis neste modo porque não é mantido um histórico git local completo.

PlataformaTipo de token
GitHubToken de acesso pessoal (ghp_…) — requer âmbito repo
GitHub EnterpriseToken de acesso pessoal — igual ao GitHub
Bitbucket CloudSenha de app (Conta → Senhas de app)
Bitbucket ServerToken de acesso pessoal ou token de acesso HTTP

HTTPS (git binary)

Usa o executável git do sistema para todas as operações — clone local completo, commit e push. Requer Git para Windows (ou qualquer binário git no seu PATH). Suporta amend e squash. Use os mesmos campos de nome de utilizador e token do modo HTTPS (incorporado). Deve também fornecer o seu Nome e Email para identidade de commit.

SSH (ficheiro de chave)

Selecione SSH (ficheiro de chave) e forneça o caminho para a sua chave SSH privada e uma frase-passe opcional. Este modo usa paramiko para uma ligação SSH pura em Python (não é necessário cliente ssh de sistema). Se o paramiko não estiver instalado, o MarkForge recorre ao cliente ssh do sistema.

pip install paramiko

A autenticação SSH suporta amend e squash.

Armazenamento de Credenciais

Tokens, frases-passe SSH e palavras-passe de proxy são armazenadas no porta-chaves do SO (GNOME Keyring / KWallet no Linux, Keychain no macOS, Gestor de Credenciais do Windows no Windows) — não em ficheiros de configuração em texto simples.

Proxy HTTP

Se a sua rede requer um proxy HTTP para aceder a servidores Git, configure-o em Ver → Definições → Proxy HTTP:

  • URL do proxy — por exemplo http://proxy.empresa.com:8080
  • Nome de utilizador do proxy — deixe vazio se não for necessária autenticação.
  • Palavra-passe do proxy — deixe vazio se não for necessária autenticação.

Deixe todos os campos do proxy vazios para usar as definições de proxy padrão do sistema.

Fechar

Quando fecha o MarkForge enquanto um ficheiro gerido pelo Git está aberto, o MarkForge perguntará se quer eliminar o diretório de clone temporário. Eliminar o diretório libera espaço em disco; mantê-lo permite-lhe inspecionar os ficheiros clonados posteriormente.
